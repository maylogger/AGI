# Discord Brand Guidelines

來源：`sources/Discord_BrandGuideline_2021.pdf`
版本：1.0
日期：2021
研究視角：Design Infrastructure Researcher

## Summary

Discord 2021 品牌規範，將產品中的社群、對話、狀態與歸屬感轉譯為語氣、標誌、色彩、字體與應用系統。建立 Discord 是「每個人都能找到歸屬」的社群平台，而不是單一功能產品。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。品牌系統必須從產品行為抽取，而不是在產品外另造裝飾語言。product-inspired design system：profile circles → community；user status → focus state；conversation → copy system。

執行上，這份 guideline 透過 Introduction、Tone of Voice、Logo, Symbol, Wordmark and Tagline、Logo System 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。產品品牌可以把核心 UI primitives 轉為外部行銷語言，形成產品與品牌一致的閉環。

## Introduction

1. **Chapter Purpose**：建立 Discord 是「每個人都能找到歸屬」的社群平台，而不是單一功能產品。
2. **Definition**：定義 what is Discord、mission、vision、positioning、tone of voice 與 messaging variations；不直接處理 logo 或版面細節。
3. **Relationship**：此章把產品的聊天、社群、個人狀態與歸屬感轉成整套設計系統的概念來源。
4. **Design Execution**：用幽默提問、potluck metaphor、Imagine a Place 敘事與品牌價值語氣建立可被 copy 與 visual 同時引用的世界觀。
5. **Principles**：品牌系統必須從產品行為抽取，而不是在產品外另造裝飾語言。
6. **Frameworks**：product-inspired design system：profile circles → community；user status → focus state；conversation → copy system。
7. **Patterns**：適合社群產品的 identity pattern：把產品 UI 元件抽象成品牌溝通圖形。
8. **Rules**：文案應可映射到 playful、original、relatable、reliable；不應聽起來像一般科技平台；需保留人味、玩笑與可理解性。
9. **Reusable Knowledge**：產品品牌可以把核心 UI primitives 轉為外部行銷語言，形成產品與品牌一致的閉環。
10. **Abstraction**：`Brand Strategy`、`Product Language`、`Voice and Tone`。
11. **Visual Evidence**：此章使用對話式排版、profile circle 與 product-inspired diagram，功能是證明品牌源自產品。

### Tone of Voice / Messaging Variations

1. **Chapter Purpose**：把「像 Discord」的寫作方式拆成可訓練與可檢查的語氣層級。
2. **Definition**：tone 包含 playful、original、relatable、reliable；messaging 分為 brand lines、user quotes、product value。
3. **Relationship**：它把 mission 的 belonging 轉成 copywriting 工具，並支撐後續 brand in use 的範例。
4. **Design Execution**：以社群梗、日常荒謬感與功能價值混合，讓文案像朋友對話而非平台公告。
5. **Principles**：語氣要能反映用戶，而不只是品牌自說自話。
6. **Frameworks**：三層 messaging：universal Discord feeling、community-specific lingo、functional product value。
7. **Patterns**：可複用為社群平台的 copy matrix：情感層、社群層、功能層。
8. **Rules**：品牌文案應至少符合一個 messaging tier；user quote 不應被寫成企業口吻；product value 必須用 playful twist 呈現而非乾燥功能列點。
9. **Reusable Knowledge**：將語氣與訊息來源分離，可避免所有文案都變成同一種「品牌腔」。
10. **Abstraction**：`Messaging`、`Content Design`、`Voice and Tone`。
11. **Visual Evidence**：頁面展示多組短句與分類圖，功能是提供文案分類與語氣樣本。

## Logo, Symbol, Wordmark and Tagline

1. **Chapter Purpose**：保護 Discord 最可識別的 logo 系統，同時讓 Clyde、wordmark 與 Imagine a Place 可在不同版面中靈活運作。
2. **Definition**：定義 logo、small logo、Clyde、Clyde expressions、wordmark、tagline、placement、partner lockups 與 usage rules。
3. **Relationship**：承接 Introduction 的人格，將 Clyde 從單純 icon 變成可表情化角色。
4. **Design Execution**：以 preferred / fallback 層級定義 Blurple、white、black 的版本選擇，並用版面比例定義 tagline 與 logo 鎖定。
5. **Principles**：高辨識品牌需要清楚版本階層；角色化 icon 可增加情緒表達，但不可破壞主標誌。
6. **Frameworks**：preferred option → fallback option → monochrome exception；placement by format：portrait、landscape、wide、animation。
7. **Patterns**：logo 系統同時包含 brandmark、small logo、avatar-like symbol、tagline lockup，是 digital-first 品牌常見架構。
8. **Rules**：小尺寸低於 80px x 15px 使用 small logo；不可修改、扭曲或重畫 logo；partner logos 需以 1px white stroke 分隔；tagline 預設分成兩行。
9. **Reusable Knowledge**：品牌可把 format-based placement 規則寫成可供生成器選擇的 layout decision tree。
10. **Abstraction**：`Visual Identity`、`Layout System`、`Legal and Trademark`。
11. **Visual Evidence**：大量 logo、Clyde、placement 與 successful / unsuccessful examples，功能是比例、禁用與版面證據。

### Placement and Usage

1. **Chapter Purpose**：把 logo 與 tagline 的「放哪裡」變成 format-aware 的決策規則。
2. **Definition**：處理 portrait、landscape、extra-wide、animated、partner lockup 與 successful / unsuccessful usage。
3. **Relationship**：它將前面的 logo 元件轉成實際輸出物驗收條件。
4. **Design Execution**：不同畫幅有不同預設位置：portrait 下方置中、landscape 右下、extra-wide 右側垂直置中。
5. **Principles**：logo placement 需要同時考量可見性、主視覺空間與畫幅比例。
6. **Frameworks**：format → position → lockup orientation → exception。
7. **Patterns**：以畫幅比例驅動識別 placement，可用於自動版面生成。
8. **Rules**：動畫與 masked composition 應置中組合 tagline + logo；extra-wide tagline 可單行；不可 stack logo elements 或使用非核准配色。
9. **Reusable Knowledge**：把「版面情境」列入規則，比單一固定位置更適合跨媒介品牌。
10. **Abstraction**：`Layout System`、`Motion Identity`。
11. **Visual Evidence**：頁面以多格示意呈現不同畫幅，功能是流程與情境證據。

## Brand Colors

1. **Chapter Purpose**：建立以 Blurple 為核心、連接產品 UI 與行銷溝通的色彩系統。
2. **Definition**：定義 Blurple、完整品牌色盤、配色建議、禁用配色、scheme、UI colors、Community Pattern。
3. **Relationship**：色彩將 Discord 產品狀態與社群活力帶入外部品牌輸出。
4. **Design Execution**：以 Blurple 作品牌 equity，green / red 連回產品狀態，yellow / fuchsia 表現使用者內容多彩性。
5. **Principles**：色彩不只追求活潑，還必須維持可讀性與 UI 真實性。
6. **Frameworks**：core equity color + product status colors + user-content accents + contrast rules。
7. **Patterns**：可複用為 product-led color system：品牌主色負責辨識，產品狀態色負責語義。
8. **Rules**：Blurple HEX `5865F2`；避免 White+Yellow、Fuchsia+Red、Blurple+Fuchsia、Fuchsia+Green、Yellow+Green；animated communication 不得混用多套色彩方案。
9. **Reusable Knowledge**：色彩規範可明確列出「可用組合」與「禁用組合」，便於 AI 審稿與 accessibility 檢查。
10. **Abstraction**：`Color System`、`Design Tokens`、`Accessibility`。
11. **Visual Evidence**：色票、配色矩陣、禁用例、UI 狀態示例與 Community Pattern 範例是主要證據。

## Typography

1. **Chapter Purpose**：建立可辨識、緊湊、情緒化但仍可讀的文字系統。
2. **Definition**：定義 Ginto Nord、Whitney、fallback fonts、headline / paragraph styles、line heights、text lockups、quotes、3D type。
3. **Relationship**：字體承接 playful purposeful 的 positioning，讓 headline 有個性、body copy 保持可讀。
4. **Design Execution**：Ginto Nord 用於大寫 headline，Whitney 用於段落；不同長度 headline 使用不同 line-height。
5. **Principles**：字體系統需要同時規範字體、字重、行距、行長、對齊與特殊標點處理。
6. **Frameworks**：headline intensity scale：Ultra、Primary、Secondary；paragraph legibility rule：135% line-height、50-75 characters。
7. **Patterns**：微觀排版規則，例如手動調整逗號 baseline，顯示品牌對字形品質的治理深度。
8. **Rules**：headline 一律大寫；paragraph line-height 135%；段落每行約 50-75 字元；不可 warp text、混合句中字體樣式或在 static layout 使用 motion distortion。
9. **Reusable Knowledge**：大型品牌的 typography guideline 應包含 micro-typography，而不只指定字體。
10. **Abstraction**：`Typography System`、`Content Design`、`Motion Identity`。
11. **Visual Evidence**：字樣 specimen、line-height 比較、text lockup、quote styling 與成功 / 失敗案例提供可視化驗收證據。

## Brand in Use

1. **Chapter Purpose**：展示前面規則如何組裝成實際品牌輸出，讓使用者從範例學習風格邏輯。
2. **Definition**：包含 brand lines、community lines、user quotes、chat dialog、audio dialog、end cards、stationery 與 asset library。
3. **Relationship**：這是 design system 的整合驗證，將 messaging、color、type、logo、UI-inspired elements 放進實際情境。
4. **Design Execution**：以品牌短句、使用者 quote、聊天截面與 end card 展示 Discord feeling。
5. **Principles**：使用範例不是裝飾，而是系統規則在真實格式中的證明。
6. **Frameworks**：message source → layout type → brand asset combination。
7. **Patterns**：可複用為品牌手冊最後的 pattern library，讓抽象規範轉為可模仿範例。
8. **Rules**：brand lines 應能承載 Imagine a Place；user quotes 應包含使用者標籤或 avatar 情境；chat dialog 應保持產品感且不可扭曲 UI 語義。
9. **Reusable Knowledge**：產品品牌的應用章可把 UI 截面轉成品牌情境模組，強化產品與行銷一致性。
10. **Abstraction**：`Examples`、`Product Language`、`UI Components`。
11. **Visual Evidence**：大量 mockup、chat dialogue、end card 與 stationery，功能是示範、比較與情境證據。
