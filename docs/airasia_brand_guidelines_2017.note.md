# AirAsia Brand Guidelines: Visual Elements

來源：`sources/airasia_brand_guidelines_2017.pdf`
日期：2017
研究視角：Design Infrastructure Researcher

## Summary

這份短版 visual elements guideline 聚焦 AirAsia 的 logo、tagline、clear space、錯誤用法與色彩系統。用一句話界定 AirAsia 視覺識別的管理範圍，讓後續規則被視為整體品牌外觀系統，而非單一 logo 文件。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。品牌可見性來自一致的視覺元素組合，而不是孤立資產。視覺元素可分為 identity mark、verbal mark、image style、color、type、signature asset。

執行上，這份 guideline 透過 Visual Elements Overview、Logo System、Logo Construction and Minimum Size、Wordmark Usage and Extreme Formats 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。品牌 guideline 的 opening page 可作為 scope contract，決定哪些元素屬於治理範圍。

## Visual Elements Overview

1. **Chapter Purpose**：用一句話界定 AirAsia 視覺識別的管理範圍，讓後續規則被視為整體品牌外觀系統，而非單一 logo 文件。
2. **Definition**：列出 logo、tagline、photography、colour、typography、signature mark；本 PDF 實際深入處理 logo、tagline 與 colour。
3. **Relationship**：作為目錄型入口，先告訴使用者「世界如何看見我們」是此 guideline 的核心任務。
4. **Design Execution**：以大標題與短句宣告 visual elements 的品牌代表性。
5. **Principles**：品牌可見性來自一致的視覺元素組合，而不是孤立資產。
6. **Frameworks**：視覺元素可分為 identity mark、verbal mark、image style、color、type、signature asset。
7. **Patterns**：短版 guideline 可先建立元素 inventory，再選擇最易誤用的元素深入規範。
8. **Rules**：若文件聲稱管理 visual elements，至少應標示受控元素清單；AI 可檢查輸出素材是否使用清單內元素。
9. **Reusable Knowledge**：品牌 guideline 的 opening page 可作為 scope contract，決定哪些元素屬於治理範圍。
10. **Abstraction**：Knowledge Architecture、Visual Identity。
11. **Visual Evidence**：開頭頁以大字與元素列表建立範圍證據，但沒有細節圖例。

## Logo System

1. **Chapter Purpose**：保護 AirAsia 最核心識別在不同背景、比例、材質與 AirAsia X 變體中的一致性。
2. **Definition**：定義 logo 構成、最小尺寸、wordmark 特殊情境、clear space、極端比例裁切與禁止用法。
3. **Relationship**：此章是 visual elements 中最完整的治理模組，後續 tagline 與 colour 都圍繞 logo 可讀性服務。
4. **Design Execution**：以正確應用、比例標示、clear space diagram 與 don’ts 建立操作邊界。
5. **Principles**：logo 不可被改造；可讀性、留白與品牌完整性優先於版面便利。
6. **Frameworks**：使用情境 → 完整 logo 或 wordmark；背景 → white/black/red 應用；空間 → 最小尺寸與留白判斷。
7. **Patterns**：logo governance 常由 construction、minimum size、clear space、exception、misuse 五件組構成。
8. **Rules**：logo 最小尺寸 20mm；clear space 至少為 logo 半徑 1/2；不得旋轉、加 .com、分別縮放元素、換字體、改色、拉伸、加特效或裁切角落。
9. **Reusable Knowledge**：若允許極端比例裁切，必須同時提供數學比例，否則例外會破壞整個系統。
10. **Abstraction**：Visual Identity、Layout System、Governance、AI-Checkable Rules。
11. **Visual Evidence**：多頁使用 logo 圖、量測線、1/8 與 1/2 radius 標記，以及錯誤示例。

### Logo Construction and Minimum Size

1. **Chapter Purpose**：確定 logo 是不可拆解或重製的品牌資產。
2. **Definition**：AirAsia logo 由 wordmark 與 red holding space 組成，並可在白、黑、紅背景上應用。
3. **Relationship**：為 clear space、wordmark exception 與 don’ts 提供原始構成基準。
4. **Design Execution**：用元素標註與 20mm 最小尺寸建立檔案使用底線。
5. **Principles**：品牌識別應使用正式 artwork，不應由執行者自行重畫。
6. **Frameworks**：資產完整性框架：不可 alter、modify、recreate。
7. **Patterns**：最小尺寸把可讀性從主觀判斷轉為可量測規範。
8. **Rules**：輸出 logo 任一方向不得低於 20mm 指示基準；不得修改 wordmark 或 red circle/red X 的關係。
9. **Reusable Knowledge**：logo guideline 應同時說明資產組成與可視尺寸下限。
10. **Abstraction**：Visual Identity、AI-Checkable Rules。
11. **Visual Evidence**：頁面使用 logo 組成與 20mm 標示做直接證據。

### Wordmark Usage and Extreme Formats

1. **Chapter Purpose**：在空間不足時提供可控例外，避免使用者自行壓縮完整 badge。
2. **Definition**：wordmark 可在特殊情況單獨使用，例如 on-ground activation materials 或 T-shirts。
3. **Relationship**：這是 logo system 的例外管理，不是開放任意拆分。
4. **Design Execution**：用 extreme horizontal/vertical format 的 1/8 版面比例與裁切圖示規範可見度。
5. **Principles**：例外應提升辨識度，而不是讓品牌資產失去一致性。
6. **Frameworks**：空間受限 → 檢查 badge 是否不可讀 → 使用 wordmark → 套用 1/8 比例。
7. **Patterns**：在 guideline 中明確寫出例外情境，是比單純禁止更可執行的治理方式。
8. **Rules**：極端水平格式 clear/crop 參考垂直長度的 1/8；極端垂直格式參考水平長度的 1/8。
9. **Reusable Knowledge**：品牌系統應管理「合法例外」，否則現場設計會創造非正式 workaround。
10. **Abstraction**：Layout System、Operations、Governance。
11. **Visual Evidence**：比例格與 cropped wordmark 圖示提供例外邊界。

### Clear Space

1. **Chapter Purpose**：確保 logo 在複雜版面中仍能獨立被看見。
2. **Definition**：logo 周圍必須至少保留半徑 1/2 的呼吸空間，AirAsia X 同樣適用。
3. **Relationship**：支撐 logo construction 的完整性，並影響 tagline 與 CTA 的空間設計。
4. **Design Execution**：以半徑單位與 examples 展示不同版面中的留白。
5. **Principles**：留白是識別可讀性的一部分，不是裝飾空間。
6. **Frameworks**：logo geometry → derive spacing unit → apply across layouts。
7. **Patterns**：用資產自身幾何產生 spacing token，是跨尺寸可延展的規範模式。
8. **Rules**：任何外部元素不得進入 logo 半徑 1/2 的 clear space；AI 可用 bounding boxes 檢查碰撞。
9. **Reusable Knowledge**：clear space 應用相對單位比固定 px 更適合多媒介品牌。
10. **Abstraction**：Layout System、Design Tokens、AI-Checkable Rules。
11. **Visual Evidence**：頁面以 1、1/2、breathing space 標註直接示範。

### Logo Don’ts

1. **Chapter Purpose**：用具體反例封鎖最容易發生的 logo 破壞行為。
2. **Definition**：禁止旋轉、加 .com、分別縮放、換字體、改色、拉伸、特效、裁邊、裁角與 AirAsia X 錯誤拆分。
3. **Relationship**：補強正向 logo 規則，使審核者有明確拒絕依據。
4. **Design Execution**：以「BE NICE!」語氣降低法規感，同時用多格錯誤示範建立記憶點。
5. **Principles**：品牌親和語氣可以與嚴格治理並存。
6. **Frameworks**：錯誤類型可分為 transform、addition、proportion、typeface、color、effects、crop。
7. **Patterns**：don’ts 應與可檢查的視覺變形類別對應。
8. **Rules**：logo 不得旋轉角度非 0、不得出現 `.com` 附加、不得非等比縮放、不得套用陰影或效果、不得被版面邊界裁切。
9. **Reusable Knowledge**：AI 審稿可先檢測幾何變形與額外文字，再進一步檢測色彩與背景。
10. **Abstraction**：Governance、AI-Checkable Rules。
11. **Visual Evidence**：此節以錯誤圖例為核心，是典型警示型 visual evidence。

## Logo and Tagline

1. **Chapter Purpose**：把品牌口號與網站 CTA 的關係固定，避免 tagline 被孤立使用或失去轉換路徑。
2. **Definition**：規定 `Now Everyone Can Fly` 必須與 `airasia.com` 搭配，並有顏色、最小尺寸、clear space 與 grid system 用法。
3. **Relationship**：承接 logo system 的留白邏輯，延伸到 verbal identity 與 call-to-action。
4. **Design Execution**：使用黑白 tagline 以維持對比，並用 40mm 與 grid examples 控制版面。
5. **Principles**：品牌理念需要與可行動入口相連，否則只是口號。
6. **Frameworks**：Tagline → CTA → logo spacing，用 grid system 1/2 管理不同版面。
7. **Patterns**：口號與 URL 綁定是航空與服務品牌常見的 awareness-to-action 模式。
8. **Rules**：tagline 使用時必須 partnered with `airasia.com`；最小尺寸 40mm；文字應用黑或白以確保對比。
9. **Reusable Knowledge**：verbal identity guideline 應規範 slogan 是否可獨立使用、是否需搭配 CTA 或網址。
10. **Abstraction**：Messaging、Layout System、AI-Checkable Rules。
11. **Visual Evidence**：頁面以 tagline、URL、1/2 與 2/8 spacing 圖例說明。

### Tagline Usage

1. **Chapter Purpose**：保護 tagline 作為品牌 mantra 的正確呈現。
2. **Definition**：`Now Everyone Can Fly` 在廣告材料中需與 `airasia.com` 成組出現。
3. **Relationship**：把品牌口號與轉換入口關聯，支援行銷溝通。
4. **Design Execution**：用黑白色在實色或影像上維持最大對比，並設 40mm 最小尺寸。
5. **Principles**：口號必須清楚可讀，且應指向可行動的品牌入口。
6. **Frameworks**：背景複雜度 → 選黑/白；尺寸 → 不低於 40mm；用途 → 必須有 URL。
7. **Patterns**：tagline lockup 是 verbal identity 與 digital destination 的結合。
8. **Rules**：不得單獨在 advertising materials 使用 tagline；不得低於 40mm；不得使用低對比色。
9. **Reusable Knowledge**：口號若承擔 CTA 功能，應被視為 layout component 而非純文案。
10. **Abstraction**：Messaging、UI Components、Governance。
11. **Visual Evidence**：tagline 與 URL 圖例提供最小尺寸與組合證據。

### CTA and Tagline Clear Space

1. **Chapter Purpose**：確保 call to action 不被周邊視覺干擾，提升行動入口辨識。
2. **Definition**：定義 logo、CTA、tagline 在 grid system 1 與 2 下的留白關係。
3. **Relationship**：將 logo clear space 原則延伸到訊息與 CTA 群組。
4. **Design Execution**：以 1/2、1x、2/8 等比例單位建立不同 grid 的版面規格。
5. **Principles**：CTA 需要獨立呼吸空間，才不會在廣告材料中失去效力。
6. **Frameworks**：不同 grid system 對應不同 spacing token，但核心都是避免元素互相侵入。
7. **Patterns**：用版面比例而非固定數值描述 CTA spacing，可適應多尺寸廣告。
8. **Rules**：CTA 周圍需保留 guideline 圖示的 1/2 或 1x clear space；不得讓其他元素侵入。
9. **Reusable Knowledge**：call-to-action guideline 應把文案與空間視為同一個元件。
10. **Abstraction**：Layout System、Interaction Design、AI-Checkable Rules。
11. **Visual Evidence**：grid system 圖示是此節的主要證據。

## Colour System

1. **Chapter Purpose**：用有限色彩建立高度辨識，並控制次要色不稀釋 signature red。
2. **Definition**：主色為 AirAsia Red、黑、白；次要色提供變化但有比例限制。
3. **Relationship**：色彩支撐 logo 與 tagline 對比，也構成 AirAsia 品牌 tone and manner。
4. **Design Execution**：提供 Pantone、CMYK、RGB 數值，並對次要色設定不超過 50% 的使用上限。
5. **Principles**：signature color 是品牌資產，secondary palette 只能輔助而非取代。
6. **Frameworks**：Primary palette → baseline identity；secondary palette → variety under cap；special launches → exception path。
7. **Patterns**：色彩 guideline 應同時定義數值、角色、比例限制與例外情境。
8. **Rules**：搭配主色時 secondary colours 不應超過 layout 50%；AirAsia Red 使用 Pantone 485C、RGB 255/0/0。
9. **Reusable Knowledge**：色彩比例上限是比「use sparingly」更可執行的治理方式。
10. **Abstraction**：Color System、Design Tokens、AI-Checkable Rules。
11. **Visual Evidence**：色票與數值表提供可直接轉成 design token 的證據。

### Primary Colour Palette

1. **Chapter Purpose**：確保任何材料都能再現 AirAsia 最核心的紅、黑、白識別。
2. **Definition**：定義 AirAsia Red、Black、White 的 Pantone、CMYK、RGB。
3. **Relationship**：為 logo、tagline 與整體品牌 tone 提供色彩基準。
4. **Design Execution**：以色票加色值表達跨媒介規格。
5. **Principles**：主色必須穩定且可測量。
6. **Frameworks**：print 用 Pantone/CMYK；screen 用 RGB。
7. **Patterns**：每個色彩 token 應包含 name、role、print value、screen value。
8. **Rules**：AirAsia Red = Pantone 485C、C0 M95 Y100 K0、R255 G0 B0；Black = C0 M0 Y0 K100、R0 G0 B0；White = R255 G255 B255。
9. **Reusable Knowledge**：色彩 token 應避免只有品牌名稱，需提供可驗證數值。
10. **Abstraction**：Color System、Design Tokens。
11. **Visual Evidence**：色票與數值表是明確 evidence。

### Secondary Colour Palette

1. **Chapter Purpose**：允許品牌在特定活動、產品或航線中增加變化，同時避免失去 red-led identity。
2. **Definition**：定義多個 Pantone secondary colours 與使用比例限制。
3. **Relationship**：它補充 primary palette，但不能在一般情況下取代 signature red。
4. **Design Execution**：用「not exceeding 50% of the layout」把 moderating guidance 量化。
5. **Principles**：彈性必須有上限，否則品牌一致性會被例外侵蝕。
6. **Frameworks**：一般材料：secondary ≤ 50%；特殊 product/service/route launch：可作主 palette。
7. **Patterns**：secondary palette 的例外需綁定特殊商業事件。
8. **Rules**：若非特殊 launch，secondary colours 面積不應超過版面 50%；特殊 launch 使用需有情境理由。
9. **Reusable Knowledge**：可把 color dominance ratio 納入 AI 檢查，判斷主色是否被削弱。
10. **Abstraction**：Color System、Governance、AI-Checkable Rules。
11. **Visual Evidence**：次要色色票與 50% 文句構成比例規則證據。
