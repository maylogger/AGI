# Huawei Visual Identity Guidelines

來源：`sources/Huawei_Visual_Identity_Guidelines.pdf`  
版本：V 2.0  
研究視角：Design Infrastructure Researcher

## Summary

這份文件更新 Huawei 企業視覺識別，涵蓋品牌使命、logo、色彩、Huawei Line、插畫、影像、字體與多種實體、數位、展覽、標識應用。用極短篇幅建立 VI 系統的策略源頭，讓後續大量規格可回扣到品牌使命與屬性。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。全球 ICT 品牌的識別需同時表現技術可信度、開放合作與以客戶為中心。品牌策略以 mission statement + six attributes 的輕量結構支撐後續系統，不在此處展開長篇敘事。

執行上，這份 guideline 透過 Huawei Brand、Vision and Mission、Brand Attributes、Visual Identity System 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。企業 VI 可以用少量策略詞作為全文件的 semantic anchor。

## 1.0 Huawei Brand

1. **Chapter Purpose**：用極短篇幅建立 VI 系統的策略源頭，讓後續大量規格可回扣到品牌使命與屬性。
2. **Definition**：本章定義願景使命與品牌屬性；不描述具體設計語言。
3. **Relationship**：它是第 2 章 visual identity system 的判斷依據，尤其支撐 focus、innovation、robustness、harmony 的 logo rationale。
4. **Design Execution**：作者把品牌使命濃縮為「fully connected, intelligent world」，再列出 customer-centric、innovative、global、dedicated、open、trusted 作為執行詞彙。
5. **Principles**：全球 ICT 品牌的識別需同時表現技術可信度、開放合作與以客戶為中心。
6. **Frameworks**：品牌策略以 mission statement + six attributes 的輕量結構支撐後續系統，不在此處展開長篇敘事。
7. **Patterns**：對大型企業 VI，策略章可極簡，因為核心工作是把策略嵌入後續每個可複製資產。
8. **Rules**：任何識別應用需能合理支持六項屬性之一；若視覺語言削弱 trusted、global 或 customer-centric，應退回檢查。
9. **Reusable Knowledge**：企業 VI 可以用少量策略詞作為全文件的 semantic anchor。
10. **Abstraction**：`Brand Strategy`、`Knowledge Architecture`。
11. **Visual Evidence**：本章幾乎只用文字與屬性列舉，視覺證據少，功能是建立決策背景。

## 2.0 Visual Identity System

1. **Chapter Purpose**：建立可全球複製的基礎識別零件庫，確保 logo、red、line、illustration、imagery、typeface 在各事業與媒介中一致。
2. **Definition**：本章定義 corporate logo、brand color、Huawei Line、illustrations、imagery style、typography；不處理單一版面成品的全部內容。
3. **Relationship**：它是第 3-8 章所有 stationery、advertising、digital、print、exhibition、signage application 的設計 tokens 與 rules 來源。
4. **Design Execution**：作者用官方 artwork、X-based proportion、色值、背景控制、禁用圖、授權場景與錯誤示例，把企業識別轉成可審核資產。
5. **Principles**：彈性來自系統化版本與應用規則，而不是任意修改；紅色在應用中被嚴格控制為 accent，以提升識別強度而避免過度使用。
6. **Frameworks**：基礎系統可分為 identity asset、color decision、secondary graphic、illustration taxonomy、imagery style、typeface stack。
7. **Patterns**：每個基礎元素採「rationale → approved versions → measurement → background control → incorrect usage → application example」的治理模板。
8. **Rules**：不得重畫或修改 corporate logo；logo symbol 高度為 X 時 clear space 為 0.35X；horizontal lock-up 最小印刷寬度 25mm、數位 100px；flat red symbol vertical lock-up 最小 8mm、50px；背景複雜或對比不足時需改用 monotone black 或 reversed white；co-branding 必須取得授權並送 Brand Management System 審核；logo 不得改色、變形、單獨使用 symbol、加入部門名、裝飾化或置於非 Huawei 產品。
9. **Reusable Knowledge**：完整 VI 系統應同時提供 design tokens、decision logic、approval workflow 與 anti-patterns。
10. **Abstraction**：`Visual Identity`、`Color System`、`Illustration System`、`Photography System`、`Typography System`、`AI-Checkable Rules`。
11. **Visual Evidence**：本章高度依賴 logo grids、比例圖、色票、背景矩陣、禁用範例、插畫與影像樣式圖，視覺證據承擔規格、比較、警示與示範功能。

## 3.0 Stationery Application

1. **Chapter Purpose**：把基礎識別套用到最常見、最制度化的企業文具，讓日常行政接觸點也維持品牌一致。
2. **Definition**：本章定義 namecard、letterhead、envelope、fax sheet、email signature、ID card、invitation、greeting card、paper cup、notebook、pens、memo pad、folder、paper bag 等。
3. **Relationship**：它直接使用第 2 章 logo、色彩、字體與 layout rules，是基礎系統進入營運物料的第一層。
4. **Design Execution**：作者以固定尺寸、logo 位置、版面示例與印刷品類別建立可生產模板。
5. **Principles**：文具是企業身份的高頻低風險 touchpoint，但若失控會讓全球公司顯得碎片化。
6. **Frameworks**：stationery 可用 template type、fixed field、logo placement、type hierarchy、production constraint 檢查。
7. **Patterns**：對每種品項提供示例版面，可將品牌系統轉成 procurement-ready asset。
8. **Rules**：所有文具應使用官方 logo artwork、指定字體與指定色彩；email signature 與 ID 類資訊欄位需保持一致；辦公用品不得自行改造 logo 或使用非授權色彩。
9. **Reusable Knowledge**：營運物料章節應把品牌規範連接到採購與行政流程。
10. **Abstraction**：`Operations`、`Stationery System`、`Layout System`。
11. **Visual Evidence**：本章以模板與實物 mockup 為主，視覺證據承擔尺寸、生產與一致性示範功能。

## 4.0 Advertising Application

1. **Chapter Purpose**：把識別系統用於高曝光、高變異的廣告場景，避免創意自由破壞品牌辨識。
2. **Definition**：本章定義廣告版面原則、full bleed image、print advertising、co-branding advertising、billboard、poster、online 與 social media advertising。
3. **Relationship**：它延續第 2 章的 logo placement、color、Huawei Line、imagery 與 typography，並與 co-branding logo rules 呼應。
4. **Design Execution**：作者以 layout extension 和 media-specific examples 建立可擴展的廣告版型，而非單一模板。
5. **Principles**：廣告應在創意、媒體尺寸與品牌識別之間取得穩定平衡，尤其要避免 logo 與紅色元素過度競爭。
6. **Frameworks**：廣告審核可分為主視覺、logo margin、red accent、typography、co-branding treatment、媒體尺寸適配。
7. **Patterns**：從 regular layout 推展到 non-regular layout，是建立 responsive brand layout 的模式。
8. **Rules**：廣告中的 corporate logo placement、present size 與 margin 需依版面比例計算；co-branding 廣告需遵守授權、標誌等高、分隔線與優先位置；社群與線上廣告需確保小尺寸仍可辨識。
9. **Reusable Knowledge**：高變異媒介需要提供計算規則，而不只是展示漂亮範例。
10. **Abstraction**：`Layout System`、`Advertising System`、`Co-branding`、`AI-Checkable Rules`。
11. **Visual Evidence**：本章使用 print、billboard、poster、online、social examples，視覺證據用於尺寸適配、情境與比例驗證。

## 5.0 Digital Multi-media

1. **Chapter Purpose**：把 VI 擴展到動態、互動與簡報媒介，避免數位場景與傳統平面識別割裂。
2. **Definition**：本章定義 logo animation、website loading、website homepage logo、social media profile、EDM 與 PowerPoint templates。
3. **Relationship**：它承接第 2 章的 logo、色彩、字體與社群 profile rules，並補足時間性與互動媒介。
4. **Design Execution**：作者提供多種 PowerPoint 模板情境，例如 regular、meeting/presentation、speech/display、launch/keynote。
5. **Principles**：數位識別需同時考慮動態呈現、載入狀態、小尺寸頭像與企業簡報的場景差異。
6. **Frameworks**：digital system 可分為 motion identity、loading state、web header identity、social avatar、EDM layout、presentation mode。
7. **Patterns**：針對簡報情境提供不同視覺版本，是企業內部與外部溝通常見模板治理 pattern。
8. **Rules**：社群 profile 應遵守 square/round area 的比例與留白；PowerPoint 應使用指定模板與字體；EDM 不得使用不符合 logo background control 的背景；logo animation 不得改變 logo 形態或比例。
9. **Reusable Knowledge**：數位章節應包含動態、載入、社群與簡報，不應只把平面 logo 搬到螢幕。
10. **Abstraction**：`Motion Identity`、`Digital Governance`、`Presentation System`。
11. **Visual Evidence**：本章以動畫幀、網站示意、profile 示意、EDM 與簡報模板為視覺證據，承擔情境與狀態示範功能。

## 6.0 Print Application

1. **Chapter Purpose**：把企業識別應用到長篇、資訊密度高的出版品，確保報告、白皮書與文件即使內容不同仍像同一品牌。
2. **Definition**：本章定義 cover/back cover、inside page grid、annual report、leaflet、manual、white paper、tender document、marketing publications、internal publications、corporate brochure。
3. **Relationship**：它依賴第 2 章的 logo、red、line、imagery、typography，並將之轉為 editorial layout system。
4. **Design Execution**：作者建立封面、封底、內頁 grid 與內容類型對應的範例，處理長文件一致性。
5. **Principles**：印刷出版物需要比單張廣告更強的資訊架構，品牌識別需服務閱讀節奏而非只做裝飾。
6. **Frameworks**：print application 可分為 cover identity、inside grid、content hierarchy、publication type、co-branding print。
7. **Patterns**：以 grid system 管理內頁，是將品牌從封面延伸到資訊架構的關鍵模式。
8. **Rules**：封面與內頁需遵守指定 grid；co-branding publications 需遵守共同品牌標誌規格；tender、white paper 等正式文件應優先確保清晰度與專業可信感；不得任意更改 Huawei Line 與 logo 比例。
9. **Reusable Knowledge**：出版型 guideline 應包含內頁系統，不只提供封面範例。
10. **Abstraction**：`Editorial Design`、`Grid System`、`Information Design`。
11. **Visual Evidence**：本章以封面、內頁 grid、報告與出版物 mockup 為主，視覺證據承擔資訊架構與版面一致性功能。

## 7.0 Exhibition and Display Application

1. **Chapter Purpose**：把品牌識別延伸到三維與大型展示環境，確保展場、旗幟、背板與車輛的遠距辨識。
2. **Definition**：本章定義 pull up banner、outdoor flag、backdrop、podium stand、equipment plate、red element in booth design、exhibition vehicle。
3. **Relationship**：它使用 logo size、red accent、imagery、line 與 layout rules，但需重新處理空間尺度與觀看距離。
4. **Design Execution**：作者把紅色元素、標誌位置與展示載具轉成展場視覺規則。
5. **Principles**：展覽系統要在複雜環境中創造一致、醒目、可遠距辨識的企業 presence。
6. **Frameworks**：可用 viewing distance、asset scale、red element proportion、logo visibility、booth context 檢查。
7. **Patterns**：將品牌紅作為環境中的可控 accent，而不是大面積任意鋪色，是大型展示的可複用 pattern。
8. **Rules**：展場物料需使用官方 logo artwork；紅色元素應按規範比例出現；backdrop、podium、vehicle 不得讓 logo 被遮擋、變形或置於低對比背景；equipment plate 需保持最小尺寸與可讀性。
9. **Reusable Knowledge**：環境應用需要把平面 rules 轉換成距離、尺度與材質約束。
10. **Abstraction**：`Environmental Graphics`、`Exhibition System`、`Spatial Branding`。
11. **Visual Evidence**：本章以展架、旗幟、背板、講台、展車與 booth 示例為主，視覺證據承擔情境、尺度與遠距辨識功能。

## 8.0 Signage and Wayfinding Application

1. **Chapter Purpose**：處理企業建築、接待與外部導視等高可信度空間接觸點，確保品牌在實體場所中被一致識別。
2. **Definition**：本章定義 external wayfinding signage、corporate reception、subsidiaries/research institutes reception 與 building facade sign。
3. **Relationship**：它是 VI 系統的空間治理層，依賴 logo、色彩與材質控制，也連接企業總部與各地機構的一致性。
4. **Design Execution**：作者將標誌、接待區與建築立面作為標準化節點，以降低各地空間設計的差異。
5. **Principles**：導視與建築標識必須優先確保可辨識、專業、耐久與一致，創意自由低於品牌可信度。
6. **Frameworks**：signage 可分為 exterior wayfinding、reception identity、facade sign、subsidiary adaptation。
7. **Patterns**：總部與子公司使用同一識別邏輯但允許場地適配，是全球企業導視治理模式。
8. **Rules**：建築與接待標識不得改造 corporate logo；需符合背景對比與最小尺寸；標誌應在高可見位置，且不得被材質、光線或複雜背景削弱；特殊場地需諮詢 Brand Management System。
9. **Reusable Knowledge**：空間導視規範應處理「同一品牌在不同建築條件下如何保持一致」。
10. **Abstraction**：`Signage System`、`Wayfinding`、`Spatial Branding`、`Governance`。
11. **Visual Evidence**：本章以外部導視、接待區與建築立面示例為主，視覺證據承擔空間情境與治理證明功能。
