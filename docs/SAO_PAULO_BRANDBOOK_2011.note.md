# São Paulo Brandbook

來源：`sources/SAO_PAULO_BRANDBOOK_2011.pdf`
版本：v1.4 Digital English
日期：2011-12
研究視角：Design Infrastructure Researcher

## Summary

São Paulo 旅遊城市品牌書，從品牌支柱、品牌本質與策略視角，延伸到 Logo、圖樣、攝影、字體、色彩、品牌並置規則、應用範例與電子原稿命名。先宣告 São Paulo 品牌不是單一廣告活動，而是可被城市利益關係人共同使用的信任與一致性基礎。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。開放使用與嚴格治理並存；越多人使用，越需要規則。shared destination brand -> trustworthiness -> consistent application -> better communication performance。

執行上，這份 guideline 透過 Presentation、Brand Pillars、Brand Essence and Strategic View、Visual Elements 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。公共城市品牌應明確處理「誰可使用」與「為何仍需嚴格」的張力。

## Presentation

1. **Chapter Purpose**：先宣告 São Paulo 品牌不是單一廣告活動，而是可被城市利益關係人共同使用的信任與一致性基礎。
2. **Definition**：本章定義品牌書的用途：協助任何建立 São Paulo 品牌的人，以嚴格原則強化城市溝通。
3. **Relationship**：它把後續品牌支柱、視覺元素與應用規則放在「城市品牌公共基礎設施」脈絡中。
4. **Design Execution**：用城市目的地的敘事把品牌使用權擴展給多方，但同時強調 strict principles。
5. **Principles**：開放使用與嚴格治理並存；越多人使用，越需要規則。
6. **Frameworks**：shared destination brand -> trustworthiness -> consistent application -> better communication performance。
7. **Patterns**：城市品牌 guideline 常以「共同推廣工具」開場，建立公共品牌的合法性。
8. **Rules**：AI 可檢查準則：任何 São Paulo 品牌應用都應服務於城市作為 unique destination 的可信推廣；若不遵守本書原則，應視為降低品牌一致性的風險。
9. **Reusable Knowledge**：公共城市品牌應明確處理「誰可使用」與「為何仍需嚴格」的張力。
10. **Abstraction**：`Brand Strategy`、`Governance`、`Destination Branding`。
11. **Visual Evidence**：開場以文字為主，功能是治理與目的宣告。

## Brand Pillars

1. **Chapter Purpose**：把城市的抽象吸引力拆成可溝通的情感與社會支柱，避免視覺系統只剩觀光符號。
2. **Definition**：本章定義 unexpected、respect differences、live in the present、communities 等核心敘事。
3. **Relationship**：這些支柱支撐 brand essence「experience it all」、攝影風格的不尋常角度與 Logo 動態變化。
4. **Design Execution**：作者用城市生活、差異、創新、社群與照片 credit 交錯，讓品牌從真實城市性格中抽取。
5. **Principles**：São Paulo 的魅力來自對比、多元、現在感與社群，而非單一地標。
6. **Frameworks**：emotional surprise -> diversity ethics -> present pleasure -> community reconnection。
7. **Patterns**：城市品牌可用「社會洞察 + 品牌回應」作為支柱結構。
8. **Rules**：AI 可檢查準則：品牌內容應至少對應 unexpected、diversity、present experience 或 community connection 其中一項；不得只呈現通用觀光明信片感而缺少 surprising perspective。
9. **Reusable Knowledge**：Destination guideline 應把城市價值翻成影像、文字與應用判斷的 criteria。
10. **Abstraction**：`Brand Strategy`、`Messaging`、`Photography System`。
11. **Visual Evidence**：大量照片、關鍵詞與頁面節奏承擔情感證據功能，展現城市多元與意外感。

## Brand Essence and Strategic View

1. **Chapter Purpose**：將支柱收斂成「experience it all」品牌本質，並界定旅遊、商業、授權商品等使用範圍。
2. **Definition**：定義 Brand Proposal、Tagline、Brand Potential、Brand Expression 與 Brand Appreciation。
3. **Relationship**：向前整合 Brand Pillars，向後支撐 Visual Elements 與 Application Rules。
4. **Design Execution**：用 subjective language 與 objective language 分離內容/語氣與視覺/規格，建立完整品牌表達架構。
5. **Principles**：城市品牌價值必須同時被感受與被看見。
6. **Frameworks**：brand values -> subjective content/message + objective visual identity -> brand appreciation -> desire for experiences。
7. **Patterns**：把品牌表達分成主觀與客觀語言，是可重用的品牌系統框架。
8. **Rules**：AI 可檢查準則：São Paulo 品牌應連結 tourism、business、licensed products 或 city advantage；應傳達 experience it all，不應只做抽象城市標記。
9. **Reusable Knowledge**：品牌本質應同時導出市場用途、內容語氣與視覺規範。
10. **Abstraction**：`Brand Strategy`、`Knowledge Architecture`、`Messaging`。
11. **Visual Evidence**：使用策略圖、品牌價值詞與主觀/客觀表達圖，功能是框架證據。

## Visual Elements

1. **Chapter Purpose**：把 experience it all 的城市策略轉成六個可組合的視覺元素，形成高變化但可治理的系統。
2. **Definition**：本章定義 Logo、colors、typography、secondary graphics、photography、pattern，以及它們在不同應用中的組合。
3. **Relationship**：承接策略視角，並為 Application Rules、Examples 與 Electronic Originals 提供可用元件。
4. **Design Execution**：作者將 Logo 版本、動態、圖樣、攝影、字體與色彩一一規格化，並保留可變性代表城市移動與驚喜。
5. **Principles**：城市品牌的可變性必須來自預先定義的元件，而不是任意自由設計。
6. **Frameworks**：six-element visual system + controlled variation + example-based validation。
7. **Patterns**：可變 Logo、pattern、secondary graphics 與攝影風格共同構成 destination brand 的 flexible identity。
8. **Rules**：AI 可檢查準則：應用可使用一個或多個六大視覺元素，但元素需來自本書與 electronic originals；Logo 可變位置只能使用預定 symbol positions；不得任意改變透視、顏色、字體比例或清楚度。
9. **Reusable Knowledge**：彈性識別系統需要「可變範圍」與「不可變限制」同時明確。
10. **Abstraction**：`Visual Identity`、`Visual Language`、`Design Tokens`。
11. **Visual Evidence**：本章大量使用 Logo、色票、照片、字體 specimen、pattern 與禁用圖，功能是示範、比較、警示與 token 化。

### Logo Versions and Care

1. **Chapter Purpose**：保護 São Paulo Logo 在多語、多背景、多媒介與動態使用中的完整性。
2. **Definition**：涵蓋 preferred logo、symbol positions、colored background version、description versions、restricted use、tagline versions、motion、non-interference area、minimum reduction 與 inappropriate use。
3. **Relationship**：Logo 是 visual elements 的主體，後續 association rules 都依其權重與模組 P 測量。
4. **Design Execution**：以可變 symbol positions 表達城市變動，以固定電子原稿控制變動邊界。
5. **Principles**：Logo 可以動、可以換位置、可以多語，但必須遵守預設版本與比例。
6. **Frameworks**：use case -> preferred/description/restricted/tagline/motion -> background/space/size -> prohibited operations。
7. **Patterns**：可變城市 Logo 需使用官方 variation set，而不是開放設計者自行變形。
8. **Rules**：AI 可檢查準則：description versions 只在需表達 cidade de/city of/ciudad de 時使用；restricted versions 僅例外且需 brand management approval；tagline 字體必須 Museo Sans 900；tagline 與 São Paulo words 對齊，距離至少為 P 高度一半；non-interference area 至少等於 letter P 高度；最小一般印刷尺寸為 10mm；不得 distortion、改形式/透視、加外框、浮水印、改 symbol position、改色、低對比背景、旋轉、陰影/特效、改 typography proportions。
9. **Reusable Knowledge**：可變 Logo 必須建立變體庫、clear space、最小尺寸、背景版本與禁用規則。
10. **Abstraction**：`Logo System`、`Motion Identity`、`AI-Checkable Rules`。
11. **Visual Evidence**：使用正負版、多語描述、tagline、motion frame、clear space 與禁用示意作為強視覺證據。

### Patterns and Secondary Graphics

1. **Chapter Purpose**：提供非 Logo 的視覺能量，使品牌能生成機構與促銷材料。
2. **Definition**：定義 preferred pattern、secondary monochromatic pattern 與可變 secondary graphics。
3. **Relationship**：延伸 Brand Pillars 的活力與城市多面性，補充 Logo 無法承擔的版面表現。
4. **Design Execution**：用 pattern 建立 texture 與 framing，用 secondary graphics 建立透視與動態感。
5. **Principles**：輔助圖形可變，但視覺邏輯不可變。
6. **Frameworks**：official originals -> pattern usage -> monochrome variants -> perspective rule。
7. **Patterns**：可用圖樣與透視圖形作為 flexible identity 的二級語彙。
8. **Rules**：AI 可檢查準則：patterns 必須使用 electronic originals；secondary graphics 可改變，但所有 forms 必須 converge to a single point 且 perspectives remain unaffected。
9. **Reusable Knowledge**：輔助圖形比 Logo 更適合承擔創意彈性，但需透視或構圖規則約束。
10. **Abstraction**：`Visual Language`、`Illustration System`、`Layout System`。
11. **Visual Evidence**：以 pattern application 與 secondary graphic 圖示展示 texture、framing 與透視證據。

### Photographic Style

1. **Chapter Purpose**：避免城市影像落入觀光 cliché，讓照片直接傳達「從新角度體驗 São Paulo」。
2. **Definition**：定義 unusual angles、chromatic pattern、overlays、overexposure、lomography、lightpainting，以及 ordinary photo adaptation。
3. **Relationship**：與 Brand Pillars 的 unexpected、contrast、diversity、present pleasure 直接相連。
4. **Design Execution**：用色彩覆蓋與 Photoshop blending 技術將一般照片轉成品牌風格。
5. **Principles**：攝影不是紀錄地標，而是創造新視角與情緒。
6. **Frameworks**：subject matter -> unusual perspective -> chromatic adaptation -> brand effect。
7. **Patterns**：城市品牌可用後製技法將素材庫統一成品牌語言。
8. **Rules**：AI 可檢查準則：照片應避免 cliché，優先 unusual angles；應符合 brand chromatic pattern；可使用 overlay、hard light、overexposure、lomography、lightpainting；普通照片需符合上述標準才可改作。
9. **Reusable Knowledge**：攝影 guideline 應同時定義拍攝視角與後製方式。
10. **Abstraction**：`Photography System`、`Visual Language`。
11. **Visual Evidence**：以原始照片、adapted photograph、color layers 與 blending diagrams 作為技術證據。

### Typography and Colors

1. **Chapter Purpose**：建立資訊傳達的字體與城市記憶色彩，讓品牌兼具可讀性與地點性。
2. **Definition**：定義 Museo Sans 100/500/700/900、Arial 例外、MASP Red、Liberdade Yellow、Ibirapuera Green、Paulista Blue、Tietê Blue。
3. **Relationship**：Typography 支撐訊息清楚，Colors 支撐 pattern、photography、background 與城市地標聯想。
4. **Design Execution**：用字體 specimen 與跨媒介色值將感性城市命名轉成可用 token。
5. **Principles**：色彩可以帶地點記憶，但必須具備印刷與電子媒介對應值。
6. **Frameworks**：type family -> weights -> exception font -> color names -> Pantone/CMYK/RGB/HEX。
7. **Patterns**：以城市地標命名色彩，將 token 與文化記憶綁定。
8. **Rules**：AI 可檢查準則：主要字體為 Museo Sans；presentation、email、digital materials 可例外用 Arial；MASP Red Hex #ea1b36、Liberdade Yellow #ffc808、Ibirapuera Green #a6ce39、Paulista Blue #0088cf、Tietê Blue #003e6c。
9. **Reusable Knowledge**：色彩 token 若與地點敘事連結，能提升 guideline 的記憶性與教育效果。
10. **Abstraction**：`Typography System`、`Color System`、`Design Tokens`。
11. **Visual Evidence**：字體 specimen 與色票是主要視覺證據。

## Application Rules

1. **Chapter Purpose**：解決城市品牌與政府、旅遊公司、合作夥伴、第三方並用時的權重與授權問題。
2. **Definition**：本章定義 convergence scenarios、authorization、foreground/background brand weight、association order、separator 與最大高度。
3. **Relationship**：把 Visual Elements 放入真實品牌關係與政治/商業場景。
4. **Design Execution**：作者依 geography、interest ownership 與 user type 決定 São Paulo 品牌或其他品牌在 foreground。
5. **Principles**：公共品牌的權重應依溝通利益歸屬調整，而不是永遠最大。
6. **Frameworks**：actor type + geography + whose interest -> brand hierarchy。
7. **Patterns**：城市品牌需要比一般企業更多的 co-branding governance。
8. **Rules**：AI 可檢查準則：使用 São Paulo brand 前需寄信至 marcasp@spturis.com 取得 authorization；São Paulo Turismo 在國內外使用時 São Paulo brand 在 foreground，其他品牌 background equal weights；第三方需事先授權。
9. **Reusable Knowledge**：多方公共品牌應建立「情境式權重矩陣」。
10. **Abstraction**：`Governance`、`Brand Architecture`、`Compliance`。
11. **Visual Evidence**：以 convergence scenarios 圖與範例應用展示權重與場景。

### Convergence Scenarios

1. **Chapter Purpose**：在 São Paulo Turismo、Government and Entities、Partners、Third Parties 四類使用者中建立清楚權重。
2. **Definition**：依 abroad、in Brazil defending São Paulo interests、in Brazil defending individual interests 決定 foreground/background。
3. **Relationship**：是 Association Rules 的情境前置條件。
4. **Design Execution**：用分類表與範例頁降低政治與商業合作中的判斷歧義。
5. **Principles**：品牌優先權應反映溝通代表誰的利益。
6. **Frameworks**：user group -> geography -> interest -> brand weight。
7. **Patterns**：destination brand 的共同使用可以用情境矩陣治理。
8. **Rules**：AI 可檢查準則：Government/Entities 或 Partners 在國外與在巴西維護 São Paulo interests 時，São Paulo brand foreground、其他品牌 background smaller weights；在巴西維護 individual interests 時，partner/government brand foreground、São Paulo background smaller weight。
9. **Reusable Knowledge**：co-branding 不只看合作對象，也看地理與利益主體。
10. **Abstraction**：`Brand Architecture`、`Governance`。
11. **Visual Evidence**：以範例頁展示不同主體權重，功能是情境比較。

### Association Rules

1. **Chapter Purpose**：將多品牌並列從策略權重轉成實際左右排序、分隔線與高度規則。
2. **Definition**：涵蓋 governmental brands、other brands、event brand 的 signature order 與 size。
3. **Relationship**：承接 convergence scenarios 並規範實際簽名區。
4. **Design Execution**：以政府層級、市場品牌與活動品牌分三種 rules。
5. **Principles**：多品牌併列需要秩序與視覺公平，不能依版面臨時排列。
6. **Frameworks**：association type -> order -> separator -> max height。
7. **Patterns**：用 module P 或 logo height 管理其他品牌最大高度。
8. **Rules**：AI 可檢查準則：政府品牌 signature 左至右為 São Paulo、City、State、Federal Government，且 São Paulo 與 governmental brands 間必須有 line；與其他品牌並列時右至左為 São Paulo Brand、Other Brands，其他品牌最大高度 3P；與 event brand 並列時右至左為 Event Brand、São Paulo Brand，event brand 最大高度等於 São Paulo brand height。
9. **Reusable Knowledge**：co-branding rule 應給出方向、順序、分隔與最大高度四類資料。
10. **Abstraction**：`Layout System`、`Brand Architecture`、`AI-Checkable Rules`。
11. **Visual Evidence**：以 dummy logos、分隔線與最大高度標註作為可測量視覺證據。

## Examples of Application

1. **Chapter Purpose**：展示前述規則在多種傳播平台中如何形成一致但多變的城市品牌。
2. **Definition**：本章主要是範例參數，未新增大量規格；用途是驗證與教育。
3. **Relationship**：承接 Visual Elements 與 Application Rules，讓使用者看到 practice what you have learned 的結果。
4. **Design Execution**：透過連續視覺頁展示 logo、color、pattern、photography 與 layout 的組合。
5. **Principles**：範例不是裝飾，而是 best practices 的視覺證據。
6. **Frameworks**：platform example -> element combination -> concept verification。
7. **Patterns**：大型 guideline 應以應用範例降低抽象規則的理解成本。
8. **Rules**：AI 可檢查準則：新應用應能對照 examples 是否保留 experience it all 概念、六大元素的一致使用與品牌權重規則。
9. **Reusable Knowledge**：範例頁可作為訓練資料，但應和規則頁分離，不要把單一範例誤當唯一模板。
10. **Abstraction**：`Examples`、`Visual Language`。
11. **Visual Evidence**：此章視覺證據最強，主要承擔示範、情境與綜合驗證功能。

## Electronic Originals

1. **Chapter Purpose**：把複雜的 Logo、tagline、pattern、secondary graphics 與 motion assets 管理成可搜尋、可授權、可選擇的檔案系統。
2. **Definition**：定義 SP file code、brand identifier、symbol variation、color type、version code、appearance、file format，以及各資產列表。
3. **Relationship**：是整份 guideline 的執行後端，讓所有 visual rules 對應到正確素材。
4. **Design Execution**：以 SP01、SP02、SP03、SP04、SP05、SP06、SP07 等版本族群與 POS/NEG/RED/VER/AMA 等 appearance code 建立資產 taxonomy。
5. **Principles**：品牌應用前必須取得授權並使用官方 electronic originals。
6. **Frameworks**：asset family -> variation -> appearance -> color mode -> file format。
7. **Patterns**：可變品牌系統尤其需要嚴格檔名規則，否則變體會失控。
8. **Rules**：AI 可檢查準則：使用或散布 São Paulo brand 前需先取得 São Paulo Turismo authorization；檔名應包含 SP brand identifier、symbol variation A-D、appearance POS/NEG/RED/VER/AMA、color CMYK/RGB/1 COLOR 與格式 .ai/.tiff/.jpg/.png；animated/insertion video 應使用指定 mov/mp4/swf originals。
9. **Reusable Knowledge**：資產命名系統是 design infrastructure 的核心，能把視覺規則轉成檔案治理。
10. **Abstraction**：`Operations`、`Design Tokens`、`Knowledge Architecture`。
11. **Visual Evidence**：檔名拆解與完整資產列表是主要證據，功能是資產目錄與審核依據。

## Credits and Legal Notices

1. **Chapter Purpose**：揭示圖片著作權、組織責任與版本資訊，保護公共品牌使用的法律邊界。
2. **Definition**：包含 SPTuris、聯絡方式、圖片使用限制、版本、政府與專案團隊名單。
3. **Relationship**：補足 Presentation 的公共使用宣告與 Application Rules 的授權要求。
4. **Design Execution**：把 credits、legal notices 與官方聯絡資訊集中於結尾。
5. **Principles**：公共品牌開放不等於素材著作權開放。
6. **Frameworks**：asset use -> copyright ownership -> prior authorization -> official contact。
7. **Patterns**：影像豐富的 guideline 必須在尾端處理第三方影像權利。
8. **Rules**：AI 可檢查準則：guidebook images 僅供 illustration purposes；不得未經 Strategic Actions and Communication Department 事先明確授權而部分或全部重製；品牌問題應聯絡 marcasp@spturis.com。
9. **Reusable Knowledge**：Design knowledge base 應保留法律限制，避免將範例圖片誤視為可重用素材。
10. **Abstraction**：`Legal and Trademark`、`Operations`。
11. **Visual Evidence**：以 credit 與法律文字為主，功能是權利與責任證據。
