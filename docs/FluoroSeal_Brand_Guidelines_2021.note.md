# FluoroSeal Brand Manual

來源：`sources/FluoroSeal_Brand_Guidelines_2021.pdf`
版本：MKT 0022 - V11/2021
研究視角：Design Infrastructure Researcher

## Summary

The official FluoroSeal brandbook defining brand strategy, logo usage, colors, typography, icon style, imagery levels, color filters, and stationery applications.。建立 FluoroSeal 的策略語言與組織共識，讓後續視覺規範不只是設計元素，而是品牌承諾的表現。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。integrity、responsibility、legacy、quality、customer commitment、innovation、partnership。品牌框架是「value proposition -> mission -> vision -> values -> essence -> voice -> history」。

執行上，這份 guideline 透過 Brand、Mission, Vision and Values、Brand Identity and Brand Essence、Tone of Voice 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。B2B industrial brand guideline 應同時處理 technical competence 與 relationship promise。

## Brand

1. **Chapter Purpose**：建立 FluoroSeal 的策略語言與組織共識，讓後續視覺規範不只是設計元素，而是品牌承諾的表現。
2. **Definition**：定義 brand promise、mission、vision、values、brand identity、brand essence、tone of voice、company story。
3. **Relationship**：是 Visual Elements、Imagery、Applications 的語義基礎，特別連到「Building Value Together」與 customer partnership。
4. **Design Execution**：作者先說明品牌是 impressions 的總和，再把工業專業、客戶合作、低排放創新與全球歷史放入同一 narrative。
5. **Principles**：integrity、responsibility、legacy、quality、customer commitment、innovation、partnership。
6. **Frameworks**：品牌框架是「value proposition -> mission -> vision -> values -> essence -> voice -> history」。
7. **Patterns**：工業品牌用價值與歷史建立信任，再讓視覺語言承接技術與人性兩種面向。
8. **Rules**：AI 可檢查：品牌表述應包含 partnership/customer value；不得只談產品規格而忽略 customer commitment；核心 essence 應維持 “Building Value Together”。
9. **Reusable Knowledge**：B2B industrial brand guideline 應同時處理 technical competence 與 relationship promise。
10. **Abstraction**：`Brand Strategy`、`Voice and Tone`、`Knowledge Architecture`。
11. **Visual Evidence**：時間軸與價值頁主要承擔歷史與信任證據功能。

### Mission, Vision and Values

1. **Chapter Purpose**：把公司存在理由、未來目標與行為標準明確化。
2. **Definition**：定義 high-quality industrial valves、advanced metallurgical methods、exceptional reliability、low cost of ownership、low-emissions innovation，以及六組 values。
3. **Relationship**：支撐 brand essence 與 imagery 的 technical/high-quality tone。
4. **Design Execution**：以 value proposition、mission、vision 三段遞進，再列出 values 與行為句。
5. **Principles**：customer outcome、reliability、environmental responsibility、continuous improvement。
6. **Frameworks**：策略框架是「what we offer -> why we exist -> where we are going -> how we behave」。
7. **Patterns**：將 values 寫成行為動詞，比單字口號更容易轉入 voice 與影像審核。
8. **Rules**：AI 可檢查：品牌文案需能反映 integrity、responsibility、legacy、quality、customer commitment、innovation；不可宣稱與 industrial valve expertise 無關的品牌定位。
9. **Reusable Knowledge**：mission/values 章應提供後續 design decision 的評估語言。
10. **Abstraction**：`Brand Strategy`、`Governance`。
11. **Visual Evidence**：此節以文字分層為主，功能是策略證據而非視覺示範。

### Brand Identity and Brand Essence

1. **Chapter Purpose**：把多個策略元素壓縮成可被全公司記住與應用的核心訊息。
2. **Definition**：定義 brand identity 是希望 stakeholders 聯想到 FluoroSeal 的 associations；brand essence 是 “Building Value Together”。
3. **Relationship**：連接 values 與 Visual Elements，讓 logo、色彩、影像都有共同語義。
4. **Design Execution**：作者用 partnership、in-house high standard、customers’ customers impacted 等敘事擴大品牌影響範圍。
5. **Principles**：collaboration、value creation、stakeholder impact、shared future。
6. **Frameworks**：essence 框架是「association -> promise -> partnership -> end-user impact -> shared future」。
7. **Patterns**：一句 essence 可作為 guideline 的跨章節 evaluation rubric。
8. **Rules**：AI 可檢查：溝通內容若提到 FluoroSeal 應能連到 building value 或 partnership；不得只呈現 isolated product without customer value。
9. **Reusable Knowledge**：brand essence 要能同時驅動文案、影像與應用版面。
10. **Abstraction**：`Brand Strategy`、`Messaging`。
11. **Visual Evidence**：此節以大字標語與品牌語句作為語義錨點。

### Tone of Voice

1. **Chapter Purpose**：將「我們的人就是品牌」轉成溝通行為，確保工業專業不變成傲慢或冷硬。
2. **Definition**：定義 personal、knowledgeable、respectful、professional、open。
3. **Relationship**：延續 values，影響文案、客戶互動與人像影像。
4. **Design Execution**：作者用短段說明每個 voice attribute 的行為邊界，例如專家但不傲慢、誠實且容易接近。
5. **Principles**：expert humility、personal touch、accountability、straightforwardness。
6. **Frameworks**：voice 框架是「customer-tailored -> expert insight -> respect -> professionalism -> openness」。
7. **Patterns**：B2B voice 可以用「能力 + 謙遜」平衡可信度與親近性。
8. **Rules**：AI 可檢查：文案應避免 arrogant tone；應有 personal touch；若不知道答案，應表達知道如何取得答案；應 straight-forward、honest、authentic。
9. **Reusable Knowledge**：voice guideline 應把品牌人格轉成可觀察的服務行為。
10. **Abstraction**：`Voice and Tone`、`Content Design`。
11. **Visual Evidence**：此節視覺簡潔，證據主要是分欄 voice attributes。

### FluoroSeal Company Story

1. **Chapter Purpose**：用時間軸證明品牌的 legacy、全球能力與持續投資，不讓創新承諾停留在口號。
2. **Definition**：定義從 1983 founding 到工廠、辦公室、收購、新產品線與 R&D investment 的里程碑。
3. **Relationship**：支撐 values 中的 legacy 與 innovation，也為 imagery 的全球 workforce 提供背景。
4. **Design Execution**：以年份節點呈現公司擴張，讓品牌信任來自歷史證據。
5. **Principles**：heritage as proof、global growth、continuous reinvestment。
6. **Frameworks**：故事框架是「founding -> ownership/facility shifts -> global expansion -> acquisitions -> today’s investment」。
7. **Patterns**：B2B guideline 可用 timeline 把抽象 brand credibility 實證化。
8. **Rules**：AI 可檢查：品牌歷史敘述應維持正確時間順序；不得將未載明的 milestone 編入官方故事。
9. **Reusable Knowledge**：company story 可作為品牌信任與人才/客戶溝通素材。
10. **Abstraction**：`Brand Strategy`、`Messaging`。
11. **Visual Evidence**：時間軸頁承擔歷史順序與成長證據功能。

## Visual Elements

1. **Chapter Purpose**：把品牌策略轉成可複製的視覺工具箱，讓所有市場都能一致地建立 FluoroSeal 形象。
2. **Definition**：定義 logo、blue/gray/black colors、Lato typography、icons、imagery style、key visual components。
3. **Relationship**：將 Brand 的專業與合作語義落到 logo、色彩、字體與圖示。
4. **Design Execution**：作者將每個視覺元素拆成版本、限制、數值與應用原則。
5. **Principles**：technical clarity、blue/white dominance、legibility、consistency、controlled accents。
6. **Frameworks**：視覺元素框架是「logo -> color -> typography -> icon -> imagery bridge」。
7. **Patterns**：先定核心元素，再定應用與例外，是品牌手冊常見可複用架構。
8. **Rules**：AI 可檢查：所有 key visual elements 應使用 primary colors 或 white；secondary colors 只能作補充且一次一色；不得引入未核准字體。
9. **Reusable Knowledge**：視覺系統應用 token、variant、usage principle 三層建立。
10. **Abstraction**：`Visual Identity`、`Design Tokens`、`Visual Language`。
11. **Visual Evidence**：logo、色票、字樣、icon 範例共同承擔系統校準證據。

### Logo System

1. **Chapter Purpose**：保護 FluoroSeal 最重要的視覺元素，並將其 fluidity/adaptability 的象徵語義固定在正確用法中。
2. **Definition**：定義 master/negative/mono logo、inline versions、icon version、misuse、placement、clear space、minimum size。
3. **Relationship**：從 brand essence 延伸出 visual embodiment，並支援 stationery、digital、print。
4. **Design Execution**：以使用情境選 logo 版本，並用禁止清單防止字體、顏色、比例與角度被破壞。
5. **Principles**：recognition integrity、background contrast、approval for rare icon use、minimum legibility。
6. **Frameworks**：logo 框架是「background -> variant -> placement -> clear space -> minimum width -> misuse check」。
7. **Patterns**：把 icon version 限定為少數情境且需批准，是控制品牌縮寫的治理模式。
8. **Rules**：AI 可檢查：master blue logo 用於淺色背景；negative logo 用於 corporate blue 背景；mono logo 限於不可彩色印刷；preferred placement 為 top center，否則 bottom left/right；logo 最小寬度應大於 25 mm / 95 px；不得旋轉、拉伸、改色、加漸層、描邊、改字體或使用 busy background。
9. **Reusable Knowledge**：logo guideline 應將例外情境、批准角色與禁止動作寫清楚。
10. **Abstraction**：`Visual Identity`、`Governance`、`AI-Checkable Rules`。
11. **Visual Evidence**：logo variations、misuse grid、clear space/minimum size 圖承擔示範與警示證據。

### Color System

1. **Chapter Purpose**：建立以藍白為主的工業專業感，同時保留少量輔助色增加生命力與區分。
2. **Definition**：定義 Corporate Blue #00558C、Marine Blue #5E8AB4、Sky Blue #69B3E7、Purple #535486、Plum #642F6C、Slate #888B8D、white。
3. **Relationship**：支援 logo、typography、slogan、icons、backgrounds 與 applications。
4. **Design Execution**：作者提供 CMYK、Pantone、RGB、HEX，並明確限制 secondary colors 的混用。
5. **Principles**：blue/white dominance、accent restraint、background clarity、balance through white.
6. **Frameworks**：色彩框架是「primary background -> key visual elements -> secondary accent -> white separation」。
7. **Patterns**：一次只用一個 additional color 是可移植的色彩治理規則。
8. **Rules**：AI 可檢查：key visual elements 只能用 primary colors 與 white；secondary colors 不應混用；使用 primary/secondary colors 時應以 white 分隔；Corporate Blue 應使用 #00558C。
9. **Reusable Knowledge**：色彩系統應不只列 token，也要定義組合規則。
10. **Abstraction**：`Color System`、`Design Tokens`、`AI-Checkable Rules`。
11. **Visual Evidence**：色票、tints 與 usage principle 頁承擔 token 與組合證據。

### Typography and Icon Style

1. **Chapter Purpose**：建立文字層級與簡潔圖示語言，讓技術品牌同時專業、可讀與友好。
2. **Definition**：定義 Lato、Open Sans、Arial fallback、print/web typography roles、single-color glyph icons。
3. **Relationship**：Typography 支援品牌語氣，Icon Style 支援資訊快速理解。
4. **Design Execution**：作者將 Lato 用於 headlines/subtitles、intro text，Open Sans 用於 body/informative elements，Arial 作為系統 fallback。
5. **Principles**：legibility、technical precision、humanism、role-based type hierarchy、icon consistency。
6. **Frameworks**：字體框架是「preferred fonts -> fallback -> role mapping -> print/web variants -> size examples」；icon 框架是「single-color glyph -> consistent size/style -> palette color」。
7. **Patterns**：以角色分配字體而非僅列字體，可轉成審核規則。
8. **Rules**：AI 可檢查：headlines/subtitles 用 Lato Bold；intro paragraphs 用 Lato Bold Italic；body text 用 Open Sans；小文字/符號/captions 用 Open Sans；若 Lato/Open Sans 不可用，使用 Arial；每個 icon 或 icon group 僅使用一種 palette color。
9. **Reusable Knowledge**：B2B guideline 應將可讀性與技術氣質同時納入 typography decisions。
10. **Abstraction**：`Typography System`、`Iconography System`、`AI-Checkable Rules`。
11. **Visual Evidence**：font specimens、size usage examples、print/web sample layouts 與 icon examples 承擔實作校準證據。

## Imagery

1. **Chapter Purpose**：用三層影像系統把產品品質、工業應用與人員合作區分開，讓 imagery 對應不同溝通任務。
2. **Definition**：定義 Level I Product Showcase、Level II Reference Images、Level III People、color filters。
3. **Relationship**：承接 Brand 中的 product quality、global workforce、customer partnership，也使用 Color System 的藍紫色調。
4. **Design Execution**：作者將影像分成產品白底、工業環境、真實人物互動，再提供 color filter 製作方法。
5. **Principles**：modern clarity、technical high quality、authentic human interaction、depth of field、cool chromacity。
6. **Frameworks**：影像框架是「communication need -> image level -> subject/focus -> background/context -> usage examples -> optional filter」。
7. **Patterns**：三層 imagery taxonomy 是可複用的影像治理架構，避免所有照片承擔同一目的。
8. **Rules**：AI 可檢查：Level I 產品圖應高解析、白底、清楚細節、移除非產品背景；Level II 應呈現 location/plant/equipment，branding discrete 且盡量無人；Level III 應呈現真實工作場景，不可 staged；color filter 應先 grayscale，再 multiply adjustment layer 並使用核准色碼。
9. **Reusable Knowledge**：影像系統可用「產品、環境、人」三層對應 B2B 購買旅程。
10. **Abstraction**：`Photography System`、`Visual Language`、`AI-Checkable Rules`。
11. **Visual Evidence**：各 level 範例與 color overlay 對照承擔分類、示範與製作證據。

## Applications

1. **Chapter Purpose**：將品牌元素落到文具等高頻正式觸點，讓印刷與數位簽名都能一致。
2. **Definition**：定義 stationery showcase、business cards、letterheads、email signature、envelopes，以及尺寸、紙張、bleed、margin、window placement 等參數。
3. **Relationship**：把 Logo、Color、Typography 轉成生產規格。
4. **Design Execution**：作者用 imperial/metric 雙制與紙張/印刷參數降低跨地區供應商誤差。
5. **Principles**：production precision、formal consistency、supplier clarity、metric/imperial interoperability。
6. **Frameworks**：應用框架是「item -> dimensions -> paper/printing -> margins/bleed/window -> visual example」。
7. **Patterns**：將 brand applications 寫成 production spec，是讓 guideline 從設計走向營運的必要模式。
8. **Rules**：AI 可檢查：business card 尺寸應為 3.5 x 2 in / 88.9 x 50.8 mm，含 bleed 為 3.75 x 2.25 in；letterhead US letter 應為 8.5 x 11 in，A4 為 210 x 297 mm；email signature 應為 613 px x 150 px；#10 envelope 應為 4 1/8 x 9 1/2 in。
9. **Reusable Knowledge**：品牌應用 guideline 需同時支援設計審核與採購製作。
10. **Abstraction**：`Operations`、`Print Production`、`Brand Applications`。
11. **Visual Evidence**：stationery mockups 與參數表承擔生產與位置證據。

## Contact

1. **Chapter Purpose**：提供 guideline 例外、疑問與品牌治理的正式入口。
2. **Definition**：定義 FluoroSeal Group Marketing and Communications 與聯絡資訊。
3. **Relationship**：收束所有可能需要品牌團隊判斷的使用情境，特別是 icon logo approval 與 brand questions。
4. **Design Execution**：以簡短 contact page 作為支援性結尾。
5. **Principles**：central ownership、supportive governance、traceable escalation。
6. **Frameworks**：治理框架是「疑問/例外 -> Marketing and Communications -> 批准或指引」。
7. **Patterns**：完整 brand manual 仍需要 human owner 來處理未覆蓋情境。
8. **Rules**：AI 可檢查：任何未被明確允許的 logo/icon/brand use 應導向 marketing@fluorosealgroup.com 或相關品牌團隊。
9. **Reusable Knowledge**：brand guideline 應以清楚 owner 降低自由詮釋。
10. **Abstraction**：`Governance`、`Operations`。
11. **Visual Evidence**：此章視覺證據少，主要功能是責任定位。
