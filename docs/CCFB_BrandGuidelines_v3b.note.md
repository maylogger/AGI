# Central California Food Bank Brand Identity Guidelines

來源：`sources/CCFB_BrandGuidelines_v3b.pdf`
版本：3B
日期：2018-02-23
研究視角：Design Infrastructure Researcher

## Summary

Central California Food Bank 的品牌識別指南，涵蓋品牌地圖、logo 組成、色彩、檔案格式、留白、字體、禁用規則與 stationery package。在進入 logo 技術前先定義組織如何被理解，避免 food bank 的視覺符號只被看成插圖。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。非營利品牌需要同時表達 compassion 與 operational credibility。`definition -> difference -> relevance -> competencies -> reputation -> personality`。

執行上，這份 guideline 透過 Brand Map、Logo System、Elements of the Logo、Color Palettes and Logo Variations 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。小型品牌可用 one-page brand map 作為所有視覺規範的語義 anchor。

## Brand Map

1. **Chapter Purpose**：在進入 logo 技術前先定義組織如何被理解，避免 food bank 的視覺符號只被看成插圖。
2. **Definition**：本章定義 CCFB 的 mission、difference、relevance、core competencies、reputation 與 personality。
3. **Relationship**：它是後續 logo bug 符號與色彩語彙的語義來源，讓 orange、wheel、coin slot 等圖形有組織意義。
4. **Design Execution**：用條列品牌 building blocks，將「gather and distribute food」「partnered agencies」「professional integrity」等抽象定位轉成設計可引用的語彙。
5. **Principles**：非營利品牌需要同時表達 compassion 與 operational credibility。
6. **Frameworks**：`definition -> difference -> relevance -> competencies -> reputation -> personality`。
7. **Patterns**：品牌地圖把策略詞彙切成可供 logo、copy、色彩與應用設計引用的 metadata。
8. **Rules**：AI-checkable criteria：品牌材料應能至少對應到 caring/compassionate、trustworthy、resourceful、professional、strategic 中一項；不得只呈現需求悲情而缺少可靠分配能力。
9. **Reusable Knowledge**：小型品牌可用 one-page brand map 作為所有視覺規範的語義 anchor。
10. **Abstraction**：`Brand Strategy`、`Messaging`。
11. **Visual Evidence**：此章以文字表格為主，視覺證據功能是品牌屬性分類而非版面示範。

## Logo System

1. **Chapter Purpose**：把 CCFB 的 logo 從單一圖形轉成可被生產、授權與檢查的系統。
2. **Definition**：本章定義 logo components、symbol meanings、palette、variations、file formats、color formats、required whitespace 與 format variations。
3. **Relationship**：承接 Brand Map 的公益與分發任務，並銜接 Typography、Incorrect Usage 與 Stationery。
4. **Design Execution**：以每頁一個規格面向的方式，先講 logo 意義，再講色彩、檔案、空間與格式。
5. **Principles**：logo governance 應同時管理 meaning、appearance、production 與 misuse。
6. **Frameworks**：`symbol -> palette -> variation -> file format -> whitespace -> format variation`。
7. **Patterns**：先定義 preferred lock-up，再列 limited-use standalone bug 與 horizontal variation，是常見的 logo family pattern。
8. **Rules**：AI-checkable criteria：preferred logo 應含 Feeding America lock-up；Feeding America logo 在 color/grayscale 時水平至少 7/16 inch，solid black 至少 3/8 inch，web/on-screen 至少 55 px；logo 周圍至少保留 1.5 倍 x-height 空間。
9. **Reusable Knowledge**：logo 系統應把 partner lock-up 的尺寸條件納入規則，而不只是提供圖片。
10. **Abstraction**：`Visual Identity`、`AI-Checkable Rules`、`Operations`。
11. **Visual Evidence**：logo component diagram、palette swatches、variation specimens 與 whitespace diagram 承擔示範、比例與規定證據。

### Elements of the Logo

1. **Chapter Purpose**：讓 logo 的每個符號都能回應組織任務，避免視覺圖案被任意替換。
2. **Definition**：定義 logo bug 與 logo type，並解釋 orange fruit、double C、distribution wheel、five lugnuts、coin slot/red line 的含義。
3. **Relationship**：把 Brand Map 中的 food distribution、regional service、bank concept 轉成圖形語法。
4. **Design Execution**：用 callout 直接標示 logo 元件與象徵功能。
5. **Principles**：非營利 logo 可透過多義符號同時承載地域、功能、服務範圍與組織名稱。
6. **Frameworks**：`graphic element -> operational meaning -> brand relevance`。
7. **Patterns**：multi-symbol logo explanation 可用於組織需要教育使用者理解新標誌時。
8. **Rules**：AI-checkable criteria：不得改變 orange wheel 與 text 的關係；logo bug 單獨使用需 limited basis；任何替代圖形若無法保留 food、Central California、distribution、five counties、bank 概念，不應視為同等 logo。
9. **Reusable Knowledge**：logo explanation 應把符號與品牌 proof points 建立映射。
10. **Abstraction**：`Visual Identity`、`Brand Strategy`。
11. **Visual Evidence**：logo callout 是關鍵視覺證據，功能是定義與語義解碼。

### Color Palettes and Logo Variations

1. **Chapter Purpose**：統一 CCFB 與 Feeding America 的色彩關係，確保本地品牌與母／夥伴系統一致。
2. **Definition**：定義 CCFB colors 與 Feeding America PMS/RGB/CMYK/hex，並列出 full color、2-color、bug、horizontal、with lock-up variations。
3. **Relationship**：接在 logo meaning 之後，將符號轉成可印刷與數位使用的色彩資產。
4. **Design Execution**：以 swatch 與 file name 配對，讓設計與 production 可以同時查找。
5. **Principles**：色彩不只是品牌感，還是 partner affiliation 與生產模式的約束。
6. **Frameworks**：`brand palette -> partner palette -> logo variation -> file name`。
7. **Patterns**：把 preferred variation 明確標示，避免所有版本被視為同等優先。
8. **Rules**：AI-checkable criteria：full-color preferred logo 使用 `CCFBlogo_wFAlock_cmy`；2-color versions 使用 Feeding America PMS colors；horizontal variation 僅在必須符合水平空間時使用。
9. **Reusable Knowledge**：variation 規則需含 priority、context 與 file-name mapping。
10. **Abstraction**：`Color System`、`Visual Identity`。
11. **Visual Evidence**：palette swatches 與 logo variation specimens 承擔比較與選擇證據。

### File Formats and Color Formats

1. **Chapter Purpose**：避免供應商用錯檔案造成低解析、不可放大或色彩失真。
2. **Definition**：定義 commercial printing、screen、PDF logo、CMYK、Pantone、RGB、reverse 的使用情境。
3. **Relationship**：把視覺規格銜接到實際 production workflow。
4. **Design Execution**：依媒體分類說明 .AI、.EPS、.JPG、.PNG、.SVG、PDF 與各色彩模式。
5. **Principles**：設計指南必須替非設計者說清楚「什麼檔案給誰用」。
6. **Frameworks**：`medium -> file type -> scalability/resolution -> color mode`。
7. **Patterns**：file format decision table 是品牌資產治理的基本模組。
8. **Rules**：AI-checkable criteria：commercial printing 優先使用 AI/EPS/JPG；JPEG 只能縮小不可放大；screen 使用 PNG/SVG；spot color 適用於 screen printing 或 embroidery；dark background 可使用 white/reversed logo。
9. **Reusable Knowledge**：把 file format 與 use case 綁定，可降低 production error。
10. **Abstraction**：`Operations`、`Design Tokens`。
11. **Visual Evidence**：本節文字為主，視覺證據是檔案格式分類與色彩模式說明。

### Required Whitespace and Format Variations

1. **Chapter Purpose**：保護 logo legibility 與品牌完整性，並提供低色彩限制場景的合法替代版本。
2. **Definition**：定義 x-height、minimum open/white space、solid black、two-color spot、reverse、grayscale。
3. **Relationship**：為 Incorrect Usage 提供可量化的判準。
4. **Design Execution**：以 pink guide lines 示範 1.5X 留白，同時警告 pink lines 不應被印出。
5. **Principles**：空間是 logo 的一部分；替代版本應由 reproduction constraint 觸發。
6. **Frameworks**：`logo type x-height -> 1.5X clear space -> approved reproduction variants`。
7. **Patterns**：以相對單位而非固定尺寸定義 clear space，能適用於不同 logo 尺寸。
8. **Rules**：AI-checkable criteria：所有 logo variations 含 Feeding America lock-up 與 bug-only 版本周圍必須有 1.5X 空間；pink measuring lines 不得出現在最終輸出；reverse logo 只能放在足夠暗且主要為 solid area 的背景。
9. **Reusable Knowledge**：clear space 應用相對測量 token，並明確標示示意線不是輸出物。
10. **Abstraction**：`Layout System`、`AI-Checkable Rules`。
11. **Visual Evidence**：whitespace diagram 與 variation examples 承擔比例與合法版本證明。

## Typography and Incorrect Usage

1. **Chapter Purpose**：保護 logo 不被當作一般字體重組，並建立搭配字體的品牌語氣。
2. **Definition**：定義 logo type 是 altered/kerned Brandon Text Black，不建議作為 accompanying type style；搭配字體為 Gotham、Georgia，fallback 為 Arial；同時列出 logo 禁用行為。
3. **Relationship**：承接 logo system，補上 typography 與 misuse governance。
4. **Design Execution**：用 specimen 展示可用字體，用 all-caps warning 與錯誤案例列出禁止項目。
5. **Principles**：wordmark 與品牌字體需分離治理；logo 是 artwork，不是可重打文字。
6. **Frameworks**：`wordmark type origin -> accompanying fonts -> fallback fonts -> misuse prohibitions`。
7. **Patterns**：小型品牌常用 commercially available fonts 搭配 customized wordmark，此時必須說清不可混用。
8. **Rules**：AI-checkable criteria：不得 reposition logo elements；不得 skew/distort；不得改變 orange wheel 與 text 關係；不得放在 busy/distracting background；不得改 logo element colors；不得在 minimum whitespace 內放文字；Gotham 不可用時可用 Arial。
9. **Reusable Knowledge**：incorrect usage 應轉為 visual lint rules，而不只是設計師提醒。
10. **Abstraction**：`Typography System`、`AI-Checkable Rules`、`Visual Identity`。
11. **Visual Evidence**：字體 specimen 與 incorrect usage examples 承擔示範與警示功能。

## Stationery Package

1. **Chapter Purpose**：把品牌系統落地到最常見的正式溝通物，確保名片、信封、信紙與卡片可由供應商準確製作。
2. **Definition**：定義 stationery package items、InDesign package folders、required fonts、linked images、for-position-only data 與 non-printing specification layers。
3. **Relationship**：將前述 logo、color、typography、spacing 規則實作在 business communication templates。
4. **Design Execution**：列出 horizontal/vertical business card、#10R envelope、A6 envelope、letterhead、notecards 等 artwork 與 callout screenshots。
5. **Principles**：guideline 不只提供規則，也要提供可生產模板與驗證資訊。
6. **Frameworks**：`stationery item -> packaged source file -> placeholder data -> non-printing specs -> production verification`。
7. **Patterns**：以 InDesign package 管理字體與 linked images，是 print production governance pattern。
8. **Rules**：AI-checkable criteria：production 前需替換並驗證 names、titles、phone numbers、addresses、url；non-printing layer 不得輸出到成品；letterhead sample letter 使用 10pt/15pt Georgia Regular。
9. **Reusable Knowledge**：品牌 stationery module 應含模板、資產依賴、placeholder data policy 與輸出檢查。
10. **Abstraction**：`Operations`、`Layout System`、`Governance`。
11. **Visual Evidence**：stationery screenshots 與 callouts 是主要證據，功能是版面示範與製作規格。
