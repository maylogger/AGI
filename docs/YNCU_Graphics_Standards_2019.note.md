# YNCU + Community First Graphics Standards

來源：`sources/YNCU_Graphics_Standards_2019.pdf`
版本：YNCU + CF Logo Guide 2018
日期：2018
研究視角：Design Infrastructure Researcher

## Summary

這份 graphics standards 為 YNCU 與 Community First 的雙品牌識別建立 logo、色彩、最小尺寸、隔離空間、字體、招牌、活動與社群素材規則。建立 graphic standards 的必要性，讓內部員工、夥伴與供應商知道品牌一致性不是美感偏好，而是信用合作社整體形象與信任的防線。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。一致性來自共同語言；若使用者不知道元件名稱，就無法正確遵守比例、隔離與版本規則。Knowledge onboarding framework：why standards matter → component terminology → glossary → usage rules。

執行上，這份 guideline 透過 Introduction、About Graphics Standards、Terminology and Glossary、YNCU Logo 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。在視覺規範前先建立術語表，可降低跨部門誤用。

## Introduction

1. **Chapter Purpose**：建立 graphic standards 的必要性，讓內部員工、夥伴與供應商知道品牌一致性不是美感偏好，而是信用合作社整體形象與信任的防線。
2. **Definition**：定義手冊範圍是 YNCU 與 Community First 的基本視覺識別；不完整涵蓋 writing tone、social content 或 iconography 等延伸指南。
3. **Relationship**：此章提供術語與治理理由，後續 logo、色彩、字體與應用章節都依此共同語言運作。
4. **Design Execution**：作者先說明 standards 的歷史與品牌保護功能，再建立 corporate symbol、emblem、logotype 等元件名稱。
5. **Principles**：一致性來自共同語言；若使用者不知道元件名稱，就無法正確遵守比例、隔離與版本規則。
6. **Frameworks**：Knowledge onboarding framework：why standards matter → component terminology → glossary → usage rules。
7. **Patterns**：小型品牌 guideline 可用簡短 introduction + glossary 快速校準非設計使用者。
8. **Rules**：AI 可檢查：logo files 不得被任何方式改動；任何 stationery、business card、publication、newsletter、ad、website、flyer 或 sign 都應被視為品牌應用物。
9. **Reusable Knowledge**：在視覺規範前先建立術語表，可降低跨部門誤用。
10. **Abstraction**：Knowledge Architecture、Governance、Visual Identity。
11. **Visual Evidence**：使用 logo 元件標註圖與術語排版，功能是定義元件邊界與命名。

### Terminology and Glossary

1. **Chapter Purpose**：讓後續規範可以精準引用 corporate symbol、emblem、logotype、clear space、PMS、RGB 等語彙。
2. **Definition**：定義設計、印刷與版面相關名詞；不提供深度印刷教學。
3. **Relationship**：它是所有 AI-checkable criteria 的語義基礎，例如「clear space」與「reverse」必須先被定義。
4. **Design Execution**：以短定義取代長篇說明，使非設計角色能快速對齊。
5. **Principles**：可執行規範必須有穩定 vocabulary。
6. **Frameworks**：Term taxonomy：brand concepts、logo components、color reproduction、layout geometry、paper/print conditions。
7. **Patterns**：術語表可獨立成通用 guideline module。
8. **Rules**：AI 可檢查：若文件使用 PMS/CMYK/RGB/reverse/clear space 等詞，應與手冊定義一致；logo 應被視為 emblem + logotype 的固定組合。
9. **Reusable Knowledge**：所有品牌知識庫都應將專有名詞與可操作定義分離保存。
10. **Abstraction**：Knowledge Architecture、Design Tokens。
11. **Visual Evidence**：主要為文字與元件標註，功能是語義校準。

## YNCU Logo

1. **Chapter Purpose**：保護 YNCU 主品牌識別在不同印刷與數位環境下的可讀性、比例與完整性。
2. **Definition**：定義 YNCU logo 的 primary colours、secondary colours、minimum size、scaling 與 area of isolation；不授權自行建立變體。
3. **Relationship**：此章是 Community First 章節的母型，兩者共享同一套 logo 使用邏輯。
4. **Design Execution**：用版本分類、英寸最小尺寸與 clear space 圖，把 logo 從檔案變成可驗證規格。
5. **Principles**：logo 必須作為整體縮放；可讀性與辨識度優先於版面填滿。
6. **Frameworks**：Logo usage framework：version selection → minimum size → proportional scaling → isolation area。
7. **Patterns**：primary/full/emblem 分拆；background condition 對應 logo 版本；用 icon 內部幾何作為 clear space 單位。
8. **Rules**：AI 可檢查：standard logo 寬度不得低於 1.125 inch；full version 寬度不得低於 2.625 inch；emblem 寬度不得低於 0.3 inch；logo 與 emblem 必須等比縮放，不得改變個別元件比例、尺寸或位置。
9. **Reusable Knowledge**：小型品牌可用最小尺寸與隔離空間兩個 token 管理大部分 logo 誤用。
10. **Abstraction**：Visual Identity、Logo System、AI-Checkable Rules。
11. **Visual Evidence**：logo 版本、尺寸圖與 isolation diagram 是主要證據，功能是比例、限制與版本比較。

### Primary and Secondary Colours

1. **Chapter Purpose**：讓使用者在彩色、黑白、反白、品牌色底與灰底情境中選擇正確 logo。
2. **Definition**：定義 primary colour version 用於可支援 CMYK/PMS 或 screen 的情境；secondary versions 用於不可全彩或需反白的情境。
3. **Relationship**：連到 brand specifications 的色彩值與 contrast guide。
4. **Design Execution**：用同一 logo 在不同背景上的版本展示選擇邏輯。
5. **Principles**：版本選擇應由 reproducción/背景條件決定，而非使用者偏好。
6. **Frameworks**：Version decision matrix：full color、black/white、reversed、on green、on grey。
7. **Patterns**：背景專用 logo asset 能避免使用者自行反白或改色。
8. **Rules**：AI 可檢查：在綠色品牌色底應使用專用 green background 版本；灰底版本不得泛用於所有深色背景；全彩可行時優先使用 primary colour version。
9. **Reusable Knowledge**：不同背景的官方檔案應明確命名並列入 directory。
10. **Abstraction**：Logo System、Color System。
11. **Visual Evidence**：各版本 logo 圖是直接視覺證據，承擔選擇比較功能。

## Community First Logo

1. **Chapter Purpose**：在雙品牌系統中讓 Community First 保有識別一致性，又與 YNCU 使用共同規則語法。
2. **Definition**：定義 Community First standard、division tag、secondary colours、minimum size、scaling 與 isolation。
3. **Relationship**：平行於 YNCU Logo 章節，讓兩個品牌共享治理機制但保留不同尺寸門檻。
4. **Design Execution**：以相同頁面結構重複 primary/secondary/minimum/isolation，降低使用者切換品牌時的認知成本。
5. **Principles**：雙品牌手冊應保持規則一致、數值差異明確。
6. **Frameworks**：Parallel brand module：same rule categories, brand-specific tokens。
7. **Patterns**：把不同品牌的 logo rules 做成同構章節，是多品牌指南可複用模式。
8. **Rules**：AI 可檢查：Community First standard logo 寬度不得低於 1.6875 inch；emblem 不得低於 0.3 inch；logo 必須作為單一單位等比縮放；周圍不得有 artwork、type 或 panel edges 侵入 clear space。
9. **Reusable Knowledge**：多品牌 guideline 應避免每個品牌重寫不同邏輯，而是共享規則模板、替換 tokens。
10. **Abstraction**：Brand Architecture、Logo System、AI-Checkable Rules。
11. **Visual Evidence**：版本、尺寸與隔離圖提供平行品牌的比較證據。

### Area of Isolation

1. **Chapter Purpose**：防止 logo 被文字、圖形或版面邊界壓迫，維持可讀性。
2. **Definition**：定義 minimum bounding area，並以 house half-circle mouth 的寬度作為距離基準。
3. **Relationship**：與最小尺寸共同構成 logo 可讀性的兩個主要防線。
4. **Design Execution**：用 diagram 把抽象 white space 轉成從 logo 內部取出的測量單位。
5. **Principles**：clear space 應隨 logo 等比縮放，而非固定絕對值。
6. **Frameworks**：Geometric clear-space framework：extract unit from logo → apply around all sides → prohibit intrusion。
7. **Patterns**：用 logo 自身幾何作為清除區單位，跨尺寸穩定。
8. **Rules**：AI 可檢查：任何文字、圖像、panel edge 不得落入指定 bounding area；clearance distance 應與 logo 尺寸同步縮放。
9. **Reusable Knowledge**：clear space token 最好取自品牌符號，使規則具可視化記憶點。
10. **Abstraction**：Layout System、Logo System。
11. **Visual Evidence**：clear space diagram 是核心證據，功能是比例與防干擾。

## Brand Specifications

1. **Chapter Purpose**：把 logo 規範擴展成可落地的品牌系統，包含顏色、對比、字體、招牌、CityScape、活動、社群與海報。
2. **Definition**：定義品牌色值、背景對比門檻、Montserrat 字體階層、分行招牌特例、CityScape graphic 與各媒介應用元素。
3. **Relationship**：承接 logo 章節，進一步規範使用 logo 時的環境條件與輔助視覺語言。
4. **Design Execution**：作者以數值、背景深淺百分比、字體用途與應用頁面說明，建立執行層規範。
5. **Principles**：小型品牌也需要從靜態 logo 走向應用系統；可讀性、對比與固定輔助圖形是跨媒介一致性的核心。
6. **Frameworks**：Application system：color tokens → contrast rules → typography roles → environmental graphics → campaign templates。
7. **Patterns**：contrast-based logo switching、headline/heading/body font hierarchy、cityscape as reusable graphic device。
8. **Rules**：AI 可檢查：green 應為 RGB 40/182/122、HEX #28B67A、CMYK 75/0/71/0；dark grey 應為 RGB 67/67/69、HEX #434345、CMYK 68/61/57/43；10% black 或更淺背景使用 full colour；30% black 背景使用 100% black；50% black 或更深背景使用 reverse；textured background 需人工確認 legibility。
9. **Reusable Knowledge**：用背景 luminance/黑度建立 logo 版本規則，比單純列出「深色/淺色」更可檢查。
10. **Abstraction**：Color System、Typography System、Environmental Graphics、AI-Checkable Rules。
11. **Visual Evidence**：色票、對比範例、字體樣張、招牌照片/示意、社群與海報 mockup 共同構成應用證據。

### Signage and CityScape Graphic

1. **Chapter Purpose**：治理實體分行空間與地方感視覺元素，避免招牌與 cityscape 被任意複製或改造。
2. **Definition**：定義 stacked/justified signage treatments 只供 signage 使用；CityScape 可出現在框架任一側、可搭配或不搭配 tagline。
3. **Relationship**：它把品牌識別延伸到環境與地方語境，是 logo system 的實體媒介擴展。
4. **Design Execution**：用分行條件與可接受例子說明如何在不同空間中保持一致。
5. **Principles**：環境識別常需要為空間條件創造特例，但特例不得被泛用。
6. **Frameworks**：Environmental application framework：branch space → allowed treatment → placement/lighting → non-recreation boundary。
7. **Patterns**：signage-only variant、location-based graphic motif、tagline optionality。
8. **Rules**：AI 可檢查：stacked 與 justified logo treatments 僅可用於 signage；Community First branch unique applications 不得在其他處重新建立；CityScape line 或文字不得改動；URL 可與 line graphic 一起使用。
9. **Reusable Knowledge**：實體空間特例應明確標註「只可用於某媒介」，避免變成未授權 logo variant。
10. **Abstraction**：Environmental Graphics、Governance。
11. **Visual Evidence**：招牌與 CityScape examples 是主要證據，功能是情境示範與例外界定。

### Event, Social and Poster Materials

1. **Chapter Purpose**：將品牌規範帶入社群、活動與分行溝通這些高頻低控管媒介。
2. **Definition**：定義活動物料種類、社群貼文結構、in-branch posters/screenscapes 的固定元素。
3. **Relationship**：它驗證前面色彩、字體與 CityScape 在 campaign context 的可複用性。
4. **Design Execution**：以 profile photo、single word colour、CityScape graphic、poster element list 建立簡單模板。
5. **Principles**：高頻媒介需要結構化模板，否則一致性會被速度消耗。
6. **Frameworks**：Template framework：brand marker + cityscape/tagline + URL + contrast-aware type treatment + photography。
7. **Patterns**：社群文字放在 cityscape 對角、Montserrat ExtraBold、in-branch poster 固定 emblem in top right。
8. **Rules**：AI 可檢查：一般 branded social post 應盡量使用 cityscape opposite corner 的文字布局；文字應使用 Montserrat ExtraBold；poster/screenscape 應包含 CityScape graphic、welcome tagline、URL、top-right emblem 與依背景對比調整的品牌色文字。
9. **Reusable Knowledge**：品牌 guideline 應為高頻媒介提供結構，而非只提供 logo。
10. **Abstraction**：Social Media System、Marketing Templates。
11. **Visual Evidence**：社群與海報 mockup 承擔模板示範功能。

## File Directory

1. **Chapter Purpose**：把規範連到實際可交付檔案，避免使用者自行命名或找錯版本。
2. **Definition**：列出 YNCU、Community First 與 emblem 的 EPS/PNG、full、black-white、white、green background、grey background 版本。
3. **Relationship**：它是整份規範的資產出口，支援前面所有版本選擇規則。
4. **Design Execution**：作者以檔名清單讓 designer/vendor 能直接核對。
5. **Principles**：資產治理需要標準命名；規範與檔案庫不可分離。
6. **Frameworks**：Asset directory framework：brand → mark type → version → format。
7. **Patterns**：檔名包含 brand、logo type、colour/background condition、format。
8. **Rules**：AI 可檢查：輸出物若使用 YNCU 或 CF logo，檔名應對應 directory 中列出的官方資產；不得使用拼錯或未列出的派生檔作為正式品牌檔。
9. **Reusable Knowledge**：每份 logo guideline 都應有 asset manifest，方便自動稽核素材來源。
10. **Abstraction**：Operations、Design Asset Governance。
11. **Visual Evidence**：以檔名表與 logo 圖示作為版本證據，主要功能是資產索引。
