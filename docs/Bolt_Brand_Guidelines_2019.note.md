# Bolt Brand Guidelines

來源：`sources/Bolt_Brand_Guidelines_2019.pdf`
版本：Version 1.0
日期：March 31, 2019
研究視角：Design Infrastructure Researcher

## Summary

Bolt 的品牌規範，將即時、一致、可信的商務定位轉成 messaging、visual identity、brand architecture、usage、color/typography 與應用規則。

最上層的原則是把品牌精神轉成可重複判斷的系統，而不是只提供單次設計範例。文件前段尤其強調定義概覽、人格、品牌表達、產品表達與 taglines；以 Instant、Consistent、Trusted 與人格光譜描述品牌語氣。

執行上，這份 guideline 透過 Messaging、Overview and Personality Tenets、Expressing Brand, Product and Taglines、Visual Identity 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。其最高治理原則是在不同接觸點維持一致性，同時保留足夠彈性，讓品牌能依媒介與情境延展而不失去辨識度。

## Document Architecture

文件採用高度精簡的 startup brand system 結構：先用 messaging 定位品牌承諾，再定義標誌與架構，接著以 usage parameters、color/typography 和 applications 控制日常輸出。

## Messaging

1. **Chapter Purpose**：先定義 Bolt 如何被理解，再讓視覺系統有明確語義來源。若少了本章，logo 與色彩只能表現速度感，無法支撐信任與一致性。
2. **Definition**：本章定義品牌 mission、personality tenets、brand expression、product expression 與 taglines，負責訊息方向而非版面細節。
3. **Relationship**：它是後續 visual identity 與 type hierarchy 的語氣基礎，也與 Marketing 的 How Bolt Speaks 文件形成外部依賴。
4. **Design Execution**：作者用三個高密度關鍵詞 Instant、Consistent、Trusted 與人格光譜，把品牌定位壓縮成簡短創作判準。
5. **Principles**：快速商務品牌必須同時傳達效率與可靠性；過度 playful 或過度權威都會削弱 premium trust。
6. **Frameworks**：personality spectrum 框架透過 friend/classic/rebel/authority、premium/mass appeal 等軸線定位品牌語氣。
7. **Patterns**：可複用模式是用『+ / –』語氣規則列出應追求與應避免的寫法。
8. **Rules**：AI-checkable：文案應偏 precise、direct、premium、ambitious、professional；不得 overtly fun、self-important、careless、fluffy、apologetic 或 average。
9. **Reusable Knowledge**：品牌訊息 guideline 可用少量關鍵詞與排除規則建立 LLM 可評分的語氣模型。
10. **Abstraction**：Messaging、Voice and Tone、AI-Checkable Rules。
11. **Visual Evidence**：本章以簡潔頁面、語氣軸線與加減清單呈現，證據功能是定位與比較。

## Visual Identity

1. **Chapter Purpose**：建立 Bolt 最小可辨識資產，讓品牌能在產品、行銷與外部素材中快速被辨認。
2. **Definition**：它定義 logomark、inverse logomark、primary logo 與 inverse logo，不處理子品牌或聯名結構。
3. **Relationship**：它將 messaging 的 instant/consistent/trusted 轉成核心符號，並為 brand architecture 提供母體。
4. **Design Execution**：作者以正反版本和主標誌系統，確保標誌在明暗背景上維持可讀與一致。
5. **Principles**：早期成長品牌需要簡潔可擴張的標誌資產，避免太多變體造成辨識分散。
6. **Frameworks**：identity kernel 框架：logomark 是最小單位，primary logo 是常態使用單位，inverse 是環境適配。
7. **Patterns**：可複用模式包括 mark/logo 分離、正反版本、主使用優先與變體限制。
8. **Rules**：AI-checkable：使用標誌時需選擇核准 primary 或 inverse 版本；不得自行重組、加效果、改比例或在低對比背景使用。
9. **Reusable Knowledge**：visual identity 章可先建立最小資產集合，再將所有應用連回這個 kernel。
10. **Abstraction**：Visual Identity、Logo System、AI-Checkable Rules。
11. **Visual Evidence**：本章以標誌正反圖與應用背景展示，承擔示範與可讀性證據。

## Brand Architecture

1. **Chapter Purpose**：防止產品、子品牌或活動名稱各自發展標誌，維持 Bolt 品牌系統的可擴張性。
2. **Definition**：本章定義 sub-brand system、horizontal lockup、vertical lockup 與 construction。
3. **Relationship**：它依賴 visual identity 的主標誌，並與 usage parameters 共同決定組合是否規定。
4. **Design Execution**：作者提供水平與垂直 lockup 的 construction，將品牌關係變成可製作結構。
5. **Principles**：品牌架構的核心不是增加標誌，而是用一致層級表達母品牌與延伸名稱的關係。
6. **Frameworks**：lockup framework 以 orientation、spacing、size relationship 與 text pairing 管理子品牌。
7. **Patterns**：可複用模式是以固定 lockup 管理延伸品牌，而非讓每個子品牌生成獨立 logo。
8. **Rules**：AI-checkable：子品牌需使用核准 horizontal 或 vertical lockup；間距與對齊需符合 construction；不得讓子品牌文字大於或視覺上壓過母品牌。
9. **Reusable Knowledge**：brand architecture guideline 應把組織關係轉成視覺層級與 construction rule。
10. **Abstraction**：Brand Architecture、Governance、Layout System。
11. **Visual Evidence**：本章以 lockup construction 圖作為比例與結構證據。

## Usage Parameters

1. **Chapter Purpose**：補上標誌使用的防護欄，避免正確資產在錯誤空間、背景或搭配方式下失效。
2. **Definition**：它定義 clear space、pairing 與 do nots，負責使用條件而非創造新資產。
3. **Relationship**：它治理 visual identity 與 brand architecture 的實際使用。
4. **Design Execution**：作者用最小空間、搭配規則與錯誤示例，把抽象一致性轉成可檢查限制。
5. **Principles**：標誌可讀性和品牌權重需要空間保護；錯誤示例比單純描述更能降低誤用。
6. **Frameworks**：usage control framework 包含 spacing、pairing、background、transformation 與 prohibited treatments。
7. **Patterns**：可複用模式是 clear-space token + do-not gallery。
8. **Rules**：AI-checkable：標誌周圍需留有指定安全距離；不得拉伸、旋轉、加陰影、改色、置於複雜或低對比背景；partner pairing 不得破壞 Bolt 層級。
9. **Reusable Knowledge**：usage parameter 是把視覺品質轉成自動審核條件的最佳章節。
10. **Abstraction**：Governance、AI-Checkable Rules、Logo System。
11. **Visual Evidence**：本章高度依賴錯誤圖與安全距離圖，承擔警示與比例證據功能。

## Color & Typography

1. **Chapter Purpose**：建立可延展的視覺語言，讓 Bolt 在不依賴 logo 的情況下也能維持一致品牌感。
2. **Definition**：它定義 core、neutral、accent palette、typography、font sheet、type hierarchy 與 typographic do nots。
3. **Relationship**：它從 messaging 的 premium/direct 語氣延伸出色彩與文字層級，並支援 applications。
4. **Design Execution**：作者把色彩分成角色型 palette，再用 font sheet 和 hierarchy 控制文字密度與視覺節奏。
5. **Principles**：色彩與字體必須同時支援辨識、速度感、可信度與產品式清晰。
6. **Frameworks**：design token framework：core colors、neutral colors、accent colors、font family、type scale、hierarchy。
7. **Patterns**：可複用模式包括核心色維持識別、中性色支援介面、accent 色控制重點、字級階層管理資訊。
8. **Rules**：AI-checkable：素材需使用核准色票；accent 不得取代 core；標題與內文需符合 type hierarchy；不得過度混色、亂用字重或建立未核准字級。
9. **Reusable Knowledge**：startup brand guideline 可把 color/type 直接視為 design tokens，利於產品與行銷共用。
10. **Abstraction**：Color System、Typography System、Design Tokens、AI-Checkable Rules。
11. **Visual Evidence**：本章以色票、font sheet、階層示例與錯誤示例提供示範與比較證據。

## Applications

1. **Chapter Purpose**：展示品牌系統如何落在可穿戴與贈品等高曝光實體物，避免品牌在 swag 中變成裝飾。
2. **Definition**：它定義 wearables 與 swag 的使用方向，不涵蓋完整廣告或產品 UI。
3. **Relationship**：它依賴前面所有識別與使用參數，是品牌系統的低風險測試場。
4. **Design Execution**：作者用應用 mockup 展示標誌、色彩與排版在實物上的可讀性和品牌感。
5. **Principles**：實體應用需要簡化、耐看與遠距可辨識，不能把所有品牌元素堆上去。
6. **Frameworks**：application framework 包含 substrate、scale、contrast、placement 與 production constraint。
7. **Patterns**：可複用模式是先用核心標誌和色彩建立辨識，再依材質限制調整細節。
8. **Rules**：AI-checkable：實體物標誌需保持可讀；避免低對比印刷；不得新增口號或圖形破壞品牌 hierarchy。
9. **Reusable Knowledge**：application 章應把設計規範與製作限制連接，特別適用於 merchandise governance。
10. **Abstraction**：Applications、Operations、Production Rules。
11. **Visual Evidence**：本章以 wearables 與 swag mockup 作為情境與製作證據。
