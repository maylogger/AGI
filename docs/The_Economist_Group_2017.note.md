# The Economist Group Brand Style Guide

來源：`sources/The_Economist_Group_2017.pdf`
版本：V1.0
日期：October 2017
研究視角：Design Infrastructure Researcher

## Summary

這份 The Economist Group 品牌風格指南定義品牌敘事、字體、logo、色彩、signature element、illustration、iconography、UI components 與 e-newsletter 範例。先定義 The Economist 不是狹義經濟雜誌，而是一套獨立、嚴謹、全球化的新聞與知識立場。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。設計系統的可信度來自清楚的 editorial worldview；品牌不是風格集合，而是判斷世界的方式。heritage-to-system framework：founding belief、editorial stance、global readership、digital expansion、brand elements。

執行上，這份 guideline 透過 Our Brand、Typography、Logos and Positioning、Core Logo Rules 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。品牌指南可用歷史與哲學作為設計基礎設施，讓系統規則不只是視覺偏好。

## Our Brand

1. **Chapter Purpose**：先定義 The Economist 不是狹義經濟雜誌，而是一套獨立、嚴謹、全球化的新聞與知識立場。
2. **Definition**：定義品牌的內容範圍、歷史起源、自由市場與知識交換信念、國際讀者與數位產品版圖。
3. **Relationship**：這一章為後續字體、logo、色彩與 UI 規範提供文化來源：所有視覺決策都應表現 rigour、independence、curiosity。
4. **Design Execution**：用長篇歷史敘事與編輯立場建立品牌權威，而非直接從 logo 開始。
5. **Principles**：設計系統的可信度來自清楚的 editorial worldview；品牌不是風格集合，而是判斷世界的方式。
6. **Frameworks**：heritage-to-system framework：founding belief、editorial stance、global readership、digital expansion、brand elements。
7. **Patterns**：知識型媒體品牌常先建立思想傳統，再進入視覺規則。
8. **Rules**：AI 可檢查：品牌描述應包含 independent editorial stance、clear analysis、global affairs 或 naturally curious audience；不得把 The Economist 簡化為只談 economics。
9. **Reusable Knowledge**：品牌指南可用歷史與哲學作為設計基礎設施，讓系統規則不只是視覺偏好。
10. **Abstraction**：`Brand Strategy`、`Content Design`、`Knowledge Architecture`。
11. **Visual Evidence**：此章以文字敘事為主，視覺證據功能是品牌定位與世界觀定錨。

## Typography

1. **Chapter Purpose**：建立一套兼具傳統新聞質感、數位可讀性與資訊層級的字體系統。
2. **Definition**：Milo Serif 用於 body copy 與 headline，Econ Sans 用於 subheadings、navigation、metadata、captions、calls-to-action、subscription information。
3. **Relationship**：字體系統支撐 logo 之外的可讀性與品牌聲音，並直接延伸到 UI components 與 e-newsletters。
4. **Design Execution**：以字級範圍、leading 120%、kerning metrics、tracking zero、old style figures 與 web-safe fonts 建立精細排版規則。
5. **Principles**：新聞品牌的視覺權威很大程度來自文字節奏與數字處理。
6. **Frameworks**：editorial typography framework：serif for reading/heritage、sans for navigation/action、web-safe fallbacks。
7. **Patterns**：serif/sans 雙軸系統，依內容深度與操作功能分配。
8. **Rules**：AI 可檢查：body copy at or below 11pt 使用 Milo Serif Text；body copy 不得低於 7pt；headlines above 18pt primarily print 使用 Milo Serif Medium；navigation/subscription copy 不得低於 7pt；leading 建議為 font size 120%；kerning set to metrics、tracking set to zero；web-safe fonts 為 Georgia 與 Helvetica。
9. **Reusable Knowledge**：字體 guideline 應明確描述數字樣式與 leading，而不只列字型名稱。
10. **Abstraction**：`Typography System`、`Design Tokens`。
11. **Visual Evidence**：字母表、數字樣式與字級說明提供字形、階層與可讀性證據。

## Logos and Positioning

1. **Chapter Purpose**：在 The Economist Group 多品牌、多產品、多媒介環境中維持 logo 完整性、比例、可讀性與 sub-brand 架構。
2. **Definition**：涵蓋 print/digital logo、keylines、group logo、editorial logos、business logos、EIU logos、social avatars、app icons 與 sizing。
3. **Relationship**：它把品牌敘事轉成最可見的識別治理，並與色彩、UI、newsletter 應用銜接。
4. **Design Execution**：使用 artwork-only、minimum size、clear space、keyline border、contrast check 與不同媒介尺寸建立完整 logo QA。
5. **Principles**：在紅色背景與多尺寸數位環境中，logo 需要額外分隔與尺寸規範以保留清晰度。
6. **Frameworks**：multi-brand logo governance：core mark、keyline variants、group marks、editorial/sub-brand marks、icons、sizing references。
7. **Patterns**：核心 logo 加 keyline variant，處理低對比背景；sub-brand logos 用 primary/secondary sets 管理。
8. **Rules**：AI 可檢查：只能使用 provided vector artwork；不得自製、拉伸、扭曲或改造 logo；print logo 不得低於 6mm；digital logo 不得低於 40px high；print clear space 為 logo height 1/2；digital clear space 為 logo height 1/4；紅底或相近低對比背景須使用 keyline version。
9. **Reusable Knowledge**：logo chapter 應同時處理媒介差異、低對比補救、sub-brand 結構與尺寸範例。
10. **Abstraction**：`Visual Identity`、`Brand Architecture`、`AI-Checkable Rules`。
11. **Visual Evidence**：大量 logo、keyline、avatar、app icon、版位尺寸圖提供示範、比較與比例證據。

### Core Logo Rules

1. **Chapter Purpose**：確保核心 Economist logo 在 print 與 digital 中可讀且不可被任意改造。
2. **Definition**：定義 artwork、clear space、minimum size、Eco Red、Eco Tilting 與 keyline border 尺寸。
3. **Relationship**：它是 group、editorial、sub-brand logo 的基準。
4. **Design Execution**：將 print 與 digital 分開規範，因為 clear space 與 keyline pixel/point 尺寸不同。
5. **Principles**：同一 logo 在不同媒介需要不同量尺，但治理邏輯一致。
6. **Frameworks**：media-specific core mark rules。
7. **Patterns**：print-mm/pt 與 digital-px 的雙軌規格。
8. **Rules**：AI 可檢查：print keyline：6mm 以上 0.5pt、12mm 以上 0.5pt、24mm 以上 0.75pt、48mm 以上 1pt；digital keyline：40px/80px 以上 1px、160px 以上 2px；digital red 為 `#e3120b` / RGB 227 18 11。
9. **Reusable Knowledge**：keyline 應依 logo 尺寸級距定義，而不是單一固定線寬。
10. **Abstraction**：`Visual Identity`、`Design Tokens`。
11. **Visual Evidence**：keyline 頁面提供邊框厚度與低對比補救證據。

### Group, Editorial and Sub-Brand Logos

1. **Chapter Purpose**：管理 The Economist Group 旗下產品與業務識別，避免多品牌架構失去共同語言。
2. **Definition**：包含 group logo、editorial logos、media business logos、EIU logos、social media avatars、home screen app icons。
3. **Relationship**：它將核心 logo 的比例規則延伸到品牌家族。
4. **Design Execution**：以 primary/secondary logos、reversed versions、group signatures 與 app icon 比例建立辨識層級。
5. **Principles**：品牌家族需要一致的識別語法，但不同產品可保留名稱與媒介差異。
6. **Frameworks**：portfolio identity system。
7. **Patterns**：sub-brand logos grouped by editorial、media business、intelligence unit、apps/social icons。
8. **Rules**：AI 可檢查：group logo clear space 與 core logo 一致；group logo print/digital 不得低於 6mm 或 40px；非 core brand business logos 使用 group signatures；social avatars 與 app icons 需符合標示的 X 比例。
9. **Reusable Knowledge**：多品牌系統應把 logo 家族依用途分類，而不是把所有資產平鋪。
10. **Abstraction**：`Brand Architecture`、`Iconography System`。
11. **Visual Evidence**：logo family grids、reversed versions 與 icon 比例圖提供架構與比例證據。

### Logo Sizing

1. **Chapter Purpose**：避免 logo 在不同尺寸版面中過大或過小，破壞 The Economist 的克制與權威感。
2. **Definition**：依 A4、A5、magazine cover wrap、billboards 與 online ads 調整 logo 大小。
3. **Relationship**：它把 logo 的 minimum size 延伸為 layout-sensitive sizing。
4. **Design Execution**：用實際版位標註 10mm、15mm、30mm、200mm、300mm 等參照。
5. **Principles**：logo 大小需要與版面維度相稱，而不只是達到最低可讀性。
6. **Frameworks**：contextual logo scale framework。
7. **Patterns**：小型印刷、雜誌封套、大型戶外各自有尺寸參照。
8. **Rules**：AI 可檢查：print material 中 logo size 應與 overall dimensions sympathetic；online logo size 應對應 banner ad 尺寸；大型 billboard 需使用指定的大尺寸參照。
9. **Reusable Knowledge**：logo 規範應區分 minimum size 與 preferred contextual size。
10. **Abstraction**：`Layout System`、`Visual Identity`。
11. **Visual Evidence**：不同版位圖提供情境比例證據。

## Colour

1. **Chapter Purpose**：維持 Economist Red 的辨識強度，同時用有限輔助色支援 print/digital 介面與內容層級。
2. **Definition**：print palette 包含 Economist Red、Black、Soft Blue、Grey、Light Grey、Steel Blue、Rich Grey、Pale Blue；digital palette 包含 red、black、icons、CTA、background、member yellow、rules、beta bar、metadata、headers 等 token。
3. **Relationship**：色彩支援 logo、accent rule、UI buttons、newsletter 與資訊呈現。
4. **Design Execution**：明確要求 palette simple，Economist Red sparingly 使用以產生 heightened visual impact。
5. **Principles**：高辨識品牌色需要節制使用；太多紅會削弱紅色的信號價值。
6. **Frameworks**：restrained primary color system。
7. **Patterns**：print 使用較少語意色；digital 使用更細分的 UI token。
8. **Rules**：AI 可檢查：digital Economist Red 必須為 `#E3120B`；digital Black 為 `#121212`；CTA active 為 `#3E51B5`；background 為 `#F2F2F2`；metadata 為 `#B6B6B6`；core red 應 sparingly 使用，不應成為大面積裝飾色。
9. **Reusable Knowledge**：色彩系統可分成 print communication palette 與 digital UI semantic tokens。
10. **Abstraction**：`Color System`、`Design Tokens`。
11. **Visual Evidence**：色票與語意標籤提供 token 與使用意圖證據。

## Signature Element

1. **Chapter Purpose**：提供一個小而可辨識的品牌節奏裝置，用於聚焦標題、凸顯功能與改善文字密集頁面的節奏。
2. **Definition**：signature brand tag 是 accent rule；X 等於 section heading cap height，stroke width 取自 I，length 等於 2 x I。
3. **Relationship**：它把 editorial typography 與 Economist Red 的節制使用轉成版面系統。
4. **Design Execution**：用文字 cap height 與字母 I 的筆畫作幾何來源，讓裝飾線與字體內在比例一致。
5. **Principles**：品牌圖形裝置應由字體比例生成，而不是任意裝飾。
6. **Frameworks**：typography-derived graphic device。
7. **Patterns**：small accent rule as navigational rhythm。
8. **Rules**：AI 可檢查：accent rule 高度參照 section heading cap height；stroke width 取自 `I`；長度為 `2 x I`；用途應限於 title/header focus、feature highlight 或 text-heavy rhythm punctuation。
9. **Reusable Knowledge**：微型品牌元素可用可計算比例轉成 layout token。
10. **Abstraction**：`Visual Language`、`Layout System`。
11. **Visual Evidence**：頁面以多個圖表/標題示例重複展示 accent construction，是比例、流程與節奏證據。

## Illustrative Style

1. **Chapter Purpose**：定義 The Economist 的插畫應偏 ideas-led contemporary，而不是僅追求裝飾風格一致。
2. **Definition**：允許多種合適於品牌的 illustration styles，列出參考插畫家，禁止更改 supplied illustrations。
3. **Relationship**：插畫延伸品牌觀點與內容智慧，補足 logo/typography 無法承載的 conceptual expression。
4. **Design Execution**：不硬定單一畫風，而是以 ideas-led 與 usage permission 控制品質。
5. **Principles**：對媒體品牌而言，插畫的核心是觀點與概念，而不是視覺模板化。
6. **Frameworks**：concept-led illustration governance。
7. **Patterns**：reference artists + broad style direction + alteration ban + contact routing。
8. **Rules**：AI 可檢查：插畫 brief 應要求 ideas-led contemporary style；不得改變、扭曲、增添或改造 supplied illustrations；reuse 或 permission 問題需聯絡指定負責人。
9. **Reusable Knowledge**：插畫規範可治理創作意圖與權利，而不必過度窄化風格。
10. **Abstraction**：`Illustration System`、`Governance`。
11. **Visual Evidence**：頁面以插畫家作品作風格參照，功能是靈感與邊界證據。

## Iconography

1. **Chapter Purpose**：用成熟、開源、持續更新的 icon set 降低圖示系統維護成本，同時保留版面定位規則。
2. **Definition**：指定 Google Material Design icon set，並規範 icon 在圓形容器中的位置與大小。
3. **Relationship**：iconography 支援 UI components、apps、social 與 digital touchpoints。
4. **Design Execution**：不自建 icon library，而是採用外部標準資源並加上 placement rule。
5. **Principles**：不是所有視覺資產都需要自製；對通用互動圖示，可靠標準庫比品牌化造型更有效。
6. **Frameworks**：external-resource icon governance。
7. **Patterns**：use open source set + brand-specific containment。
8. **Rules**：AI 可檢查：icons 應來自 Google Material Design resource；icon 需依示意置於圓形內；reversed icons 需使用核准反白版本。
9. **Reusable Knowledge**：design system 可混用外部標準與品牌定位規則。
10. **Abstraction**：`Iconography System`、`UI Components`。
11. **Visual Evidence**：常用 icons 與 reversed icons 圖提供樣式與容器位置證據。

## User Interface Components

1. **Chapter Purpose**：將品牌色彩與字體轉成可實作的網站 CTA button token，支援訂閱、註冊與 editorial redirect。
2. **Definition**：定義多種 CTA button 的 height、background colour、hover、font-family、weight、padding、font-size、colour、border-radius 與 border。
3. **Relationship**：它把 typography、colour 與 icon/digital 系統落到可開發元件。
4. **Design Execution**：以 CSS-like token 直接列出規格，降低設計到前端的轉譯成本。
5. **Principles**：UI 元件規範應可被工程直接實作與測試。
6. **Frameworks**：component token specification。
7. **Patterns**：primary registration CTA、red CTA、grey CTA、editorial outline CTA。
8. **Rules**：AI 可檢查：registration CTA height 30px、font Econ Sans、font-size 14px、padding 8px 30px、border-radius 4px；blue CTA background `#3e51b5` hover `#4c60eb`；red CTA background `#e3120b` hover `#fc150d`; grey CTA background `#d7d7d7` hover `#3d51b5`; editorial CTA height 34px、background white、border 2px solid `#d7d7d7`。
9. **Reusable Knowledge**：品牌 guideline 可直接產生 design tokens 與 component lint rules。
10. **Abstraction**：`UI Components`、`Design Tokens`、`AI-Checkable Rules`。
11. **Visual Evidence**：button states 與示例 label 提供互動元件證據。

## E-newsletters

1. **Chapter Purpose**：展示品牌系統在 email newsletter 這種限制較多的媒介中如何保持可讀與一致。
2. **Definition**：涵蓋 Daily Dispatch、Editors’ Picks、Seriously Curious 等 newsletter 範例，使用 Georgia 與 Helvetica web-safe fonts。
3. **Relationship**：它延續 typography 的 web-safe fallback 與 editorial product identity。
4. **Design Execution**：以範例頁呈現內部 newsletter content 的字體選擇。
5. **Principles**：email 環境應優先可用性與跨客戶端可讀性。
6. **Frameworks**：email-safe brand application。
7. **Patterns**：newsletter examples as medium-specific proof。
8. **Rules**：AI 可檢查：e-newsletter internal content 使用 Georgia serif 與 Helvetica sans serif；不得依賴非 web-safe 字體作為唯一呈現。
9. **Reusable Knowledge**：newsletter guideline 應明確列出 email-safe fallback，而非假設網站字體可用。
10. **Abstraction**：`Content Design`、`Operations`。
11. **Visual Evidence**：newsletter 範例頁提供跨媒介應用證據。
