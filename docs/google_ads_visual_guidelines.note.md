# Google Ads Visual Guidelines

來源：`sources/google_ads_visual_guidelines.pdf`
日期：2017
研究視角：Design Infrastructure Researcher

## Summary

Google Ads visual guidelines defining brand promise, visual direction, logo lockups, color palette and accessibility, photography, illustration, iconography, animation/video, typography, and related guideline references.。建立 Google Ads 的品牌承諾、視覺方向與 logo 使用規範，讓所有後續設計能回到同一品牌感受。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。simplicity、cohesion、trust、approachability、professional friendliness。品牌框架是「brand promise -> emotional attributes -> logo system -> usage matrix」。

執行上，這份 guideline 透過 Brand Guidelines、Brand Promise and Visual Direction、Logo Lockups and Usage、Color 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。產品品牌 guideline 應同時處理品牌感受與資產選擇情境。

## Brand Guidelines

1. **Chapter Purpose**：建立 Google Ads 的品牌承諾、視覺方向與 logo 使用規範，讓所有後續設計能回到同一品牌感受。
2. **Definition**：定義 brand promise、visual direction、Google Ads logo lockups、master logo、do’s/don’ts、logo pairs、usage matrix。
3. **Relationship**：是 Color、Imagery、Typography 的上層語義來源，特別是 trustworthy、approachable、welcoming、smart、empowering、professional。
4. **Design Execution**：作者先從「makes it simple」定位產品價值，再用 logo matrix 把品牌識別落到產品與行銷場景。
5. **Principles**：simplicity、cohesion、trust、approachability、professional friendliness。
6. **Frameworks**：品牌框架是「brand promise -> emotional attributes -> logo system -> usage matrix」。
7. **Patterns**：先建立情緒與產品價值，再建立資產使用表，是大型產品品牌 guideline 的可複用架構。
8. **Rules**：AI 可檢查：Google Ads 素材應支持 simple、trustworthy、approachable、welcoming；logo 應依 product/marketing context 選擇；不得讓 logo pair 彼此競爭。
9. **Reusable Knowledge**：產品品牌 guideline 應同時處理品牌感受與資產選擇情境。
10. **Abstraction**：`Brand Strategy`、`Visual Identity`、`Governance`。
11. **Visual Evidence**：brand promise 頁、logo lockup diagrams、do/don’t examples 與 usage matrix 承擔情緒、比例與治理證據。

### Brand Promise and Visual Direction

1. **Chapter Purpose**：把產品價值與視覺情緒連接，避免視覺只像 Google 而沒有 Google Ads 的產品人格。
2. **Definition**：定義 Google Ads makes it simple，並以 trustworthy、approachable、welcoming、smart、empowering、professional 形容視覺語言。
3. **Relationship**：直接影響 photography 的人物與情境、color 的白底與 core colors、typography 的清晰可讀。
4. **Design Execution**：作者用「one smart interface」與「hint of possibility」將複雜廣告產品轉成簡單、可期待的敘事。
5. **Principles**：product simplification、empowerment、friendly professionalism、future possibility。
6. **Frameworks**：方向框架是「product complexity -> simplified interface -> customer empowerment -> friendly visual tone」。
7. **Patterns**：把軟體產品的功能價值轉譯為情緒詞，是品牌系統設計前的必要層。
8. **Rules**：AI 可檢查：視覺與文案應降低複雜感；不應呈現壓迫、冷硬或過度企業化；應讓人物/場景帶出 possibility 與 empowerment。
9. **Reusable Knowledge**：visual direction 應從產品任務出發，不只是套用母品牌元素。
10. **Abstraction**：`Brand Strategy`、`Product Language`。
11. **Visual Evidence**：開場大圖與短句承擔品牌情緒樣本功能。

### Logo Lockups and Usage

1. **Chapter Purpose**：治理 Google Ads 在產品、行銷、影片、社群、活動與 partner pairing 中的識別一致性。
2. **Definition**：定義 Google logo lockup、master logo、horizontal/vertical lockups、one-color reverse、Grey 700、Product Sans、logo pairs 與 usage matrix。
3. **Relationship**：承接品牌章，並為所有 touchpoint 建立最小識別規則。
4. **Design Execution**：作者用多種 lockup variant 對應不同場景，但用 do/don’t 和 usage table 限制選擇。
5. **Principles**：contextual variant selection、contrast, spacing, non-competing partner presence、product/marketing separation。
6. **Frameworks**：logo 決策框架是「context -> variant -> background contrast -> spacing/divider -> usage permission」。
7. **Patterns**：usage matrix 是大型組織控制資產濫用的強力模式。
8. **Rules**：AI 可檢查：product lockup 使用 Grey 700；不得增加或縮小 icon size；不得用 full-color Google logo 建構 icon lockup；兩個 logo 並列時需用 neutral gray vertical divider 與適當 spacing；vertical logo 只在特定情境如 video attribution 較適合。
9. **Reusable Knowledge**：logo guideline 應把「可用資產」與「使用情境」合併成可審核矩陣。
10. **Abstraction**：`Visual Identity`、`Brand Architecture`、`AI-Checkable Rules`。
11. **Visual Evidence**：logo variants、pairing do/don’t 與 usage table 是主要治理證據。

## Color

1. **Chapter Purpose**：確保 Google Ads 保持 Google 可辨識性，同時透過白底、核心色與 neutrals 建立焦點、可讀性與 accessibility。
2. **Definition**：定義 Google Blue #4285F4、Red #EA4335、Yellow #FBBC04、Green #34A853、White、dark tones、light tones、neutrals Grey 50-900、color hierarchy 與 principles。
3. **Relationship**：支援 logo、illustration、typography、icons、UI elements、data visualization。
4. **Design Execution**：作者從 palette 進到比例與原則：build from white、accessibility、avoid overload、create focus、use tonal depth。
5. **Principles**：build from white、accessibility first、controlled color energy、neutral utility、focus through restraint。
6. **Frameworks**：色彩框架是「core colors -> neutrals -> white dominance -> accessibility contrast -> focus/depth use cases」。
7. **Patterns**：以白色作為主色而非空白，是 Google 系統的關鍵設計模式。
8. **Rules**：AI 可檢查：white 應為最主要顏色並佔最多空間；Grey 700 headline contrast 6.5:1 可通過 AA；yellow headline contrast 1.7:1 不可作 headline；避免大面積彩色造成 white space 風險；links 應使用 700/800/900 values；light 50/100 適合 shading、data visualization 或 UI focus。
9. **Reusable Knowledge**：色彩 guideline 應把 brand recognition、accessibility 與 hierarchy 同時規範。
10. **Abstraction**：`Color System`、`Accessibility`、`Design Tokens`、`AI-Checkable Rules`。
11. **Visual Evidence**：色票、contrast do/don’t、color overload examples 與 tonal examples 承擔 token、警示與可及性證據。

## Imagery

1. **Chapter Purpose**：定義照片、插畫與 icon 在 Google Ads 中各自何時使用、如何幫助使用者、如何避免裝飾性浪費。
2. **Definition**：涵蓋 photography、lifestyle、metaphorical storytelling、situational notes、image usage、brand references、recommended studios、illustration、iconography。
3. **Relationship**：承接 Brand 的 smart/empowering/trustworthy，Color 的 build from white，以及 Typography 的 clarity。
4. **Design Execution**：作者以「Focus on the user、Build for everyone、Build from white」貫穿照片與插畫。
5. **Principles**：user usefulness、inclusion、authenticity、simplicity、brand restraint、problem-solving over decoration。
6. **Frameworks**：imagery 框架是「media type -> user value -> inclusivity -> simplicity/white -> brand palette -> usage constraints」。
7. **Patterns**：將不同影像媒介統一到同一三原則，可減少跨團隊判斷分裂。
8. **Rules**：AI 可檢查：影像應對使用者有幫助；不能只填補空白；應呈現多元全球受眾；應避免複雜場景、外部品牌與競品產品。
9. **Reusable Knowledge**：影像 guideline 應先問「是否需要影像」，再問「哪種影像正確」。
10. **Abstraction**：`Photography System`、`Illustration System`、`Iconography System`、`Content Safety`。
11. **Visual Evidence**：大量照片、插畫、do/don’t 與情境範例承擔示範、比較、警示與情境證據。

### Photography

1. **Chapter Purpose**：確保照片真實、包容、聚焦使用者，並能把 Google Ads 的產品價值放進人的生活與工作脈絡。
2. **Definition**：定義 meaningful stories、products in people’s lives、diverse worldwide audience、UX personalization、art direction、lifestyle、metaphorical storytelling、situational notes、usage standards。
3. **Relationship**：把 brand attributes 轉成人物、場景、構圖與解析度規則。
4. **Design Execution**：作者用四件好照片要做到的事，加上 lifestyle/metaphorical/situational 分節建立可選圖語法。
5. **Principles**：authenticity、diversity、single focal point、simplicity、real businesses、brand cleanliness。
6. **Frameworks**：攝影框架是「story -> casting/setting -> focus -> device/product role -> resolution -> brand reference cleanup」。
7. **Patterns**：以 narrative、interaction、visual focus 三個 situational notes 管理主觀照片品質。
8. **Rules**：AI 可檢查：照片應有 single primary center of attention；背景不應 cluttered；casting 不得限於單一性別、種族或群體；slides online 應至少 72 dpi，可能印刷 PDF 應至少 300 dpi；含科技產品時只可出現 Google-based products，且不可出現 competitor products；外部品牌需移除或遮蓋。
9. **Reusable Knowledge**：攝影規範可把 inclusion、resolution、brand safety 與 narrative quality 結合成同一審核 checklist。
10. **Abstraction**：`Photography System`、`Accessibility`、`Content Safety`、`AI-Checkable Rules`。
11. **Visual Evidence**：lifestyle/metaphor pages、do/don’t image guideline、resolution/framing examples 與 brand reference examples 是核心證據。

### Illustration

1. **Chapter Purpose**：將抽象概念、流程與複雜資訊轉成清楚、可文化通用的視覺，而不是裝飾頁面。
2. **Definition**：定義 illustration 用於 expressing abstract concepts、distilling complex ideas、culturally agnostic situations/locations；包含 overview、build from white、colors、shape、light、background、people/inclusion、hands、symbols。
3. **Relationship**：延續 Color 的白底與核心色，也補足攝影無法表現的抽象或文化中立場景。
4. **Design Execution**：作者先定義「inform, not decorate」，再逐步規範色彩、形狀、光影、人物與符號。
5. **Principles**：informational usefulness、clarity at a glance、inclusivity、palette discipline、cultural mindfulness。
6. **Frameworks**：插畫框架是「problem to explain -> simple metaphor -> white field -> palette balance -> inclusive people/symbols -> one-second comprehension」。
7. **Patterns**：one-second comprehension 是可複用的插畫複雜度判準。
8. **Rules**：AI 可檢查：插畫若超過一秒才看懂則過於複雜；不得使用 palette 外色彩；不需在單張插畫強迫使用四個核心色；同一組物件應有一致光源；符號不得依賴地方性節日或文化參照；人物應反映多文化、年齡與族群。
9. **Reusable Knowledge**：插畫 guideline 應將資訊目的、文化風險與品牌 token 一起管理。
10. **Abstraction**：`Illustration System`、`Accessibility`、`Localization`、`AI-Checkable Rules`。
11. **Visual Evidence**：build-from-white、palette do/don’t、shape/light/background/people examples 承擔示範、比較與警示證據。

### Iconography

1. **Chapter Purpose**：用最小化、友善且可讀的小圖示支援命令、檔案、裝置、目錄、行動與 wayfinding。
2. **Definition**：定義 system icons as simple, friendly visual system，代表 common actions like trash、print、save 等。
3. **Relationship**：iconography 是 imagery 的功能性低階元素，支援 UI 與資訊標記。
4. **Design Execution**：作者強調 icon 應將 idea reduced to its essence，並透過單一資產庫統一管理。
5. **Principles**：minimalism、readability at small sizes、usefulness、friendly delight、centralized asset management。
6. **Frameworks**：icon 框架是「common action/object -> essential form -> small-size readability -> repository access」。
7. **Patterns**：用單一 repository 管理圖示資產，是大組織 icon governance 的核心模式。
8. **Rules**：AI 可檢查：icon 應簡單到小尺寸仍可讀；不得用過度細節削弱辨識；應從官方 icons repository 取得。
9. **Reusable Knowledge**：icon guideline 應同時定義造型原則與取得/管理流程。
10. **Abstraction**：`Iconography System`、`UI Components`、`Operations`。
11. **Visual Evidence**：icon examples 與 access instructions 承擔風格與操作證據。

## Animation and Video

1. **Chapter Purpose**：讓動態內容在增進清楚度的同時符合品質、頻寬、可讀性、包容性與品牌審核要求。
2. **Definition**：定義 h.264/prores delivery specs、resolution、bitrate、audio、frame rate、aspect ratio、bandwidth considerations、creative review checklist。
3. **Relationship**：延伸 imagery 與 typography 到時間性媒體，並把 accessibility/performance 納入創意審查。
4. **Design Execution**：作者先列低頻寬與高頻寬交付規格，再用 review questions 檢查故事與品牌適配。
5. **Principles**：clarity through motion、bandwidth empathy、quality control、inclusion、current product representation。
6. **Frameworks**：動態框架是「delivery specs -> story clarity -> bandwidth -> brand fit -> inclusion/product/currentness/readability review」。
7. **Patterns**：把 technical delivery specs 與 creative checklist 放在同章，可避免創意與工程各自脫節。
8. **Rules**：AI 可檢查：低頻寬 h.264 應為 1280 x 720、1 pass VBR、3 Mbps max、1.5 Mbps average、128 kbps audio、23.98 progressive、16:9；不得使用 QuickTime h.264；影片需檢查是否 Google Ads、是否有 relatable believable story、是否 inclusive、產品是否 up-to-date、標題/字幕是否 readable。
9. **Reusable Knowledge**：video guideline 應同時規範檔案交付、效能與創意審核。
10. **Abstraction**：`Motion Identity`、`Performance`、`Accessibility`、`AI-Checkable Rules`。
11. **Visual Evidence**：spec table、動畫前後對照與 creative checklist 承擔技術與審核證據。

## Typography

1. **Chapter Purpose**：用 Google Sans、Google Sans Display、Roboto、Product Sans 與 Noto 建立品牌層級、可讀性與多語支援。
2. **Definition**：定義 overview、three Google typefaces、Google Slides typography、titles/headlines、paragraphs、hierarchy、line height、weight、legibility、specific use cases。
3. **Relationship**：Typography 承接 focus on user、build for everyone、build from white，確保所有內容清楚可讀。
4. **Design Execution**：作者依文字大小與用途分配字體，並提供 slides scaling、case、weight、line height、line length 與非拉丁語系規則。
5. **Principles**：hierarchy、legibility、accessibility、sentence case、restraint in emphasis、script coverage。
6. **Frameworks**：字體框架是「use case/size -> typeface -> color -> hierarchy -> line height/length -> emphasis -> language coverage」。
7. **Patterns**：以 size threshold 決定 typeface 是可機器檢查的 typography pattern。
8. **Rules**：AI 可檢查：Roboto、Google Sans、Google Sans Display 是 Google Ads brand 內使用的字體；36pt+ 大標使用 Google Sans Display，18-35pt headline 使用 Google Sans，16pt 以下 body/smaller text 使用 Roboto；Google Slides 字級需除以二；Google Sans/Display 不得 all caps；title 應 sentence case；body text 使用 Google Material 700 Grey #5F6368；headline 使用 Google Material 800 Grey #3C4043；bold 只可 sparingly 用於 emphasis；Product Sans 只可用於 product logo lockups；Noto 用於 non-Latin languages。
9. **Reusable Knowledge**：typography guideline 應把字體、尺寸、語言、可讀性與強調規則整合成一套可驗證系統。
10. **Abstraction**：`Typography System`、`Accessibility`、`Localization`、`AI-Checkable Rules`。
11. **Visual Evidence**：type specimens、Slides example、headline/body examples、line height/weight/legibility do/don’t 承擔規格與警示證據。

## Related Guidelines

1. **Chapter Purpose**：將 Google Ads Visual Guidelines 放入更大的 Google guideline ecosystem，避免設計者孤立使用本文件。
2. **Definition**：列出 Google Ads Editorial Guide、Google Ads Brand Overview、Google Standards、Google Web Standards、Material Design、Imagery、Marketing Web Standards。
3. **Relationship**：為 voice、brand essence、web/product standards 提供外部依賴與延伸。
4. **Design Execution**：作者用 related materials 收束，告訴使用者何時應轉查其他 guide。
5. **Principles**：ecosystem consistency、source-of-truth routing、cross-guideline dependency。
6. **Frameworks**：關聯框架是「visual guide -> editorial/brand overview -> Google standards -> product/web/material references」。
7. **Patterns**：大型 design system 應以 related guidelines 建立知識圖譜，而不是讓每份文件孤島化。
8. **Rules**：AI 可檢查：涉及 voice 應查 Editorial Guide；涉及 brand essence/personas/value propositions 應查 Brand Overview；涉及 web/product/material 應查 Google Standards 或 Material Design。
9. **Reusable Knowledge**：Guideline knowledge base 需要明確 cross-reference，才能支援複雜組織。
10. **Abstraction**：`Knowledge Architecture`、`Governance`。
11. **Visual Evidence**：related materials list 作為知識路由證據。
