# BigCommerce Brand Guidelines

來源：`sources/BC_BrandGuidelines_2020.pdf`  
版本：Q2 2020  
日期：2020  
研究視角：Design Infrastructure Researcher

## Summary

BigCommerce 品牌指南以 Build, Innovate, Grow 訊息框架為核心，規範 logo、色彩、字體、幾何圖形、攝影、圖示、資料視覺化、數位、影片、簡報與品牌延伸。此章建立 BigCommerce 的品牌敘事骨架，讓後續視覺規則能服務 Build/Innovate/Grow 的商務成長敘事。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。原則是平台品牌要把複雜產品能力轉成可記憶的成長語言。框架是 Build -> Innovate -> Grow，對應 foundation、flexibility、scale。

執行上，這份 guideline 透過 Our Brand and Messaging Framework、2020 Updates、Build, Innovate, Grow、Our Logo 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。通用知識是 B2B SaaS 品牌可用少數動詞建立跨產品、受眾與內容的分類基礎。

## Our Brand and Messaging Framework

1. **Chapter Purpose**：此章建立 BigCommerce 的品牌敘事骨架，讓後續視覺規則能服務 Build/Innovate/Grow 的商務成長敘事。
2. **Definition**：它定義 2020 版更新重點與品牌訊息框架；邊界是不細述每個視覺資產規格。
3. **Relationship**：此章是全文件的 strategy layer，後續 logo、geometry、imagery、video 與 partner ecosystem 都應回應 go B.I.G.。
4. **Design Execution**：作者用三動詞框架把平台能力轉成高層利益，便於分類 features、audiences 與 content。
5. **Principles**：原則是平台品牌要把複雜產品能力轉成可記憶的成長語言。
6. **Frameworks**：框架是 Build -> Innovate -> Grow，對應 foundation、flexibility、scale。
7. **Patterns**：可複用模式包含年度更新頁、三段式 messaging framework、產品功能到品牌利益的映射。
8. **Rules**：可檢查規則：對外訊息應能對應 Build/Innovate/Grow 之一；不得只描述功能而不說明商家利益；2020 新素材需反映 imagery/video/slides/partners 更新。
9. **Reusable Knowledge**：通用知識是 B2B SaaS 品牌可用少數動詞建立跨產品、受眾與內容的分類基礎。
10. **Abstraction**：Guideline 模組類型：Brand Strategy、Messaging、Knowledge Architecture。
11. **Visual Evidence**：視覺證據包含更新頁、三段訊息與 TOC，功能是建立品牌語意地圖。

## Our Logo

1. **Chapter Purpose**：此章保護 BigCommerce 的成長符號與主標誌，並把 logo 使用延伸到 partner ecosystem。
2. **Definition**：它定義三角語意、wordmark、B mark、clearspace、尺寸、禁例與 partner lockup；邊界是不處理完整合作法律規範。
3. **Relationship**：Logo 章承接 Build/Innovate/Grow 的 growth/scalability 語意，並影響所有數位與合作素材。
4. **Design Execution**：作者將符號意義、尺寸門檻與錯誤用法並列，讓 logo governance 能由設計與非設計團隊共同執行。
5. **Principles**：原則是成長方向要被穩定保留，小尺寸需切換 mark，品牌不可被任意改色或變形。
6. **Frameworks**：框架是 Logo System：primary wordmark、secondary B mark、clearspace、minimum size、partner lockup。
7. **Patterns**：可複用模式包含符號語意說明、mark fallback、Y/2 留白、improper use wall、co-brand divider。
8. **Rules**：可檢查規則：B mark 只在 wordmark 低於 80px/1in 或避免重複時使用；clearspace 至少為各自高度 Y/2；不得拉伸、堆疊、改色、加陰影或修改兩個 mark；partner divider stroke 為 #AFB4BE 1.5px。
9. **Reusable Knowledge**：通用知識是 SaaS 品牌 logo 指南應處理小尺寸數位情境與合作 lockup，而非只定義主標誌。
10. **Abstraction**：Guideline 模組類型：Visual Identity、Governance、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據包含 logo、B mark、clearspace、minimum sizes、improper use 與 partner lockup 圖。

## Color Palette

1. **Chapter Purpose**：此章建立 BigCommerce 跨網站、行銷與印刷的色彩 token 系統。
2. **Definition**：它定義主色、次色、灰階、印刷/螢幕數值與漸層；邊界是不規定每個 UI state。
3. **Relationship**：Color 支撐 typography、geometry、website、data visualization 與 imagery，尤其以高對比維持可讀性。
4. **Design Execution**：作者用使用比例與 swatch breakdown，把品牌藍的歷史延續轉成現代可生產色盤。
5. **Principles**：原則是 black/white/blue 提供核心對比，secondary colors sparingly 作 accent，gradients 不得產生 banding。
6. **Frameworks**：框架是 Primary Foundation + Secondary Accent + Grayscale + Gradients。
7. **Patterns**：可複用模式包含多色值 token、use ratio、screen/print grayscale 分流、gradient stop 說明。
8. **Rules**：可檢查規則：核心 palette 應使用 #121118、#FFFFFF、#0D52FF 等核准值；purple/green 為主要 accent 且應節制；pink 為 tertiary；Twilight/Northern Lights 等漸層需保留 middle color-stops 防止 banding。
9. **Reusable Knowledge**：通用知識是色彩指南應明確描述使用比例與媒介差異，否則 token 會被過度使用。
10. **Abstraction**：Guideline 模組類型：Color System、Design Tokens、Accessibility、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據是色票、數值表、灰階梯度與漸層樣張。

## Typography

1. **Chapter Purpose**：此章讓 BigCommerce 的資訊密集 B2B 溝通具備清楚階層與強品牌聲量。
2. **Definition**：它定義 Gotham、Gotham Condensed Bold、Source Sans Pro fallback、H1-H6/body 規格與 highlight；邊界是不覆蓋所有網頁 CSS。
3. **Relationship**：Typography 承接 logo 的幾何語言與 color 的 accent 系統，並支援 website、slides、data visualization。
4. **Design Execution**：作者把網站字級作為全品牌 hierarchy foundation，讓數位產品與品牌素材共享排版起點。
5. **Principles**：原則是 legible、strong、bold、information-oriented、clear hierarchy。
6. **Frameworks**：框架是 Brand Font + Web Safe Fallback + Type Scale + Accent Highlight。
7. **Patterns**：可複用模式包含 brand font/fallback 分層、H1 highlight single-word emphasis、type scale token。
8. **Rules**：可檢查規則：Gotham 為主要字體；Source Sans Pro 只在 Gotham 不可用如 product/Google Docs 時使用；H1 highlight 使用 #03EADA 且只強調單字或單行；highlight 位置約在 x-height 一半並向右偏移約半個字寬。
9. **Reusable Knowledge**：通用知識是品牌 typography 可用現有網站字級作為跨媒介起點，但需定義 fallback 邊界。
10. **Abstraction**：Guideline 模組類型：Typography System、Design Tokens、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據包含字重樣張、type scale、highlight position diagram。

## Geometry and Patterns

1. **Chapter Purpose**：此章把 logo 的三角成長語意擴展為可大規模使用的視覺語言。
2. **Definition**：它定義幾何形、pattern 類型、尺寸限制、文字疊加與對比；邊界是不取代插畫或 UI component library。
3. **Relationship**：Geometry 連接 logo、color、imagery 與 website，讓品牌在缺少攝影時仍有可識別背景語言。
4. **Design Execution**：作者以基本形狀和 pattern 生成可擴展元素，兼顧 variety 與 cohesion。
5. **Principles**：原則是 rational universal shapes、科技感、可擴展、不可干擾內容可讀性。
6. **Frameworks**：框架是 90° Triangle -> Shape Palette -> Pattern Variants -> Legibility Controls。
7. **Patterns**：可複用模式包含幾何形作為品牌 building blocks、stroked vs bold patterns、pattern/text contrast rules。
8. **Rules**：可檢查規則：stroked pattern square 尺寸約 15px 至 115px；文字疊在 intricate pattern 上需高對比、大而粗；bold pattern 背景與形狀漸層整體對比應較低；不得讓 pattern 與前景 copy 競爭。
9. **Reusable Knowledge**：通用知識是品牌 pattern 必須附帶可讀性規則，否則最容易在行銷素材中傷害訊息。
10. **Abstraction**：Guideline 模組類型：Visual Language、Layout System、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據包含形狀組、stroked/bold pattern、尺寸與疊字示例。

## Photography

1. **Chapter Purpose**：此章用人本攝影抵消 SaaS 平台的抽象性，讓技術品牌有可感的商務成果。
2. **Definition**：它定義員工、商家、顧客與產品情境的影像方向；邊界是不規定每次拍攝的 production checklist。
3. **Relationship**：Photography 連接 messaging 的 growth story 與 website/marketing 的視覺證據。
4. **Design Execution**：作者要求展示 collaboration、interaction、merchant journeys 與 product versatility，讓品牌價值從功能變成故事。
5. **Principles**：原則是 human-centric、bright、modern、creative、真實商務情境。
6. **Frameworks**：框架是 People Behind Platform + Merchants Using Platform + Customers Experiencing Brands。
7. **Patterns**：可複用模式包含員工環境照、portrait、merchant/customer journey、重用商家自有 imagery 作平台彈性證據。
8. **Rules**：可檢查規則：影像應有人、互動或商務成果；不得只呈現無脈絡科技抽象圖；商家圖像應服務 growth/versatility 敘事；人物照應反映 bright modern creative spaces。
9. **Reusable Knowledge**：通用知識是 B2B SaaS 攝影需要把平台能力外化為人、商家與顧客互動。
10. **Abstraction**：Guideline 模組類型：Photography System、Brand Storytelling、Content Design。
11. **Visual Evidence**：視覺證據包含環境照、人像、商家/顧客/產品案例，功能是建立人本與成果證明。

## Iconography and Data Visualization

1. **Chapter Purpose**：此章處理抽象資訊和產品功能的圖像化，讓品牌在 UI、文件與行銷中保持同一資訊語言。
2. **Definition**：它定義圖示家族與 data visualization；邊界是不完整取代產品設計系統。
3. **Relationship**：Iconography/Data Visualization 依賴 geometry、color 與 typography，是資訊設計層的延伸。
4. **Design Execution**：作者按使用場景分流 icon，並以品牌色與層級處理資料呈現，降低混用風格的風險。
5. **Principles**：原則是清楚、可掃描、場景適配、資料優先於裝飾。
6. **Frameworks**：框架是 Contextual Icon Families + Branded Data Display。
7. **Patterns**：可複用模式包含行銷 icon、產品 icon、help/developer icon 分層，以及圖表色彩與標籤規則。
8. **Rules**：可檢查規則：icon 風格應符合所屬場景；資料圖不得濫用 accent 色；圖表需有清楚標籤與對比；不可用裝飾圖形模糊數值。
9. **Reusable Knowledge**：通用知識是同一品牌需要區分 marketing icons 與 product/documentation icons，因為資訊密度與使用任務不同。
10. **Abstraction**：Guideline 模組類型：Iconography System、Data Visualization、Information Design、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據包含各類 icon set 與圖表範例，功能是建立資訊語言的可比較標準。

## Digital, Media, and Brand Extensions

1. **Chapter Purpose**：此章把品牌系統推向實際媒介與子系統，處理大型 SaaS 品牌最容易碎裂的數位、影片、簡報、合作與內部品牌。
2. **Definition**：它定義多媒介應用與品牌延伸；邊界是不取代各團隊完整操作手冊。
3. **Relationship**：此章依賴前面所有 visual tokens 與 messaging，是品牌從規則走向營運規模化的層。
4. **Design Execution**：作者按媒介與生態系分章，讓 guideline 不只服務設計素材，也服務 sales enablement、partner marketing、product packaging 與內部溝通。
5. **Principles**：原則是一致品牌在不同媒介需有媒介原生規則，而不是只貼 logo。
6. **Frameworks**：框架是 Core Identity -> Channel Systems -> Ecosystem/Extension Governance。
7. **Patterns**：可複用模式包含 website rules、video production guidance、slide templates、partner ecosystem visual refresh、brand extension taxonomy。
8. **Rules**：可檢查規則：數位/影片/簡報需使用核准 logo、色彩、字體與圖像風格；partner 素材需使用標準化 partnership system；BigCommerce Essentials 與 extensions 不得建立無關主品牌的新視覺語言。
9. **Reusable Knowledge**：通用知識是成熟品牌指南要管理 channel-specific systems 和 brand extensions，否則核心規則無法覆蓋營運現實。
10. **Abstraction**：Guideline 模組類型：Operations、Governance、Digital Design、Motion Identity、Brand Architecture。
11. **Visual Evidence**：視覺證據包含網站、isometric imagery、video frame、slide templates、partner 與品牌延伸範例，功能是展示跨媒介一致性。
