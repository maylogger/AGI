# Olympic Games Tokyo 2020 Emblem Usage Guidelines for National Olympic Committees

來源：`sources/Tokyo_2020_NOC_Olympic_emblem_2018.pdf`
版本：2.0
日期：April 2018
研究視角：Design Infrastructure Researcher

## Summary

Tokyo 2020 提供給 National Olympic Committees 的 emblem 使用規範，聚焦非商業授權、標誌組成、色彩、背景、錯誤使用、wordmark、字體與特定媒介範例。先界定 NOC 可用 Tokyo 2020 IP 的法律與商業邊界，避免 emblem 被誤用為商業背書。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。奧運品牌治理優先處理權利與關聯風險，再處理視覺一致性。rights-first guideline：user role → purpose → allowed assets → association limits → approval/access path。

執行上，這份 guideline 透過 Introduction and NOC Usage Conditions、Emblem System、Emblem Components、Clear Space, Size, Colour and Background 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。品牌知識庫應將「使用者角色與用途」列為第一層規則，優先於視覺 token。

## Introduction and NOC Usage Conditions

1. **Chapter Purpose**：先界定 NOC 可用 Tokyo 2020 IP 的法律與商業邊界，避免 emblem 被誤用為商業背書。
2. **Definition**：定義 Tokyo 2020 IP、NOC 非商業使用權、服裝使用限制、commercial entities 權利結構、素材取得與聯絡流程。
3. **Relationship**：此章是後續所有視覺規則的前置條件；若使用情境不合法，後面比例與色彩即使正確也不可用。
4. **Design Execution**：用明確角色分類 NOC、IF、Worldwide Olympic Partners、Tokyo 2020 Sponsors，並把 apparel 使用條件轉成尺寸與組合限制。
5. **Principles**：奧運品牌治理優先處理權利與關聯風險，再處理視覺一致性。
6. **Frameworks**：rights-first guideline：user role → purpose → allowed assets → association limits → approval/access path。
7. **Patterns**：對高價值 IP，guideline 應先設 permission scope，而非直接展示 logo artwork。
8. **Rules**：NOC 使用限非商業目的；Tokyo 2020 Emblem 或 Wordmark 每件 clothing 只能使用一次，最大 30 cm2 或 44mm width；不得用於 licensed / replica merchandise；wordmark 不得與 manufacturer identification 關聯；emblem 與其他 authorized identification 不得並置。
9. **Reusable Knowledge**：品牌知識庫應將「使用者角色與用途」列為第一層規則，優先於視覺 token。
10. **Abstraction**：`Legal and Trademark`、`Compliance`、`Governance`。
11. **Visual Evidence**：此章主要是權利與條款文字，功能是授權範圍與風險邊界證據。

## Emblem System

1. **Chapter Purpose**：保護 Tokyo 2020 emblem 的組成完整性、可辨識性與 Olympic symbol 的權威。
2. **Definition**：包含 emblem components、clear space、minimum size、display colours、background coordination 與 incorrect usage。
3. **Relationship**：承接 usage conditions，將可用情境轉成具體視覺驗收條件。
4. **Design Execution**：把 Games Distinctive Element、wordmark、Olympic symbol 與 trademark 視為不可任意拆分的整體，並以尺寸、間距與背景版本控制再現。
5. **Principles**：大型國際賽事識別的信任來自「完整呈現」與「不得與其他識別混淆」。
6. **Frameworks**：emblem integrity model：component completeness → isolation → size → colour reproduction → background suitability → misuse rejection。
7. **Patterns**：多元素 emblem 需同時規範 component、spacing、exception、reproduction medium 與 prohibited manipulation。
8. **Rules**：Games Distinctive Element、wordmark、Olympic symbol 不得分離，除非特別授權；一般應包含 TM；emblem 寬度小於 30mm 時可省略 TM；full-colour 與 monochrome emblem 最小不得低於 12mm / 35px；必須使用原始資料。
9. **Reusable Knowledge**：AI 審稿可先檢查 emblem 是否完整、是否保留必要商標符號，再檢查尺寸與色彩。
10. **Abstraction**：`Visual Identity`、`Legal and Trademark`、`AI-Checkable Rules`。
11. **Visual Evidence**：組成圖、0.7X clear space、色票、背景示例與 incorrect usage grid 是主要比例與警示證據。

### Emblem Components

1. **Chapter Purpose**：明確說明每個 emblem 元件的角色，避免使用者只拿其中一部分代表 Tokyo 2020。
2. **Definition**：定義 Games Distinctive Element、Wordmark、Olympic symbol 與 Trademark。
3. **Relationship**：此節支撐後續 incorrect usage 中「不得改順序、不得獨立使用元素」的規則。
4. **Design Execution**：用 component label 建立識別結構，讓使用者理解它不是普通 logo，而是多重權利元素的組合。
5. **Principles**：當標誌包含國際組織符號時，元件拆分等於權利與意義錯置。
6. **Frameworks**：component dependency：distinctive element + event wordmark + Olympic symbol + trademark。
7. **Patterns**：可複用於任何包含 partner / governing body symbol 的活動品牌。
8. **Rules**：不得單獨使用 Games Distinctive Element；wordmark 必須保持官方 city name + year 組成；Olympic symbol 不得與 emblem 拆分；TM 預設應包含。
9. **Reusable Knowledge**：多權利標誌可在知識庫中標記 required components 與 allowed exceptions。
10. **Abstraction**：`Visual Identity`、`Legal and Trademark`。
11. **Visual Evidence**：元件標註圖用來證明組成、順序與必要元素。

### Clear Space, Size, Colour and Background

1. **Chapter Purpose**：確保 emblem 在各種媒介中清楚、可辨識且不被背景干擾。
2. **Definition**：定義 clear space、minimum size、Olympic 色彩、monochrome versions、boxed emblem 與背景協調。
3. **Relationship**：把 component integrity 轉成版面與輸出條件。
4. **Design Execution**：使用 X 值與 0.7X 留白，指定白底 full colour、複雜或色底 boxed emblem，以及黑白 / 白色例外。
5. **Principles**：背景複雜度是 logo version selection 的關鍵，不應只靠設計者目測。
6. **Frameworks**：background decision tree：white background → full colour；photographic / coloured / complex background → boxed, white, or black variant per contrast；avoid busy monochrome use。
7. **Patterns**：以背景類型驅動版本選擇，可用於自動品牌 asset selection。
8. **Rules**：clear space 應至少 0.7X；print 最小 12mm，digital 最小 35px；full-colour emblem 應用白底；photo 或 colour background 應使用 boxed emblem；不得在色底上直接使用 full-colour emblem；不得加邊框。
9. **Reusable Knowledge**：Logo rule engines 應將背景色 / 複雜度、對比與可用版本納入檢查。
10. **Abstraction**：`Layout System`、`Color System`、`Accessibility`。
11. **Visual Evidence**：色票、背景對照、boxed emblem 與禁用例提供可判斷的視覺樣本。

### Incorrect Usage

1. **Chapter Purpose**：以負面範例建立不可跨越的視覺邊界。
2. **Definition**：列出 outline、gold / silver print、hot stamping、改色、變形、旋轉、換字體、改順序、contour、3D、shadow、flip、組合其他元素、單獨使用 Games element 等禁用。
3. **Relationship**：此節是 emblem system 的執法層，補足正向規範無法涵蓋的常見錯誤。
4. **Design Execution**：用 grid 式錯誤樣本讓審核者快速比對。
5. **Principles**：對高辨識活動標誌而言，任何視覺特效或重組都可能稀釋官方性。
6. **Frameworks**：misuse taxonomy：appearance effects、geometry changes、component changes、association changes。
7. **Patterns**：incorrect usage 應以清楚動詞開頭，讓規則可被直接轉成審核條件。
8. **Rules**：不得 outline、金銀燙、改色、變形、傾斜、旋轉、替換 wordmark font、改變元件順序、創建 contour area、改比例、3D、陰影、翻轉、與其他元素組合或單獨使用 distinctive element。
9. **Reusable Knowledge**：這類清單非常適合轉成 computer vision / asset metadata 檢查項。
10. **Abstraction**：`AI-Checkable Rules`、`Compliance`。
11. **Visual Evidence**：錯誤示例頁本身就是視覺比對資料集。

## Tokyo 2020 Wordmark

1. **Chapter Purpose**：針對 NOC 在服裝上的 wordmark 使用建立更細的關聯與分離規則。
2. **Definition**：定義非競賽服、競賽服、訓練服上的 wordmark 使用，以及與 manufacturer logo、NOC emblem、IF identification 的關係。
3. **Relationship**：延伸 Introduction 的服裝規則，並補上 wordmark 專屬 clear space、minimum size 與 incorrect usage。
4. **Design Execution**：允許 wordmark 出現在特定服裝類型，但要求與製造商識別保持距離，與 NOC emblem 共同使用時需置於下方並清楚分隔。
5. **Principles**：wordmark 可提升 Olympic identity，但不可被商業品牌借位。
6. **Frameworks**：apparel association model：item type → allowed mark → prohibited adjacency → hierarchy → approval。
7. **Patterns**：服裝應用章可將 sponsorship / manufacturer / governing body marks 的相對位置寫成規則。
8. **Rules**：非競賽服可使用 wordmark，但不得直接與 manufacturer logo 關聯；競賽服與訓練服若與 NOC emblem 一起使用，wordmark 必須在 NOC emblem 下方且有清楚 separation；使用需送 IOC approval；minimum size print 12mm、digital 35px；clear space 0.5X。
9. **Reusable Knowledge**：多品牌服裝規範可抽象成「關聯距離」與「垂直階層」檢查。
10. **Abstraction**：`Co-branding`、`Legal and Trademark`、`Compliance`。
11. **Visual Evidence**：wordmark 頁以 clear space 圖、最小尺寸與 apparel 情境文字提供規則證據。

## Text Fonts

1. **Chapter Purpose**：為 Tokyo 2020 相關溝通材料建立一致的文字外觀，同時保留媒介例外。
2. **Definition**：指定 DIN family 的 Light、Regular、Bold、Black 字重，並允許在廣告、網站或印刷材料中使用更合適字體。
3. **Relationship**：它是 emblem / wordmark 之外的輔助視覺語言，支撐 news release、handbook、website 等應用。
4. **Design Execution**：以 DIN 作優先字體，透過字重搭配控制官方感與清晰度。
5. **Principles**：活動品牌可指定 preferred font，但在特殊媒介上允許務實例外。
6. **Frameworks**：preferred typeface with contextual exceptions。
7. **Patterns**：字體規範不一定要完全封閉，可用「preferably」與 deemed appropriate 控制彈性。
8. **Rules**：Tokyo 2020 相關溝通材料應優先使用 DIN Light、Regular、Bold 或 Black；若廣告、網站或印刷情境有更適合字體，可在合理情況下使用其他字體。
9. **Reusable Knowledge**：字體規則應標記 mandatory vs preferred，讓 AI 不會把建議規則誤判為絕對禁止。
10. **Abstraction**：`Typography System`、`Operations`。
11. **Visual Evidence**：此章文字與字體說明為主，視覺證據較少，功能是輔助識別標準。

## Application Examples

1. **Chapter Purpose**：展示 NOC 可用 emblem 的實際媒介，同時提醒商業與第三方品牌限制。
2. **Definition**：包含 news release、handbook、letterhead、business card、website 與 incorrect website usage。
3. **Relationship**：把前面權利、emblem、wordmark、font 規則放進真實輸出場景。
4. **Design Execution**：用範例頁說明哪些媒介是 editorial / official context，哪些會因 commercial association 被禁止。
5. **Principles**：Examples 不只是美術示範，而是權利邊界的場景化解釋。
6. **Frameworks**：application governance：medium → user eligibility → commercial separation → third-party brand restrictions。
7. **Patterns**：高風險 IP guideline 應在範例中直接寫出「看似合理但禁止」的情境。
8. **Rules**：printed materials / publications 不得用於 commercial or promotional purposes；business card 僅限參與 Tokyo 2020 Games 的 NOC management staff，且 Tokyo 2020 emblem 必須小於 NOC emblem；envelopes、postcards 與含第三方品牌材料不得使用；website 上 Tokyo 2020 emblem 必須與 commercial contents 明確分離，僅 Worldwide Olympic Partners 可作為商業品牌例外。
9. **Reusable Knowledge**：應用範例可轉成媒介政策表，檢查角色資格、第三方品牌、商業內容與相對尺寸。
10. **Abstraction**：`Examples`、`Compliance`、`Legal and Trademark`。
11. **Visual Evidence**：news release、handbook、letterhead、business card、website mockup 與 incorrect usage 是主要情境證據。

## Revision and Contact

1. **Chapter Purpose**：提供變更與諮詢路徑，維持規範隨賽事治理調整的能力。
2. **Definition**：說明 Guidelines 可能不定期修訂，並提供 NOC Services Relations 聯絡方式。
3. **Relationship**：承接所有需送審、例外或不確定的規則，形成 governance loop。
4. **Design Execution**：以簡短修訂聲明與 email 收束文件。
5. **Principles**：大型活動 IP 規範必須可更新，因 sponsor、權利與營運條件會變動。
6. **Frameworks**：living guideline + contact owner。
7. **Patterns**：在 time-bound event brand 中，revision clause 是必要操作元件。
8. **Rules**：若 guideline 有修訂，使用者應以最新版本為準；任何疑問應聯繫 NOC Services Relations。
9. **Reusable Knowledge**：Knowledge Base 應紀錄版本與日期，避免舊規則被錯用。
10. **Abstraction**：`Operations`、`Governance`。
11. **Visual Evidence**：此章以文字聯絡資訊為主，功能是流程證據而非視覺示範。
