# ARTE Charte Graphique / Grafikrichtlinien Habillage/Senderdesign 2008

來源：`sources/arte_2008_compressed.pdf`
日期：2008
研究視角：Design Infrastructure Researcher

## Summary

ARTE 2008 法德雙語視覺識別規範，從開放、尊重、溫暖與好奇的品牌哲學，建立 logo、-4° 角度、cartouche/Podest、色彩、字體、版面、claim、範例、禁用與伺服器資產治理。把 ARTE 的人文價值轉為可操作的視覺語法，避免後續規範只剩任意的角度、色彩與形式。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。一致性與變化共存；規則必須強制維持視覺同質性，但保留多種變體。Value-to-grammar framework：humanist values -> formal constants -> flexible applications。

執行上，這份 guideline 透過 Philosophie、Composition Graphique Generale / Grafische Grundlagen、Three Fundamental Elements、Le Logo ARTE / Das ARTE-Logo 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。Design KB 應把價值詞彙映射到可觀察設計屬性，例如 openness = airy layout、respect = legibility、warmth = bright colours/human proximity。

## Philosophie

1. **Chapter Purpose**：把 ARTE 的人文價值轉為可操作的視覺語法，避免後續規範只剩任意的角度、色彩與形式。
2. **Definition**：本章定義 Ouverture、Respect、Chaleur、Curiosite / Offenheit、Respekt、Warme、Neugier 作為品牌態度，並提出 logo、-4° 角度、cartouche/Podest、強勢字體與傾斜水平帶。
3. **Relationship**：它是所有後續章節的設計意圖來源；logo、Podest、typography 與 layout 都是這些價值的形式化。
4. **Design Execution**：以開放留白、清晰可讀、明亮色彩、人像親近感與 playful curiosity 轉成視覺原則。
5. **Principles**：一致性與變化共存；規則必須強制維持視覺同質性，但保留多種變體。
6. **Frameworks**：Value-to-grammar framework：humanist values -> formal constants -> flexible applications。
7. **Patterns**：可複用模式是先寫品牌價值的視覺翻譯，再導入具體資產規則。
8. **Rules**：所有承載 ARTE brand 的媒介都需遵守基本圖形規則；logo 與 -4° angle 是不變常數；三大設計元素為彩色 cartouche、強勢彩色 typography、-4° horizontal band。
9. **Reusable Knowledge**：Design KB 應把價值詞彙映射到可觀察設計屬性，例如 openness = airy layout、respect = legibility、warmth = bright colours/human proximity。
10. **Abstraction**：Brand Strategy、Visual Language、Knowledge Architecture。
11. **Visual Evidence**：哲學章以雙語文字為主，後續頁面用三大元素示範價值如何具象化。

## Composition Graphique Generale / Grafische Grundlagen

1. **Chapter Purpose**：建立 ARTE 2008 識別的基本文法，讓所有印刷與外部溝通能用同一組構成元素產生變化。
2. **Definition**：定義三個 fundamental elements：彩色 cartouche/Podest、強勢彩色 typography、傾斜 -4° horizontal band。
3. **Relationship**：它承接哲學章，並引出 logo、cartouche、typography、layout 的專章。
4. **Design Execution**：cartouche 將 logo 推向觀者；typography 與 cartouche 色彩對話；-4° band 分配影像與白色空間。
5. **Principles**：每個元素都同時承擔功能與識別：焦點、訊息、空間結構。
6. **Frameworks**：Three-element grammar：anchor device + expressive type + structural plane。
7. **Patterns**：可複用於任何 brand system 的「少量核心構件生成多種版面」。
8. **Rules**：cartouche 是 logo 的主要載體；typography 應強勢、大面積、常用大寫並與 cartouche 色彩呼應；-4° band 必須協助圖像與文字空間劃分。
9. **Reusable Knowledge**：品牌 guideline 可把「核心構件」拆成 role、required conditions、allowed variations。
10. **Abstraction**：Visual Language、Layout System、Design Tokens。
11. **Visual Evidence**：章節用並列法德說明與圖形範例展示三元素如何形成基礎語法。

### Three Fundamental Elements

1. **Chapter Purpose**：把設計系統的複雜度壓縮成三個容易記憶且可組合的元件。
2. **Definition**：cartouche 是視覺焦點，typography 是聲量，-4° band 是空間骨架。
3. **Relationship**：這三者分別在後文被細化為 logo podium、type rules 與 page layout。
4. **Design Execution**：以透視、漸層、鮮色與傾斜角建立動感。
5. **Principles**：識別不是單一 logo，而是 logo 與版面物理關係的集合。
6. **Frameworks**：Element role matrix：attention, communication, organization。
7. **Patterns**：可複用為 design grammar minimum viable set。
8. **Rules**：任何完整 ARTE graphic composition 應至少能辨識出三元素之一，重要品牌溝通應使用三元素的組合。
9. **Reusable Knowledge**：可將設計系統元素以「必要/可選/例外」關係建模。
10. **Abstraction**：Visual Language、Layout System、AI-Checkable Rules。
11. **Visual Evidence**：頁面以大圖示例三元素，證明其構成角色。

## Le Logo ARTE / Das ARTE-Logo

1. **Chapter Purpose**：保護 ARTE logo 的 typographic originality 和 -4° 品牌常數，避免執行者重繪、改字距、改角度或改色。
2. **Definition**：logo 是原創字標，只能等比縮放；基本色為 orange，黑白印刷以 50% black 或 white，不用純黑。
3. **Relationship**：logo 是 cartouche、sublogos、claim 與 assets 的核心內容。
4. **Design Execution**：用 Pantone/CMYK/RGB/K 值、-4° 情境規則與特殊背景規則把 logo 治理具體化。
5. **Principles**：logo 的角度、字距與色彩是不可拆分的品牌語法；例外只為可讀性或製作條件服務。
6. **Frameworks**：Logo context framework：in design context -> -4° tilted; impossible/contextual listing -> upright allowed; orange-like background -> white logo。
7. **Patterns**：可複用為 angle-based identity system 與 background contrast exception。
8. **Rules**：logo 不可變形、重繪、改字距；基本 orange 為 Pantone 1505 / CMYK 0,65,100,0 / RGB 255,95,29；黑白印刷用 50% black 或 white，不用 black；設計情境中 logo 需 -4°，但 logo 列舉、共同製作、外部製作與 merch 可直立或傾斜。
9. **Reusable Knowledge**：AI 可檢查 logo angle、aspect ratio、colour token、letter spacing 與背景對比。
10. **Abstraction**：Visual Identity、Color System、AI-Checkable Rules。
11. **Visual Evidence**：logo 色票、文具範例、背景特殊情境與直立/傾斜範例提供判斷證據。

## Marque ARTE - Marques Derivees / Sublogos

1. **Chapter Purpose**：讓 ARTE 母品牌能擴展到 diffusion、marketing、merchandising、communication 與 programme titles，而不失去母品牌歸屬。
2. **Definition**：衍生品牌由 ARTE logo 加 wordmark 或 programme title 構成，通常使用 Helvetica Neue Heavy 與 capitals。
3. **Relationship**：它將 logo 規則延伸到 brand architecture，並與 cartouche positioning 連接。
4. **Design Execution**：提供直接相接或下方放置兩種關係，並允許 programme title 使用個別色彩或自由 signet，但需與 ARTE logo 保持視覺一致。
5. **Principles**：母品牌歸屬必須清晰，但節目品牌需有空間表現多樣性。
6. **Frameworks**：Sublogo architecture framework：masterbrand + descriptor, programme title with ARTE, programme title without ARTE, exception for magazine cover。
7. **Patterns**：可複用於 media brand 的節目/子品牌標誌治理。
8. **Rules**：衍生 wordmark 一般為 Helvetica Neue Heavy、capitals；設計情境中 sublogos 傾斜 -4°；含 ARTE 的節目 title 跟隨 ARTE logo，可搭配特定色碼；照片上 title 可用 white；無 ARTE 字樣的 programme/case logo 可自成一體也可與 ARTE logo 組合。
9. **Reusable Knowledge**：Brand architecture 應為子品牌定義「固定母品牌元件」與「可自由設計元件」。
10. **Abstraction**：Brand Architecture、Visual Identity、Governance。
11. **Visual Evidence**：多個 sublogo 排列和節目名稱範例提供母子品牌關係證據。

## Cartouche / ARTE-Logo-Podest

1. **Chapter Purpose**：把 ARTE logo 從平面標記轉為具有透視、深度和聚焦功能的主要識別裝置。
2. **Definition**：cartouche/Podest 是承載 ARTE logo 的彩色透視形體，可有四種 perspectives、不同大小與多種漸層色。
3. **Relationship**：它是哲學章「modernity/originality」與 composition 的最重要視覺元件，並支撐後續 layout、claim 與 examples。
4. **Design Execution**：以 logo 定位、-4° 傾斜、版面邊緣貼附、水平/垂直間距與純色漸層 palette 控制使用。
5. **Principles**：裝置需同時創造空間、吸引注意、維持品牌一致；自由變化必須在固定比例和位置內。
6. **Frameworks**：Podium device framework：carrier, perspective, gradient, edge placement, logo-based spacing, special-case removal。
7. **Patterns**：可複用為 brand container / logo stage system。
8. **Rules**：cartouche 與其中 logo 一律 -4°；cartouche 永遠位於版面邊緣；水平間距用 1x ARTE logo 長度，垂直間距用 2x ARTE logo 高度；可從任何邊進入版面且有上/下方向；主色需使用純色構成漸層；無法確保平滑漸層、單色印刷或無框可定位時，不使用 cartouche。
9. **Reusable Knowledge**：複雜 logo container 應有 spacing unit、edge anchoring、gradient tokens、production exceptions。
10. **Abstraction**：Visual Language、Layout System、Color System、AI-Checkable Rules。
11. **Visual Evidence**：位置圖、四方向/透視範例、色票與特殊情境頁構成 cartouche 規則證據。

### Position and Colour Palettes

1. **Chapter Purpose**：將 cartouche 從自由圖形變成可度量、可生產的設計 token。
2. **Definition**：位置以 ARTE logo 自身為測量單位，色彩以多組 CMYK 漸層命名。
3. **Relationship**：它服務所有印刷應用、typography 色彩與後續 server assets。
4. **Design Execution**：以 Tequila Sunrise、Purple、Azure、Emerald、Lime、Sunflower 等主 palette 與 Dawn、Night、Ocean、Grass、Fuchsia、Orange Juice 等延伸 palette 建立系統。
5. **Principles**：明亮、純粹、同質的漸層比隨機顏色更能形成 ARTE 世界。
6. **Frameworks**：Gradient token framework：name, start CMYK, end CMYK, logo association permission。
7. **Patterns**：可複用為 gradient design token schema。
8. **Rules**：Fuchsia 與 Orange Juice 通常不與 ARTE logo 結合，只作為構圖元素；typography 顏色可取自 cartouche 漸層兩端。
9. **Reusable Knowledge**：Design tokens 可不只記 hex，也記可否搭配 logo、主要/延伸用途與印刷限制。
10. **Abstraction**：Color System、Design Tokens、AI-Checkable Rules。
11. **Visual Evidence**：色票與 CMYK 值是明確規格證據。

### Special Cases

1. **Chapter Purpose**：明確指出製作條件不足時不應勉強使用 cartouche，避免壞品質比缺席更傷品牌。
2. **Definition**：特殊情境包括絲印等無法平滑漸層、單色印刷、無可對齊邊界的物件如 T-shirt。
3. **Relationship**：它保障 cartouche 系統在生產限制下不被破壞。
4. **Design Execution**：在例外中讓 logo 單獨使用，並要求設計一開始就考慮此限制。
5. **Principles**：例外不是自由，而是防止低品質復刻。
6. **Frameworks**：Production feasibility framework：gradient possible? monochrome? frame available?
7. **Patterns**：可複用於所有依賴特殊印刷效果的品牌資產。
8. **Rules**：若印刷無法保證 continuous gradient，禁止使用 cartouche；單色印刷不使用 cartouche；無版面邊界可定位時不使用 cartouche。
9. **Reusable Knowledge**：Guideline 應把 production constraints 納入設計決策樹。
10. **Abstraction**：Operations、Production Design、AI-Checkable Rules。
11. **Visual Evidence**：特殊情境頁以文字說明為主，依賴生產條件而非視覺範例。

## Typographie

1. **Chapter Purpose**：讓文字系統與 cartouche 的聲量和角度一致，並同時維持正文可讀性。
2. **Definition**：外部溝通使用 Helvetica Neue 各字重，內部與書信用 Arial；標題/副標可用 -4°、右對齊、彩色；正文使用大小寫、齊行、緊湊且不傾斜。
3. **Relationship**：typography 是三大基礎元素之一，與 cartouche 色彩和 layout band 直接互動。
4. **Design Execution**：以標題大寫、副標大小寫、色彩取自 cartouche、正文不過度使用多字重建立層級。
5. **Principles**：品牌聲量與可讀性需分層處理：headline 可強勢，copy text 必須清楚。
6. **Frameworks**：Type hierarchy framework：headline/subheadline expressive, copy compact/readable, internal correspondence fallback。
7. **Patterns**：可複用於以同一字族建立外部/內部、標題/正文的字體治理。
8. **Rules**：外部 print communication 使用 Helvetica Neue；correspondence/internal notes 使用 Arial；headline/subheadline 顏色匹配 cartouche，可 -4°，右對齊；headline 用 capitals，subheadline 用大小寫；正文 always upper/lowercase、justified、prefer compact and non-tilted；避免過長行與過多字體。
9. **Reusable Knowledge**：Typography 規範應標記每種文字角色的 angle、alignment、case、colour source、readability constraints。
10. **Abstraction**：Typography System、Content Design、AI-Checkable Rules。
11. **Visual Evidence**：字體樣張、標題/正文範例、色彩頁與版面示例提供規則證據。

## Mise en Page / Layout

1. **Chapter Purpose**：把 logo cartouche、typography 和 -4° band 組成完整頁面架構，避免元素各自存在。
2. **Definition**：layout 章定義 cartouche composition、typography positioning、horizontal inclined band 的角色與變體。
3. **Relationship**：它是前面元素專章的整合層，後續 cartouche utilizations 與 examples 都在此基礎上運作。
4. **Design Execution**：提供 aligned-to-cartouche 或 freely positioned type，並用 -4° band 分隔 image/text/white space。
5. **Principles**：版面識別來自空間比例與傾斜結構，不只是資產貼上。
6. **Frameworks**：Page composition framework：carrier, type position, inclined plane, image/white-space proportion。
7. **Patterns**：可複用為 visual layout grammar，讓同一品牌可生成多種出版品。
8. **Rules**：傾斜 band 可放不同高度以支持文字與影像；其功能是識別、組織圖文區、強化 typography 位置；band 色彩可取自 image 或 cartouche，並可 opaque、transparent、solid 或 gradient。
9. **Reusable Knowledge**：AI 可檢測版面中是否存在指定角度平面、是否建立圖文區分、色彩是否與指定來源相容。
10. **Abstraction**：Layout System、Visual Language、AI-Checkable Rules。
11. **Visual Evidence**：layout 範例頁用圖像/白空間/色帶關係示範構圖邏輯。

## Cartouche - Utilisations / Podest Anwendungen

1. **Chapter Purpose**：擴展 cartouche 的角色，使它不只能放 logo，也能作為資訊載體或自由構圖元素，但仍避免同頁過度使用。
2. **Definition**：cartouche 可作為 logo+information、information carrier、free graphic element、stationery background。
3. **Relationship**：它將 cartouche 從核心 logo stage 擴展為版面語法，但需要 don’ts 章防止濫用。
4. **Design Execution**：以 ARTE logo 的 t 作為內部文字間距，並限制資訊不要太長；內頁或封底可用無 logo cartouche 作資訊背景。
5. **Principles**：核心資產可擴展用途，但必須有互斥規則，避免同一頁多個焦點。
6. **Frameworks**：Device role framework：identity carrier, information carrier, background, free fragment。
7. **Patterns**：可複用於品牌容器元件的多用途治理。
8. **Rules**：logo+information 情境文字不可過長；以 logo 的 t 測量元素間距；資訊載體只適用於內頁/封底且 logo 不在 cartouche；每頁不超過一個 cartouche；背景使用不得與另一 cartouche 同頁混用；背景/資訊兩種自由用法不可合併。
9. **Reusable Knowledge**：Guideline 應為 reusable visual device 建立「互斥條件」和「頁面數量上限」。
10. **Abstraction**：Layout System、Visual Language、AI-Checkable Rules。
11. **Visual Evidence**：本章以多種內頁、地址、網址、文具和背景示例展示用途界線。

## La Signature / Der Claim

1. **Chapter Purpose**：控制品牌 claim 與 cartouche 的關係，確保口號具有足夠可讀性與視覺階層。
2. **Definition**：claim 可放在 cartouche 中，或放在 cartouche 右側；一律大寫並使用 ARTE orange。
3. **Relationship**：它將品牌語氣加入 logo/cartouche 系統，特別用於 image campaign。
4. **Design Execution**：compact 版把 claim 置於 cartouche 中 logo 下方；priority 版將 claim 放右側，必要時用 -4° band 強化可讀性。
5. **Principles**：口號位置需依版面緊湊度與背景複雜度決定。
6. **Frameworks**：Claim placement framework：inside cartouche for compact use, beside cartouche for claim priority, add inclined band for busy backgrounds。
7. **Patterns**：可複用為 slogan lockup variants。
8. **Rules**：claim 使用 capitals 和 ARTE orange；在 cartouche 中永遠位於 ARTE logo 下方；在右側配置時放在 cartouche 右邊；背景過於 movemented/繁忙時移除/加入傾斜 band 以提升可讀性。
9. **Reusable Knowledge**：品牌口號規則應依背景複雜度與主次階層設置不同 lockup。
10. **Abstraction**：Messaging、Visual Identity、Layout System。
11. **Visual Evidence**：image campaign 範例以 claim inside/beside cartouche 展示不同階層。

## Exemples / Beispiele

1. **Chapter Purpose**：把前面規則轉為實際出版品示範，幫助使用者理解三元素如何在不同版型中組合。
2. **Definition**：範例涵蓋 institutional brochures、activity reports、press dossiers、press invitations 與內頁/封面。
3. **Relationship**：它是 rules-to-application 的橋，驗證 guideline 不只適合單一英雄版面。
4. **Design Execution**：用彩色 intense Podest 承載 headlines，或用 Pearl 灰背景形成低調企業文具；press dossier 可有或無 -4° type tilt。
5. **Principles**：範例應展現系統彈性：assertive vs understated、cover vs interior、title vs copy。
6. **Frameworks**：Example taxonomy：institutional brochure, press dossier title, press dossier interior, press invitation。
7. **Patterns**：可複用為 application gallery，將每個範例標記為 which rules are demonstrated。
8. **Rules**：press dossier 標題頁與孤立 headline 建議使用 -4° typography；內頁可依嚴格 grid 決定是否傾斜；三大元素應在範例中保持可辨識。
9. **Reusable Knowledge**：範例不應只展示美感，應標註「展示哪些規則」與「何時採用」。
10. **Abstraction**：Examples、Layout System、Governance。
11. **Visual Evidence**：本章視覺證據最多，包含大量 brochure/dossier/invitation mockups；但其中許多正文是假文，分析時只抽取版面與規則，不把假文當內容知識。

## A Eviter / Don’ts

1. **Chapter Purpose**：將常見錯誤集中成負面規則庫，讓審核者能快速判定不符合規定輸出。
2. **Definition**：本章列出 logo 與 cartouche 的禁止事項，包括裁切、改色、框線、透明、輪廓、比例、角度、重複與漸層錯誤。
3. **Relationship**：它是前面規則的反向檢查清單，也是最容易轉成 AI-checkable criteria 的章節。
4. **Design Execution**：用短句和錯誤圖把每種違規獨立呈現，降低判斷成本。
5. **Principles**：負面規範不是附錄，而是品牌治理的執行介面。
6. **Frameworks**：Error taxonomy framework：logo form, colour, angle, transparency, outline, clearspace, cartouche geometry, gradient, duplication。
7. **Patterns**：可複用為 do-not rule dataset，用於視覺 QA 或訓練 AI 審稿。
8. **Rules**：logo 不得裁切、改色、加框、改構成、非 -4° 傾斜、透明、outline、改比例、加輪廓、用黑色、被外來形狀干擾留白；cartouche 不得變形、改比例、任意放置、非 -4°、階梯狀漸層、自由放置 logo、過度裁切、窄邊進入畫面、同頁出現兩次、透明、單色、漸層轉白/黑/灰或互補色。
9. **Reusable Knowledge**：Don’ts 可直接結構化為 validation rules：detect crop, aspect ratio, colour, angle, opacity, duplicates, gradient endpoints。
10. **Abstraction**：AI-Checkable Rules、Governance、Visual Identity。
11. **Visual Evidence**：每條 don’t 搭配錯誤圖，是建立視覺錯誤分類資料集的最佳來源。

### Logo and Cartouche Don’ts

1. **Chapter Purpose**：把兩個核心資產的錯誤型態拆開，方便審核工具或人工審核逐項檢查。
2. **Definition**：logo don’ts 側重字標完整性，cartouche don’ts 側重形體、漸層、位置與同頁數量。
3. **Relationship**：這是 logo 與 cartouche 規則的 enforcement layer。
4. **Design Execution**：以雙語短句和錯誤示意形成判準。
5. **Principles**：核心資產的不可變項比建議項更適合自動化檢查。
6. **Frameworks**：Validation checklist framework。
7. **Patterns**：適合轉成 visual linter。
8. **Rules**：同上，並可拆為 `angle == -4deg`、`opacity == 100%`、`logo count per page <= 1 when in cartouche context`、`cartouche count per page <= 1`、`aspect ratio unchanged`、`no black logo`、`no monochrome podium` 等條件。
9. **Reusable Knowledge**：每個 don’t 都應有 rule id、failure condition、allowed exception、visual example。
10. **Abstraction**：AI-Checkable Rules、Quality Assurance。
11. **Visual Evidence**：錯誤示例頁是直接訓練資料。

## Elements sur Serveur / Elemente auf Server

1. **Chapter Purpose**：把 guideline 與實際 asset library 連接，讓設計者使用正確檔案而非自行製作 cartouche。
2. **Definition**：本章列出伺服器上的 cartouche assets、四種 perspective、不同 gradient、Illustrator EPS/JPG 版本與製作注意事項。
3. **Relationship**：它是 Cartouche 規則的營運層，對應前面所有 palette 和 perspective。
4. **Design Execution**：以檔名清單與 Illustrator blend setting 注意事項，防止放大時產生階梯色。
5. **Principles**：複雜視覺元件應由中心提供，地方只選擇與置入，不重新建構。
6. **Frameworks**：Asset distribution framework：server location, file formats, variants, editing notes, production risks。
7. **Patterns**：可複用為 design asset registry。
8. **Rules**：cartouche assets 應使用提供的 Illustrator .eps 或 .jpg；放大 cartouche 時必須檢查 Illustrator blend settings，避免印刷色階；可在 Illustrator 移除 ARTE logo 以單獨使用 Podest。
9. **Reusable Knowledge**：Design KB 應保存 asset filename pattern、variant dimensions、production warnings 與「不要重建」規則。來源 PDF 內含舊式伺服器連線資訊與帳密，知識庫不應重刊敏感連線細節。
10. **Abstraction**：Operations、Asset Management、Production Design、Governance。
11. **Visual Evidence**：檔名清單與 asset previews 提供資產變體證據；文字說明提供生產風險證據。
