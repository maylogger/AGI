# Calvin Klein Logo & Graphic Guidelines

來源：`sources/CalvinKlein_Logo&GraphicGuidelines_2017.pdf`
版本：v01
日期：2017-06-16
研究視角：Design Infrastructure Researcher

## Summary

Calvin Klein 2017 rebranding 的 logo、品牌架構與 Klein 字體技術規範，重點在跨產品線標誌治理、比例、禁用規則與資產交付。建立 2017 rebranding 的操作邊界，先解釋為何 Calvin Klein 從個人姓名式標誌轉向更企業化、系統化的大寫識別，避免後續 logo 規範看起來只是任意的檔案清單。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。核心原則是 evolution over rupture、global consistency with tier specificity、logo as business architecture，而不是單一美學偏好。主要框架是「brand ethos -> business line architecture -> logo usage chart -> asset governance」。

執行上，這份 guideline 透過 Introduction、How To Use These Guidelines、Brand Ethos and Historical Continuum、Logo Architecture Chart 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。大型品牌可先建立 identity rationale 與 architecture map，讓後續所有視覺規範都有決策來源。

## Introduction

1. **Chapter Purpose**：建立 2017 rebranding 的操作邊界，先解釋為何 Calvin Klein 從個人姓名式標誌轉向更企業化、系統化的大寫識別，避免後續 logo 規範看起來只是任意的檔案清單。
2. **Definition**：本章定義指南用途、品牌演化論述、產品線命名與 logo architecture；它不負責提供完整 trim/design standards，原文明確指出會另由全球設計團隊發布。
3. **Relationship**：Introduction 是 Section B 與 C 的語義前置層，把「歷史連續性」「業務線差異」「產品層級」轉成後續 logo 與字體規則的決策依據。
4. **Design Execution**：作者用 historical comparison、business line past/present chart 與 logo architecture chart，把抽象品牌轉型轉成可查詢的產品線矩陣。
5. **Principles**：核心原則是 evolution over rupture、global consistency with tier specificity、logo as business architecture，而不是單一美學偏好。
6. **Frameworks**：主要框架是「brand ethos -> business line architecture -> logo usage chart -> asset governance」。
7. **Patterns**：先給策略敘事，再給架構圖，最後進入逐 logo 規格，是大型品牌更新常見的 adoption pattern。
8. **Rules**：AI-checkable criteria：文件若引用 2017 rebranding，必須能指出 logo 所屬 business line；任何 logo 使用判斷都應能對應到 usage chart；若用途屬 trim/design standards，不能只依此文件判定。
9. **Reusable Knowledge**：大型品牌可先建立 identity rationale 與 architecture map，讓後續所有視覺規範都有決策來源。
10. **Abstraction**：`Brand Strategy`、`Knowledge Architecture`、`Visual Identity`。
11. **Visual Evidence**：本章大量依賴 logotype 歷史對照、past/present 表格與 logo architecture chart；視覺證據承擔「演化證明」與「治理地圖」功能。

### How To Use These Guidelines

1. **Chapter Purpose**：降低實作端誤用風險，讓內部、licensees、vendors 知道這不是品牌故事書，而是 rebranding implementation package。
2. **Definition**：定義交付物包含 logo artwork、fonts/assets、usage、restrictions、violations、clear space、positioning、scale、file names。
3. **Relationship**：它把文件權限與資產取得流程接到 PVH Digital Asset Library 與 Global Marketing Group。
4. **Design Execution**：用簡短段落列出可操作範圍，避免使用者把指南延伸到尚未發布的 trim 或 design standards。
5. **Principles**：規範要跟資產庫綁定；正確使用不是重畫，而是取用授權 master file。
6. **Frameworks**：`scope -> asset source -> distribution request -> separate standards`。
7. **Patterns**：把 guideline 當成 production routing document，直接告訴使用者去哪裡拿檔案、何時需另請求。
8. **Rules**：AI-checkable criteria：logo/font 應引用官方 asset library；對外 creative partner/vendor distribution 需經 Global Marketing Group；不得把本文件用作 trim/design standards 的唯一依據。
9. **Reusable Knowledge**：品牌規範應包含「適用範圍」與「資產取得路徑」，否則 rules 無法落地。
10. **Abstraction**：`Operations`、`Governance`、`Legal and Trademark`。
11. **Visual Evidence**：此小節文字主導，視覺證據較少；其功能是流程說明而非示範。

### Brand Ethos and Historical Continuum

1. **Chapter Purpose**：為 uppercase logo、Klein font 與保留部分 lower/uppercase underwear heritage 提供設計理由。
2. **Definition**：定義 2017 識別是對 Calvin Klein 1975 與 1992-2016 logo 的演化，而不是斷裂式換標。
3. **Relationship**：此小節把後續 master logo、underwear logo、Klein font 的差異化規則連到品牌歷史。
4. **Design Execution**：透過 Peter Saville 的文字論述，把 capitalization 解釋為從個人到 entity 的視角轉換。
5. **Principles**：品牌更新可以用「形式微調 + 語義重定位」完成；歷史熟悉感是全球品牌 scale 的資產。
6. **Frameworks**：`heritage mark -> semantic change -> system font -> channel differentiation`。
7. **Patterns**：用字母大小寫承載品牌親密度與企業性，是可複用的 typographic positioning pattern。
8. **Rules**：AI-checkable criteria：master brand 使用 uppercase；underwear 部分可保留 upper/lowercase 以維持 intimate signature；Klein font 用於建立各 brand channel 的 coherent relationships。
9. **Reusable Knowledge**：品牌字體與 logo 大小寫可被視為 strategy tokens，而非純造型。
10. **Abstraction**：`Brand Strategy`、`Typography System`、`Visual Identity`。
11. **Visual Evidence**：歷史 logotype 對照是主要證據，功能是比較與 lineage proof。

### Logo Architecture Chart

1. **Chapter Purpose**：處理 Calvin Klein 多業務線、多價位層、多通路的 logo 選擇複雜度。
2. **Definition**：定義各 business line 與 logo family 的對應關係，包含 Master Brand、By Appointment、205、Contemporary、Jeans、Underwear、Swimwear、Performance、Golf、Home、Watches & Jewelry、Outlet、Fragrance。
3. **Relationship**：它是 Section B 每個 logo chapter 的索引骨架。
4. **Design Execution**：用矩陣式架構圖把品牌層級、產品線與可用 logo 一次展開。
5. **Principles**：越大的品牌組合越需要 logo architecture，而不是讓每個團隊自行判斷。
6. **Frameworks**：`business line -> product tier -> channel/application -> approved mark`。
7. **Patterns**：矩陣式 architecture chart 可作為 AI 或資料庫判斷 logo 選用的上游 schema。
8. **Rules**：AI-checkable criteria：任一應用需先標記 business line；若 business line 在 chart 中未允許某 logo，不能使用；同一 category 若有 premium/mainline，必須先判斷 tier。
9. **Reusable Knowledge**：logo architecture 是品牌治理的 decision table，可轉成 rule engine。
10. **Abstraction**：`Knowledge Architecture`、`AI-Checkable Rules`。
11. **Visual Evidence**：chart 是核心視覺證據，承擔分類、決策與導覽功能。

## Logo Usage Guidelines

1. **Chapter Purpose**：把 architecture map 轉成每一個 logo 在實際生產中的可用場景、尺寸、清距、違規與檔案規格。
2. **Definition**：本章定義 logo usage chart、clear space、scale & positioning、violations、artwork/assets；它不重新定義品牌策略，而是執行策略。
3. **Relationship**：承接 Introduction 的產品線架構，並與 Section C 的 Klein font system 共用「官方檔案不可重畫」的治理邏輯。
4. **Design Execution**：每個 logo 幾乎使用同一頁型模板，讓使用者在 166 頁文件中仍能用固定順序查找。
5. **Principles**：一致性來自 repeatable specification pattern；差異化只發生在 logo 角色、tier 與 channel。
6. **Frameworks**：`logo introduction -> usage chart -> clear space -> scale/positioning -> violations -> asset list`。
7. **Patterns**：這是一個 logo governance module，可直接複用到任何多品牌、多產品線系統。
8. **Rules**：AI-checkable criteria：logo 周圍不得有物件進入 minimum/preferred clear space；logo 不得隨機放置，只能 centered 或 ranged；不得改字體、改粗細、拆分元件、用 Klein font 重建 logo；小於 14pt cap height 時使用 small version；避免低於 5pt，preferred minimum 6pt。
9. **Reusable Knowledge**：把重複規格變成模板，能同時支援人類讀者與機器驗證。
10. **Abstraction**：`Visual Identity`、`Governance`、`AI-Checkable Rules`。
11. **Visual Evidence**：本章大量使用 logo 示意、usage chart、clear-space 圖、錯誤案例與資產檔名截圖；功能是示範、比較、警示與規格證明。

### Master Brand and Corporate Logo

1. **Chapter Purpose**：定義 Calvin Klein 的 primary corporate mark，避免 master brand 被 category logo 取代。
2. **Definition**：適用於 corporate communications、multi-brand lifestyle stores，以及 fragrance、home、watches + jewelry、eyewear 等場景。
3. **Relationship**：作為其他 logo family 的基準，後續許多 category 在沒有專屬語境時回到 master brand。
4. **Design Execution**：以 usage chart 指定場景，用 clear-space 與 asset pages 保持可讀性與生產一致。
5. **Principles**：corporate mark 應保留最高一致性；其用途要由品牌層級而非設計師偏好決定。
6. **Frameworks**：`corporate context -> master mark -> standard/small asset -> production format`。
7. **Patterns**：primary mark module 可作為所有子品牌 logo 的 fallback 與 reference。
8. **Rules**：AI-checkable criteria：使用 master logo 時 cap height 小於 14pt 必須選 small version；不得 white-on-black 除非指定；不得把字距、粗細或格式改變；輸出應使用 EPS/PNG/JPEG 官方檔。
9. **Reusable Knowledge**：主品牌 logo 的規範應同時含「何時用」與「如何用」。
10. **Abstraction**：`Visual Identity`、`Legal and Trademark`。
11. **Visual Evidence**：usage chart、clear-space 圖、logo violation 對照與 asset file list 承擔示範與驗證功能。

### Business Line Logos

1. **Chapter Purpose**：讓 By Appointment、205、CK Calvin Klein、Better Sportswear 等業務線在同一母品牌下保有差異化。
2. **Definition**：定義不同 business line 的 logo 角色，例如 205 代表 designer label，CK Calvin Klein 服務 contemporary apparel，Better Sportswear 回到 1975 原始標誌語彙。
3. **Relationship**：這些 logo 是 master brand 與 category logos 之間的中層品牌架構。
4. **Design Execution**：以每個 business line 的 introduction 解釋語義，再以 usage chart 對應 labels、advertising、packaging、POS、signage 等。
5. **Principles**：logo 不只是名稱，而是產品定位、通路層級與市場區域的壓縮符號。
6. **Frameworks**：`business identity -> usage applications -> production constraints`。
7. **Patterns**：用相同規格頁型管理不同定位，降低品牌分歧。
8. **Rules**：AI-checkable criteria：CK Calvin Klein horizontal logo 僅用於 extreme horizontal applications；205 stacked/horizontal 需依應用空間選擇；所有 business line logo 均不可重畫、拉伸、改色、拆分元件。
9. **Reusable Knowledge**：中層品牌可用「角色說明 + 應用矩陣 + 禁用規則」建立可治理的 family system。
10. **Abstraction**：`Brand Architecture`、`Visual Identity`、`AI-Checkable Rules`。
11. **Visual Evidence**：各 logo overview 與 within architecture 圖示把 logo 放回品牌矩陣，功能是定位證據。

### Category and Tier Logos

1. **Chapter Purpose**：處理 Jeans、Underwear、Swimwear、Performance 這些 category 內部的 premium/mainline/master/off-price tier 差異。
2. **Definition**：定義同一品類中不同價格、通路、地區或產品層級應使用的 logo。
3. **Relationship**：比 business line logos 更接近商品與通路管理，直接影響 labels、waistbands、hangtags、packaging、advertising、POS。
4. **Design Execution**：以 logo overview 先分 tier，再用 chart 分配到 material touchpoints。
5. **Principles**：品牌一致性與商業區隔必須同時被編碼；高階線、主線與折扣線不能共用模糊標誌。
6. **Frameworks**：`category -> tier -> market/channel -> touchpoint -> logo asset`。
7. **Patterns**：tiered identity matrix 可複用於 fashion、retail、hospitality 與多產品 SaaS。
8. **Rules**：AI-checkable criteria：若 swimwear/performance premium 與 mainline 混合，signage 或 POS 預設 premium logo 並需向 Store Design/Planning 或 Global Marketing 確認；underwear premium 使用 205 logo，mass programs 使用 Master Underwear，off-price/outlet 使用 lower/uppercase underwear logo。
9. **Reusable Knowledge**：在 category 系統中，tier 是 logo 選擇的重要屬性，應成為資料欄位。
10. **Abstraction**：`Visual Identity`、`Product Language`、`Knowledge Architecture`。
11. **Visual Evidence**：tier overview、usage chart、waistband/label/POS 欄位是關鍵視覺證據，功能是分流與治理。

### Specialized Category Logos

1. **Chapter Purpose**：管理 Golf、Home、Watches & Jewelry、Fragrance、Outlet 等特殊事業線，避免每個品類自行發展孤立 logo。
2. **Definition**：定義哪些品類直接使用 master brand，哪些 premium line 需用 205 stacked logo，Outlet 則有自身 consumer marketing logo。
3. **Relationship**：這是 logo architecture 的例外與回歸邏輯：產品能自明時用 master brand，需要高階線索時借 205。
4. **Design Execution**：透過 within architecture 標示品類位置，再重複清距、尺寸、違規與資產規格。
5. **Principles**：不是每個品類都需要專屬標誌；有時一致的 corporate mark 比增加子標誌更有效。
6. **Frameworks**：`category self-explanation -> master brand use`；`premium association -> 205 use`；`outlet consumer marketing -> outlet logo`。
7. **Patterns**：以「產品是否能說明自身」決定是否需要 category logo，是降低 logo proliferation 的模式。
8. **Rules**：AI-checkable criteria：Home 使用 Master Brand Logo；Premium Watches & Jewelry 與 Premium Fragrance 使用 205 Stacked Logo；Fragrance 除 205 line 外使用 Master Brand Logo；Outlet logo 目前不應用於 store signage、trim 或 packaging。
9. **Reusable Knowledge**：特殊品類規則應寫成 exception table，而不是散落在敘述裡。
10. **Abstraction**：`Brand Architecture`、`Governance`。
11. **Visual Evidence**：within architecture diagram 與 asset pages 用來證明每個特殊品類在系統中的位置。

### Reusable Logo Rules Pattern

1. **Chapter Purpose**：本文件真正的 infrastructure 價值在於把不同 logo 全部套入一致的規格模式。
2. **Definition**：定義可重複的 logo rule template：用途、清距、尺度、定位、禁用、資產。
3. **Relationship**：此模式橫跨 Section B 全部 logo，是後續可抽象成 AGI 模組的核心。
4. **Design Execution**：使用固定頁面順序與幾乎一致的文字規則，降低讀者學習成本。
5. **Principles**：好的 guideline 不只是列規則，而是讓規則結構可預期。
6. **Frameworks**：`allowed use -> spatial integrity -> size integrity -> form integrity -> source file integrity`。
7. **Patterns**：每個 logo 都有「可用矩陣 + 清距公式 + 禁用案例 + asset manifest」。
8. **Rules**：AI-checkable criteria：任何 logo 規範若缺少 usage chart、clear space、scale/positioning、violations 或 asset list，視為治理不完整；clear space 應以 cap height C 的倍數表示；asset 應列出 format、color mode、background 與 file name。
9. **Reusable Knowledge**：這可直接變成 AGI 的 `Logo Governance Template`。
10. **Abstraction**：`AI-Checkable Rules`、`Governance`、`Knowledge Architecture`。
11. **Visual Evidence**：重複頁型本身就是治理證據；使用者可透過相同版面辨識規範類型。

## Klein Font Family Guidelines

1. **Chapter Purpose**：把 2017 rebranding 從 logo 延伸到 proprietary type system，讓品牌語氣在 print、screen、product 上一致。
2. **Definition**：定義 Klein sans serif 的歷史參照、字重、樣式、檔案與使用方式。
3. **Relationship**：它延續 Introduction 的 historical continuum，也支援 Section B 中不得用 Klein font 重建 logo 的界線。
4. **Design Execution**：先說明字體角色，再列出 Roman、Italic、Condensed assets，最後給 text/display/small text 使用規格。
5. **Principles**：字體是品牌聲音與 channel coherence 的核心，不只是排版選項。
6. **Frameworks**：`typeface concept -> asset inventory -> usage modes -> glyph coverage`。
7. **Patterns**：專屬字體系統需同時管理設計理念、檔案格式、使用情境與字元支援。
8. **Rules**：AI-checkable criteria：Klein 45 Light 可用於 10pt 以上 body copy；Klein 55 Regular 是 body default；Klein 65/75 用於 emphasis；display tracking 為 -20 或 condensed display -10；display 不使用 Klein 75 Bold；italic sparingly；condensed 用於空間受限場景。
9. **Reusable Knowledge**：typography guideline 應將字重轉成 usage roles，而非只展示 specimen。
10. **Abstraction**：`Typography System`、`Design Tokens`。
11. **Visual Evidence**：字體 specimen、OTF 檔名與 glyph chart 是主要證據，功能是資產盤點與可用性驗證。

### Font Assets and Files

1. **Chapter Purpose**：讓品牌字體能被正確安裝、交付與引用。
2. **Definition**：列出 Klein Roman、Italic、Condensed styles 與 OTF 檔名。
3. **Relationship**：是 usage rules 的資產前置條件。
4. **Design Execution**：以字母、數字、符號 specimen 搭配檔名。
5. **Principles**：字體治理必須管到檔案層級。
6. **Frameworks**：`style name -> specimen -> file format -> file name`。
7. **Patterns**：font asset manifest 可作為設計系統 token source。
8. **Rules**：AI-checkable criteria：交付檔須為 OTF；Roman 至少含 45/55/65/75；Italic 至少含 46/56/66/76；Condensed 至少含 57/67。
9. **Reusable Knowledge**：字體資產表應能支援版本管理與授權檢查。
10. **Abstraction**：`Typography System`、`Operations`。
11. **Visual Evidence**：specimen 與檔名清單承擔驗證功能。

### Font Usage Guidelines

1. **Chapter Purpose**：避免字體被任意使用，將字重與場景建立對應。
2. **Definition**：定義 text、display、small text 的 leading、tracking、字重與 emphasis 使用。
3. **Relationship**：從 asset inventory 進一步轉成排版決策規則。
4. **Design Execution**：用設定清單直接列出每種場景的可用字重。
5. **Principles**：排版一致性來自場景化規則，而非單一首選字體。
6. **Frameworks**：`content size -> reading mode -> weight/style -> tracking`。
7. **Patterns**：以 reading context 管理 typography。
8. **Rules**：AI-checkable criteria：body above 10pt 可用 Klein 45 Light；body default 為 55 Regular；small body below 10pt default 55 Regular；display tracking -20；condensed display tracking -10；italic 僅 sparingly。
9. **Reusable Knowledge**：字體規則可以轉成 lint-like typography checks。
10. **Abstraction**：`Typography System`、`AI-Checkable Rules`。
11. **Visual Evidence**：本節以文字規格為主，視覺證據是排版設定本身。

### Glyph Chart

1. **Chapter Purpose**：確認品牌字體支援必要字元，降低跨媒體文字缺字風險。
2. **Definition**：列出完整 character set。
3. **Relationship**：補足 font assets 的可用性檢查。
4. **Design Execution**：以 glyph chart 展示字元覆蓋。
5. **Principles**：品牌字體的治理包含可讀性與 character coverage。
6. **Frameworks**：`font weight -> supported characters -> application readiness`。
7. **Patterns**：glyph coverage 是 typography QA 的一部分。
8. **Rules**：AI-checkable criteria：若文本包含未列字元，應改用 fallback 或更新字體資產；品牌輸出前需確認符號與數字在目標字重中存在。
9. **Reusable Knowledge**：字元覆蓋可納入 design token QA。
10. **Abstraction**：`Typography System`、`Accessibility`。
11. **Visual Evidence**：glyph chart 承擔覆蓋驗證功能。
