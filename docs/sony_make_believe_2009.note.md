# Sony Group Brand Message make.believe Concept and Usage Guidelines

來源：`sources/sony_make_believe_2009.pdf`
版本：Version 1
日期：2009-07-31
研究視角：Design Infrastructure Researcher

## Summary

Sony make.believe guideline 將集團品牌訊息定義為統一 electronics、entertainment、design、engineering 與 consumer communication 的品牌語法，並規範 composite logo、print/web/motion 使用與治理。在跨電子、娛樂、設計與工程的 Sony 集團內建立單一品牌訊息。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。品牌訊息應能整合組織內部差異，而不是新增一個口號。brand message framework：imagination → belief → making real → electronics + entertainment unity。

執行上，這份 guideline 透過 The Meaning of make.believe、Application of make.believe、Usage of the Composite Logo、Using the Composite Logo in Print Media and Websites 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。集團品牌訊息可抽象為跨事業 alignment module。

## The Meaning of make.believe

1. **Chapter Purpose**：在跨電子、娛樂、設計與工程的 Sony 集團內建立單一品牌訊息。
2. **Definition**：定義 make.believe 為能把想像變成真實的集團品牌訊息與 mantra。
3. **Relationship**：它是後續 composite logo 與 motion logo 的語義前提；沒有此章，logo 使用只剩機械規格。
4. **Design Execution**：用引言、集團能力與 mantra 把 technology/content duality 轉成品牌承諾。
5. **Principles**：品牌訊息應能整合組織內部差異，而不是新增一個口號。
6. **Frameworks**：brand message framework：imagination → belief → making real → electronics + entertainment unity。
7. **Patterns**：以 mantra 作為跨事業語義錨點，是集團品牌治理模式。
8. **Rules**：AI 可檢查：make.believe 相關材料應指向 imagine/make real、technology/content integration 或 Sony unity；不得把它當一般裝飾文字或孤立 campaign slogan。
9. **Reusable Knowledge**：集團品牌訊息可抽象為跨事業 alignment module。
10. **Abstraction**：`Brand Strategy`、`Messaging`。
11. **Visual Evidence**：概念頁、mantra 頁與 designers/engineers/electronics/entertainment unity 圖是語義證據。

## Application of make.believe

1. **Chapter Purpose**：界定誰應使用、哪些場景必須使用，避免品牌訊息在組織中任意擴散或缺席。
2. **Definition**：定義 guideline 目的、使用者、permitted usage 與 mandatory/recommended/not permitted application matrix。
3. **Relationship**：承接 meaning，進入 governance；後續 logo 規格只適用於被允許的場景。
4. **Design Execution**：以表格列 corporate communication、ATL/BTL、web、packaging、startup screen、hardware/software 等使用狀態。
5. **Principles**：策略訊息需要分層 deployment，不是所有資產都同等套用。
6. **Frameworks**：usage matrix framework：business area → communication type → mandatory/recommended/not permitted。
7. **Patterns**：把 brand message rollout 寫成應用矩陣，是大型組織可複用模式。
8. **Rules**：AI 可檢查：若屬 Electronics & Networked Services 或 Entertainment 的 marketing/corporate communication，composite logo 應為 mandatory；content products/packaging/start-up screen 依表格為 recommended；hardware/application software 不得任意使用。
9. **Reusable Knowledge**：Design knowledge base 應保存 use-permission matrix，而不只保存資產規格。
10. **Abstraction**：`Governance`、`Operations`、`Compliance`。
11. **Visual Evidence**：permitted usage chart 是最主要的治理證據。

## Usage of the Composite Logo

1. **Chapter Purpose**：確保 Sony logotype 與 make.believe 組合在尺寸、空間、顏色與語音上固定。
2. **Definition**：定義 proper form/size、isolation zone、color/background、incorrect usage、trademark notice 與 pronunciation。
3. **Relationship**：它將 application matrix 中允許的情境轉成具體資產規格。
4. **Design Execution**：用 isolation zone、色彩背景與大量 incorrect usage 對照，保護 composite logo 不被變形。
5. **Principles**：跨組合標誌比單一 logo 更容易被錯誤改造，因此需要更細 don’ts。
6. **Frameworks**：composite mark framework：form → size → isolation zone → background → misuse → legal/pronunciation。
7. **Patterns**：把 pronunciation 納入 logo guideline，顯示品牌訊息同時是視覺與語言資產。
8. **Rules**：AI 可檢查：不得 stretch、outline、3D/drop shadow、arc、angle、condense、italicize、改字距、改字體、垂直排字、與其他 logo/元素合成或改變 Sony 與 make.believe 間距。
9. **Reusable Knowledge**：Composite logo 規範可抽象為 multi-mark lockup integrity check。
10. **Abstraction**：`Visual Identity`、`Legal and Trademark`、`Voice and Tone`。
11. **Visual Evidence**：proper form、isolation zone、incorrect usage pages 是視覺證據。

## Using the Composite Logo in Print Media and Websites

1. **Chapter Purpose**：把 composite logo 套入 print 與 web 的固定位置、比例與 header 系統。
2. **Definition**：定義 print top-left placement、尺寸公式、website header left placement、black header、utility area 與禁用情境。
3. **Relationship**：它是 logo 規格的媒介化，並把靜態標誌轉成網站 header governance。
4. **Design Execution**：print 以 page corner 與高度公式控管；web 以 65 px header、黑底白 logo、utility area 限制控管。
5. **Principles**：媒介差異需要不同規格，但識別位置與背景應保持穩定。
6. **Frameworks**：media application framework：print formula → website header → utility area → misuse rules。
7. **Patterns**：把 header 內 logo zone 與 utility zone 分離，是可複用 website brand pattern。
8. **Rules**：AI 可檢查：print composite logo 置於 top left；website header 左側顯示；header 黑底，logo 白色；header 高度需符規定範；utility area 只放 make.believe related elements；accent color 約 <=10%；不得改 header 高度、漸層、加入其他 logo/graphics。
9. **Reusable Knowledge**：Web brand guideline 應把 header 區域切成 logo isolation zone 與 utility zone。
10. **Abstraction**：`Layout System`、`Web UI`、`AI-Checkable Rules`。
11. **Visual Evidence**：print placement diagram、website header diagram、incorrect header examples 是證據。

## make.believe Motion Logo

1. **Chapter Purpose**：讓品牌訊息在動態媒介中保留可辨識的 duality 與時間規格。
2. **Definition**：定義 motion logo、blue/yellow light ignitions、dual element、1.5/3/5 秒版本、commercials 與 digital media 使用。
3. **Relationship**：它延伸 composite logo 到時間型媒體，並為廣告與數位影片提供結尾或品牌動態資產。
4. **Design Execution**：以三種長度版本與 dual element 控制不同媒介的品牌節奏。
5. **Principles**：Motion identity 應是品牌概念的動態翻譯，而非任意動畫效果。
6. **Frameworks**：motion identity framework：concept duality → visual ignition → duration variants → channel use。
7. **Patterns**：用固定秒數版本管理動態 logo，是可複用 motion governance 模式。
8. **Rules**：AI 可檢查：motion logo 僅使用官方 1.5/3/5 秒版本；digital media 可用三種版本；不得自行重製 dual element 或變更藍黃光點火效果。
9. **Reusable Knowledge**：Motion guideline 應記錄 duration tokens、channel mapping 與 source of truth。
10. **Abstraction**：`Motion Identity`、`Visual Identity`。
11. **Visual Evidence**：motion logo 說明頁與版本列表是動態資產證據。

## Other Important Information

1. **Chapter Purpose**：收束語言、本地化、權限、期限、地域與修訂流程，確保 make.believe 作為集團規範被治理。
2. **Definition**：定義 mantra、非英語例外、terms、guideline 階層、CEO approval、period of use、distribution、territory、contact。
3. **Relationship**：它把品牌訊息從設計規範放回 Sony Group Brand Management Rules 的治理體系。
4. **Design Execution**：用 subordinate guideline、CEO approval、2009-2014 period、worldwide territory 與 third-party distribution ban 管理風險。
5. **Principles**：全球品牌訊息需要明確的 authority、timebox 與 localization exception。
6. **Frameworks**：governance framework：parent rules → approved exception → revision authority → validity period → territory/contact。
7. **Patterns**：把 guideline 有效期間與修訂權寫明，是大型企業品牌治理的成熟做法。
8. **Rules**：AI 可檢查：make.believe 原則上英文使用；僅列明國家/語言可用翻譯；第三方不得分發；修訂/廢止需 Sony CEO approval；適用期間 2009-09 至 2014-08。
9. **Reusable Knowledge**：Design knowledge base 應保存 guideline 的法律階層與生命週期。
10. **Abstraction**：`Governance`、`Localization`、`Compliance`。
11. **Visual Evidence**：additional information 頁是治理、期限與 contact 證據。
