# European Committee of the Regions Visual identity guidelines

來源：`sources/European_Committee_of_the_Regions_visual_identity_EN_2019.pdf`
版本：June 2019
日期：2019-06
研究視角：Design Infrastructure Researcher

## Summary

這份文件規範 European Committee of the Regions 的 logo、graphic line、印刷媒體、電子媒體與 commissions 子識別系統。這章先建立 CoR 的視覺權威來源，避免多語、多媒體、多單位使用時發生 logo、字體、比例與聯名規則失控。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。主識別必須同時傳達 EU alignment、地方聲音匯聚、可讀性與制度一致。此章建立「symbol concept + multilingual wordmark + proportional grid + application constraints」框架。

執行上，這份 guideline 透過 Visual elements、Logo、Graphic Line、Printed media 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。複雜公共機構可用「機構敘事 + EU shared asset + internal proportional unit」把抽象政治角色轉為可執行設計規範。

## Visual elements

1. **Chapter Purpose**：這章先建立 CoR 的視覺權威來源，避免多語、多媒體、多單位使用時發生 logo、字體、比例與聯名規則失控。
2. **Definition**：它定義 logo 與 graphic line 的核心構件；不直接處理所有媒體成品，而是提供後續模板的母系統。
3. **Relationship**：Printed media、Electronic media 與 Commissions 都依賴此章的 logo、white diamond、typography 與 grid。
4. **Design Execution**：作者把聲波、EU flag 與機構名稱組成三段式 logo，再用 diamond shape 和 EU flag 尺寸衍生比例系統。
5. **Principles**：主識別必須同時傳達 EU alignment、地方聲音匯聚、可讀性與制度一致。
6. **Frameworks**：此章建立「symbol concept + multilingual wordmark + proportional grid + application constraints」框架。
7. **Patterns**：可複用模式是用 logo 內部元素作為量尺，讓 clear space、diamond、grid 與 co-branding 都有同一基準。
8. **Rules**：AI 可檢查：logo 不得變形、改色、換字體、改文字位置、加效果或 outline；negative logo 在 EU flag 周圍需有白線；clear space 等於 4X，X 為 EU flag 高度的 1/8；最小尺寸依 EU flag 寬或水平 logo 高度為 10 mm。
9. **Reusable Knowledge**：複雜公共機構可用「機構敘事 + EU shared asset + internal proportional unit」把抽象政治角色轉為可執行設計規範。
10. **Abstraction**：`Visual Identity`、`Logo System`、`Layout System`、`Governance`。
11. **Visual Evidence**：章節大量使用比例圖、語言版本矩陣、色票、misuse 對照與 grid 示意；功能是比例、比較與治理證據。

### Logo

1. **Chapter Purpose**：確保 logo 在 24 種 EU 語言、正負片、大小、背景與聯名場景中維持一致。
2. **Definition**：定義 CoR logo 的三構件、語言版本、比例、顏色、clear space、最小尺寸與可用例外。
3. **Relationship**：它是所有 stationery、web、社群與 commission materials 的簽章基礎。
4. **Design Execution**：用 EU flag 作為視覺與量尺核心，wordmark 置中對齊，聲波代表資訊收發與地區聲音。
5. **Principles**：合法性來自 EU flag，辨識度來自固定 wordmark 與聲波，彈性來自 multilingual 與 mute logo。
6. **Frameworks**：採用標準 logo、水平替代 logo、小尺寸水平 logo、mute logo 的情境化資產框架。
7. **Patterns**：為多語機構設計時，先定義語言排序與例外使用情境，再定義版面。
8. **Rules**：AI 可檢查：24 語版本需依 ISO 639-1 原文標題排序；雙語版文字上下排列且依正式標題字母順序；Brussels 外牆法規例外不可用於其他情境；聯名 logo 間距至少等於 EU flag 寬。
9. **Reusable Knowledge**：多語 logo 系統應把「語言排序、例外法律情境、聯名視覺重量」明文化。
10. **Abstraction**：`Logo System`、`Localization`、`Legal and Trademark`。
11. **Visual Evidence**：頁面提供語言矩陣、比例標尺、負片背景與 misuse 圖；它們作為機器檢查與人工審核的參考樣本。

### Graphic Line

1. **Chapter Purpose**：讓 logo 以外的頁面語彙也能保持 CoR 風格，而不是只靠標誌貼附。
2. **Definition**：定義主/副字體、Office 替代字體、title ratio、white diamond shape、grid 與 partner/event co-branding。
3. **Relationship**：它將 logo 的 EU flag 尺寸轉成 publication、poster、banner 與 co-branding 的布局語法。
4. **Design Execution**：Myriad Pro 提供跨歐語支援，Minion Pro 為內頁帶來節奏；white diamond 建立 logo 保護區與裝飾語彙。
5. **Principles**：識別不是靜態 logo，而是由字體、幾何容器、比例與位置共同運作。
6. **Frameworks**：採「typography pair + title scale ratio + diamond geometry + format grid」框架。
7. **Patterns**：用固定幾何裝置承接 logo，同時作為版面裝飾與 co-branding 容器。
8. **Rules**：AI 可檢查：headline/subtitle 對比遵守 2:1；Office 場景可用 Arial/Times 替代；white diamond 由 100 x 100 mm 方形旋轉 45°、調整為 120 x 100 mm 並設定角半徑；logo 尺寸需按格式表比例。
9. **Reusable Knowledge**：把圖形裝置的建構步驟寫成參數，可讓 AI 或設計工具重建版面。
10. **Abstraction**：`Typography System`、`Layout System`、`Design Tokens`。
11. **Visual Evidence**：頁面提供 diamond 建構步驟、format table、portrait/landscape grid 與聯名配置示意。

## Printed media

1. **Chapter Purpose**：把母系統轉成日常辦公與出版物模板，降低每次新素材重新設計的成本。
2. **Definition**：涵蓋 stationery、publications、poster、signpost、badge、roll-up、infographic、factsheet 與 political group publications。
3. **Relationship**：它依賴 Visual elements 的字體、diamond、grid 與 logo placement，並為後續 commissions 提供模板基礎。
4. **Design Execution**：作者用毫米、pt、leading、icon 尺寸、社群 footer、diamond width 與 grid column 直接規格化製作。
5. **Principles**：印刷系統需要高精度、可交付、可重複與跨供應商一致。
6. **Frameworks**：此章形成「format-specific grid + fixed contact/footer + component-level specs」框架。
7. **Patterns**：每個媒體先給 grid system，再給 examples，使規則與感性判斷同時存在。
8. **Rules**：AI 可檢查：letterhead、cards、invitations 等需使用指定尺寸、margin、字級與社群 icon 尺寸；brochure/leaflet/poster 需參照 graphic line grid；political group publication 後封面需放 CoR signature。
9. **Reusable Knowledge**：印刷 guideline 應把可測量規格與示例版面分離，前者用於自動檢查，後者用於美感校準。
10. **Abstraction**：`Print System`、`Layout System`、`Operations`。
11. **Visual Evidence**：大量使用實尺寸模板、假文、標註線與成品例圖；功能是製作規格與範例證據。

### Stationery

1. **Chapter Purpose**：統一機構日常往來文件，讓行政溝通也成為品牌觸點。
2. **Definition**：定義 letterhead、business card、compliment card、invitation、envelopes、email signature、folder、notepads、slides 與 promotional items。
3. **Relationship**：它是 graphic line 在最常見辦公物件上的落地。
4. **Design Execution**：用 A4、55 x 85 mm、210 x 105 mm 等尺寸，搭配固定 footer、地址、社群資訊與字級。
5. **Principles**：行政標準化比創意變化更重要；資訊可讀性與機構可信度優先。
6. **Frameworks**：採「document type + fixed information zones + typography specs」框架。
7. **Patterns**：同一 contact/footer 模組跨 stationery 重複使用。
8. **Rules**：AI 可檢查：business card 可有雙語；email signature 使用 Arial；social media icons 尺寸需符規定格；diamond width 與 drop shadow 需按文件標註。
9. **Reusable Knowledge**：辦公物件 guideline 應視為資訊架構與品牌治理，不只是美術模板。
10. **Abstraction**：`Stationery System`、`Operations`、`Information Design`。
11. **Visual Evidence**：每頁呈現實物 mockup 與尺寸標註，支援印刷與供應商溝通。

### CoR Publications

1. **Chapter Purpose**：讓出版物、海報與活動物料在不同內容與合作情境下仍維持 CoR 可辨識性。
2. **Definition**：定義 brochure、leaflet、poster、signpost、badges、roll-ups、infographics、factsheets 與政治團體出版物。
3. **Relationship**：它將 stationery 的精度擴展到公共傳播與活動物料。
4. **Design Execution**：用 diamond frame、light-tone photos、strong colour、negative logo、co-branding stripe 與 grid 組合出多種情境。
5. **Principles**：範例需允許內容多樣性，但 logo 與 diamond 的位置關係必須穩定。
6. **Frameworks**：採「grid first + content variant + co-branding condition」框架。
7. **Patterns**：針對共同製作、CoR 自編、hosted event 等情境提供不同版型。
8. **Rules**：AI 可檢查：co-branded publication 需依三種聯名方式配置；signpost host/co-organised 版型不得混用；badge 應按 signpost 比例縮放；infographic 應有 title header、text area 與 footer area。
9. **Reusable Knowledge**：多種出版物可共享一個 graphic line，只需為情境建立條件分支。
10. **Abstraction**：`Publication System`、`Co-branding`、`Examples`。
11. **Visual Evidence**：章節幾乎每個格式都有 grid 與 example 對照，是模板治理的主要證據。

## Electronic media

1. **Chapter Purpose**：把印刷識別轉化為網站、社群與電子新聞稿，確保數位觸點不脫離主系統。
2. **Definition**：定義網站版面、社群 channel visuals、web banners/posts、press release、joint press release 與 daily press review。
3. **Relationship**：它依賴 logo/colour/typography，但增加 pixel、container、column 與平台尺寸邏輯。
4. **Design Execution**：桌機採 1170 px max-width、3 columns、30 px gutter；社群用負片 logo 與 quote/image visual 分流。
5. **Principles**：數位環境需要 responsive、platform-specific 與可讀性優先。
6. **Frameworks**：採「web layout grid + platform template + communication document」框架。
7. **Patterns**：社群圖分 quote-visuals 與 image-visuals，根據文字或影像焦點選擇 logo 位置。
8. **Rules**：AI 可檢查：desktop home 內容容器 max-width 1170 px 且 3 columns；tablet 2 columns、smartphone 1 column；quote visuals logo 在 top left diamond；image visuals logo 可作 bottom signature；Facebook 1024 x 1024、Twitter 1024 x 611。
9. **Reusable Knowledge**：數位 guideline 可把 responsive breakpoints 與社群內容類型一起納入模板系統。
10. **Abstraction**：`Responsive Layout`、`Social Media Templates`、`Content Design`。
11. **Visual Evidence**：網站頁面用大量 UI 標註與 pixel 值，社群頁用平台 mockup 與色彩樣本。

## Commissions of the CoR

1. **Chapter Purpose**：讓六個 specialised commissions 擁有差異化識別，同時不破壞 CoR 主品牌一致性。
2. **Definition**：定義 commission colours、commission identity layout、stationery、publications、electronic media 與 conferences。
3. **Relationship**：它是主識別系統的子品牌層，依賴 diamond、logo、grid 與 typography。
4. **Design Execution**：作者為每個 commission 指定色彩，並用 duplicating/scaling diamond 的步驟建立可變的子識別結構。
5. **Principles**：子品牌應有差異，但差異必須透過主系統控制，而不是另起爐灶。
6. **Frameworks**：採「masterbrand anchor + commission colour + scaled diamond system + application templates」框架。
7. **Patterns**：用顏色與大 diamond 區分 commissions，用 CoR logo 與固定結構保持連結。
8. **Rules**：AI 可檢查：CIVEX、ENVE、COTER、NAT、ECON、SEDEC 必須使用指定 HEX；commission diamond layout 需依 62%、480%、0 mm corner 等步驟；所有 commission social visuals 必須含 CoR logo 與 commission name。
9. **Reusable Knowledge**：子品牌系統可用「共享母構件 + 專屬色彩 + 固定應用模板」達成治理。
10. **Abstraction**：`Brand Architecture`、`Sub-brand System`、`Color System`。
11. **Visual Evidence**：色票、diamond construction、各 commission 範例與 conference 對照提供子品牌治理證據。

### Commission applications

1. **Chapter Purpose**：證明 commission identity 可從文件延伸到活動、出版、社群與會議，不只是靜態標誌。
2. **Definition**：定義 folders、roll-ups、slides、brochures、studies、opinion leaflets、social posts、posters、programmes、signposts、badges。
3. **Relationship**：它將 commission core identity 接到實際工作流程與對外活動。
4. **Design Execution**：一般 publications 使用 commission branding images；external conferences 改用主辦城市或地區照片。
5. **Principles**：內容場景不同時，識別應能切換重點但仍保留系統骨架。
6. **Frameworks**：採「general publication / external conference」雙情境框架。
7. **Patterns**：會議物料以地點照片建立情境感，publication 以 commission image 建立議題識別。
8. **Rules**：AI 可檢查：external conference image 應聚焦 host city/region；general commission publication 應使用 commission visual identity；partner logo 應位於 co-branding area；programme cover/content pages 應使用各自 grid。
9. **Reusable Knowledge**：子品牌應允許按使用情境切換 imagery，但不能切換結構規則。
10. **Abstraction**：`Event System`、`Publication System`、`Governance`。
11. **Visual Evidence**：各物料提供 grid 與 examples，讓 commission system 的可延展性可被檢查。
