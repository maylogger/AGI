# Sartorius Brand Style Guide

來源：`sources/sartorius_Brand_Style_Guide_2020.pdf`
版本：February 2020
日期：2020-02
研究視角：Design Infrastructure Researcher

## Summary

Sartorius Brand Style Guide 將企業識別拆成 logo、signet、claim/layout、色彩、字體、影像、icon 與插畫系統，並以可量測比例、禁用範例與平台下載治理控制一致性。把最高風險的識別資產先固定，確保任何 touchpoint 都從同一個 word mark / figurative mark 出發。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。識別一致性優先於創意變體；logo 是品牌信任，不是可再設計素材。asset governance + measurement rules：approved download → measurable spacing → size threshold → misuse detection。

執行上，這份 guideline 透過 Logo、Clear Space, Size, Orientation and Variants、Logo Don’ts、Signet 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。用 logo 作為尺寸 token，可讓版面、claim 與 signet 共用同一套量測邏輯。

## Logo

1. **Chapter Purpose**：把最高風險的識別資產先固定，確保任何 touchpoint 都從同一個 word mark / figurative mark 出發。
2. **Definition**：定義 logo 的下載來源、視覺角色、clear space、最小尺寸、方向與背景版本；不處理整體版面。
3. **Relationship**：Logo 是 signet、claim tile、layout proportion 的尺度來源，也是後續所有視覺元件的授權前提。
4. **Design Execution**：以 S 高度建立 clear space，以 15 mm / 120 px 建立可讀性底線，以 don’ts 建立不可逾越邊界。
5. **Principles**：識別一致性優先於創意變體；logo 是品牌信任，不是可再設計素材。
6. **Frameworks**：asset governance + measurement rules：approved download → measurable spacing → size threshold → misuse detection。
7. **Patterns**：先給可用版本，再列不可用變形，是品牌資產 guideline 的高效模式。
8. **Rules**：AI 可檢查：logo width >= 15 mm 或 120 px；clear space >= S height；優先使用黃色背景；不得旋轉、改色、加效果、裁切、使用舊 logo、在黃色底上放白 logo。
9. **Reusable Knowledge**：用 logo 作為尺寸 token，可讓版面、claim 與 signet 共用同一套量測邏輯。
10. **Abstraction**：`Visual Identity`、`Design Tokens`、`AI-Checkable Rules`。
11. **Visual Evidence**：頁面以 logo 圖、clear space 圖、方向示例與 don’ts 對照作為比例與錯誤證據。

## Signet

1. **Chapter Purpose**：處理 full logo 無法放置的極端小空間，同時避免團隊濫用符號。
2. **Definition**：定義 signet 是例外資產，需要 Corporate Branding & Design 核准；它不取代標準 logo。
3. **Relationship**：承接 logo 章但收緊治理，因為 signet 缺少完整 word mark，誤用風險更高。
4. **Design Execution**：以 S 厚度定義 clear space，設定 2.7 mm / 14 px 最小寬度，並限制黑白版本使用。
5. **Principles**：越簡化的品牌符號越需要更高審批門檻。
6. **Frameworks**：exception framework：space constraint → approval → approved version → measurable use。
7. **Patterns**：把簡化資產視為 exception 而非 alternative，可降低品牌漂移。
8. **Rules**：AI 可檢查：若版面可容納 full logo 不應使用 signet；signet width >= 2.7 mm 或 14 px；不得改色、旋轉、當 pattern、與 logo 任意合成或加效果。
9. **Reusable Knowledge**：小尺寸品牌資產應同時有尺寸規則與審核流程。
10. **Abstraction**：`Visual Identity`、`Governance`、`Compliance`。
11. **Visual Evidence**：使用 signet 版本圖、clear space 圖與 don’ts 圖作為例外使用證據。

## Claim & Layout System

1. **Chapter Purpose**：把 “Simplifying Progress” 從口號轉成版面系統，而非讓各 touchpoint 自行擺放。
2. **Definition**：定義分割網格、logo proportion、logo position、claim black tile、claim minimum size 與應用版型。
3. **Relationship**：它把 logo 的尺度延伸成 layout infrastructure，並連到色彩與 typography 的實際應用。
4. **Design Execution**：以 1/4、1/3、1/2、full format 建立 flexible grid；claim 寬度由 logo 衍生，黑 tile 只保留給 claim。
5. **Principles**：品牌版面應靠少量比例規則創造可辨識性，而非硬鎖每個元素位置。
6. **Frameworks**：grid-to-asset framework：format division → logo size → claim tile → content area → application variant。
7. **Patterns**：使用 tile、比例、固定位置與禁用示例建立跨格式版面模式。
8. **Rules**：AI 可檢查：claim 必須白字置於黑 tile；claim tile 不得大於 yellow logo tile；claim 最小 30 mm / 180 px；含 claim 時同頁不得另用黑 tile；logo 位置限 top left、bottom right、centered 或 letterhead top right。
9. **Reusable Knowledge**：口號可被抽象為可量測的 lockup / tile system，而不是文案附屬品。
10. **Abstraction**：`Layout System`、`Messaging`、`Design Tokens`、`AI-Checkable Rules`。
11. **Visual Evidence**：大量 grid 圖、tile 尺寸圖、backdrop/roll-up 應用與 don’ts 建立比例證據。

## Colors

1. **Chapter Purpose**：控制 Sartorius 黃色的品牌辨識度，同時為資料、狀態與複雜 graphics 提供分層色彩。
2. **Definition**：定義 primary、secondary gray、teal highlight、tertiary palette 與 traffic-light functional colors。
3. **Relationship**：色彩章支援 layout、typography、imagery、icons、illustration，但用不同授權層級限制使用。
4. **Design Execution**：以色值、媒介轉換、比例與用途分流：主色建品牌，灰階建資訊，teal/tertiary 僅在必要時分辨資料。
5. **Principles**：強識別色需要配套節制；功能色不可被美術化。
6. **Frameworks**：color hierarchy framework：brand colors → neutral data colors → highlight → tertiary differentiation → status colors。
7. **Patterns**：以用途授權色彩，而非只列 palette，是可複用模式。
8. **Rules**：AI 可檢查：primary yellow #FFED00、black #000000；traffic-light red/yellow/green 僅用於狀態；tertiary 不得做 layout tile；不得使用透明、overlay、gradient 或錯誤 yellow shade。
9. **Reusable Knowledge**：色彩系統可拆成 brand identity、information hierarchy、functional status 三層。
10. **Abstraction**：`Color System`、`Information Design`、`AI-Checkable Rules`。
11. **Visual Evidence**：色票、比例圖、資料圖範例與 don’ts 提供色值與用途證據。

## Typography

1. **Chapter Purpose**：讓文字在全球、多媒介與不同軟體條件下保持一致且可讀。
2. **Definition**：定義 TT Norms Pro、Noto Sans、Arial fallback、字重、字色、tracking、標題大小寫、quotation marks 與 bullet。
3. **Relationship**：它連接品牌語氣與實際文件產製，並約束供應商的字體授權責任。
4. **Design Execution**：以 primary/fallback font、字重角色、左對齊、default tracking、APA headline 與方形 bullet 建立文字操作規範。
5. **Principles**：可讀性與一致性優先於裝飾性排版。
6. **Frameworks**：typography operations framework：licensed brand font → fallback → usage roles → language conventions → typographic tokens。
7. **Patterns**：在 guideline 中加入 dictionary、headline checker 與鍵盤輸入方式，是內容治理模式。
8. **Rules**：AI 可檢查：標準字色只允許 black/white；不得使用 yellow text；headline 依 APA title case；正文左對齊 default tracking；bullet 為 square；不得 underline。
9. **Reusable Knowledge**：字體規範應同時處理視覺、語言、授權與工具限制。
10. **Abstraction**：`Typography System`、`Content Design`、`Operations`。
11. **Visual Evidence**：字體 specimen、字色範例、bullet 尺寸圖與 don’ts 是主要證據。

## Imagery

1. **Chapter Purpose**：把攝影從主觀風格轉為依內容類型可審核的影像系統。
2. **Definition**：定義 product、people、architecture、scientific 四類影像的背景、焦點、光線、穿著、構圖與禁用條件。
3. **Relationship**：它承接色彩與版面，決定品牌在真實世界素材中的可信度與技術感。
4. **Design Execution**：產品用全焦與乾淨背景；人物依職能穿著且不搶產品；建築強調直線；科學影像禁止 retouch。
5. **Principles**：技術品牌的影像應清楚、可信、專業，情緒化效果不可破壞資訊。
6. **Frameworks**：content-type image framework：object clarity → human context → built environment → scientific integrity。
7. **Patterns**：以 subject category 設定不同審核規則，可複用於 B2B 與科技品牌。
8. **Rules**：AI 可檢查：產品應全焦、無硬陰影、背景為 yellow/white/black；人物須與產品互動且穿著符合場域；scientific images 不得 retouch；避免 lens flare、tint、gradient、busy composition。
9. **Reusable Knowledge**：影像 guideline 可轉成 computer vision 檢查項：焦點、背景色、人物比例、場景雜訊與禁用效果。
10. **Abstraction**：`Photography System`、`Compliance`、`AI-Checkable Rules`。
11. **Visual Evidence**：每類影像搭配應用範例與 don’ts，提供情境與錯誤對照證據。

## Icons

1. **Chapter Purpose**：建立可擴充 icon library，讓未來新增圖示仍像同一個系統。
2. **Definition**：定義 16x16 grid、直線邊、錨點、角度、gap 寬度、官方 library 與禁用樣式。
3. **Relationship**：Icons 位於 typography 與 illustration 之間，負責低複雜度資訊符號。
4. **Design Execution**：以 grid、anchor points、固定 gap 與官方下載來源讓新增圖示可被審核。
5. **Principles**：可擴展性來自 mechanics，不只是現有圖示集合。
6. **Frameworks**：icon construction framework：grid → line edge → anchor angle → gap token → library governance。
7. **Patterns**：用幾何 construction rules 控制大型 icon library 是高複用模式。
8. **Rules**：AI 可檢查：icon 必須對齊 16x16 grid；線條直邊；gap 僅 1、1.5、2 squares；不得使用舊式、handmade、圓角、實心色或不同透視。
9. **Reusable Knowledge**：任何 icon guideline 都應把 style 拆成 grid、stroke、corner、gap、perspective 與 library policy。
10. **Abstraction**：`Iconography System`、`Design Tokens`、`Governance`。
11. **Visual Evidence**：grid 圖、gap 圖、icon set 與 don’ts 提供 construction evidence。

## Illustration

1. **Chapter Purpose**：讓複雜流程、人物與技術物件用一致插畫語言表達，而不依賴任意插畫風格。
2. **Definition**：定義插畫色彩階層、rebrush、人物、表情、膚色、secondary characters、場景、室內線索與 isometric technical drawings。
3. **Relationship**：它延伸 icon 的幾何邏輯，並借用 color hierarchy 來建立更高敘事複雜度。
4. **Design Execution**：以黑灰為主、黃色 highlight、幾何形狀、自然比例與等角網格控制一致性。
5. **Principles**：插畫應降低理解成本，不應以風格炫技干擾資訊。
6. **Frameworks**：illustration system framework：palette hierarchy → character mechanics → scene composition → technical isometric drawing。
7. **Patterns**：把 existing illustration rebrush 納入規範，是品牌轉換期間維持一致的治理模式。
8. **Rules**：AI 可檢查：灰階為主、黃色先於 tertiary；secondary characters 不用臉；物件為幾何形；scenario 以置中構圖優先；technical drawings 使用等角 grid 與統一線寬；不得 gradient/drop shadow/blend mode。
9. **Reusable Knowledge**：插畫規範可抽象為 palette、geometry、character rig、scene composition、technical projection 五模組。
10. **Abstraction**：`Illustration System`、`Visual Language`、`Design Tokens`。
11. **Visual Evidence**：rebrush 前後、角色機制、場景範例、isometric grid 與 don’ts 是主要證據。

## Governance

1. **Chapter Purpose**：在文件結尾把疑問、下載與供應商使用導回正式品牌平台與品牌團隊。
2. **Definition**：定義 Brand Platform、branding contact，以及 internal / vendor 使用邊界。
3. **Relationship**：它回收前面所有視覺規則的執行權，防止使用者在不確定時自行發明例外。
4. **Design Execution**：以聯絡窗口與平台資源結束，讓 guideline 不是靜態 PDF，而是品牌作業入口。
5. **Principles**：可執行 guideline 必須提供例外處理路徑。
6. **Frameworks**：governance loop：rule → asset platform → approval/contact → vendor boundary。
7. **Patterns**：把 download links 與 approval contact 嵌入各章，是品牌營運模式。
8. **Rules**：AI 可檢查：若設計使用未列資產或例外 signet/claim/layout，需標記為需 Corporate Branding & Design review。
9. **Reusable Knowledge**：Design knowledge base 應記錄規則，也要記錄誰有權解釋規則。
10. **Abstraction**：`Governance`、`Operations`。
11. **Visual Evidence**：結尾 contact 頁與各章 info links 是治理證據。
