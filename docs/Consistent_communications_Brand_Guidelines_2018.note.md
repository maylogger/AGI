# Commonwealth Foundation Brand Guidelines

來源：`sources/Consistent_communications_Brand_Guidelines_2018.pdf`
日期：Updated March 2018
研究視角：Design Infrastructure Researcher

## Summary

Commonwealth Foundation 的品牌與一致溝通規範，整合使命價值、視覺識別、模板、可及性、影像、house style、web writing 與 partner/grant visibility。先把「brand」定義成組織人格與被感知方式，讓視覺與文字規範都回到一致溝通。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。品牌不只在 logo，而是在信封、工作坊準備、電話回應等行為中被感知。brand personality -> visual identity tools -> governance by communications team。

執行上，這份 guideline 透過 Introduction、Consistent communications、Vision and mission、Our visual identity 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。guideline 開頭應先界定 brand/identity/logo 的邊界。

## Introduction

1. **Chapter Purpose**：先把「brand」定義成組織人格與被感知方式，讓視覺與文字規範都回到一致溝通。
2. **Definition**：定義 brand、visual identity、logo、vision、mission、core values。
3. **Relationship**：為後續視覺系統、house style、visibility rules 提供共同語意基礎。
4. **Design Execution**：用簡短定義與圖示把抽象品牌概念轉成工具箱架構。
5. **Principles**：品牌不只在 logo，而是在信封、工作坊準備、電話回應等行為中被感知。
6. **Frameworks**：brand personality -> visual identity tools -> governance by communications team。
7. **Patterns**：以 definition-first 開場降低跨部門誤解。
8. **Rules**：AI 可檢查：品牌介紹需連結 civic voices、participation、peaceful/equitable society；不得只描述 visual identity。
9. **Reusable Knowledge**：guideline 開頭應先界定 brand/identity/logo 的邊界。
10. **Abstraction**：`Brand Strategy`、`Knowledge Architecture`。
11. **Visual Evidence**：brand/visual identity/logo 關係圖承擔概念分類證據。

### Consistent communications

1. **Chapter Purpose**：建立一致語氣是國際組織信任的必要條件。
2. **Definition**：定義所有 communication 都需表達組織品牌，並由 communications team overseen and managed。
3. **Relationship**：是整份文件的治理聲明。
4. **Design Execution**：把品牌投射擴展到日常行為與材料製作。
5. **Principles**：一致性來自可遵循、可複製、可實作的工具。
6. **Frameworks**：communication material -> brand expression -> communication team governance。
7. **Patterns**：把 guideline 作為組織共同操作手冊，而非設計部門內部文件。
8. **Rules**：AI 可檢查：所有材料需符合 documented personality traits、visual tools 與 tone rules。
9. **Reusable Knowledge**：品牌治理可由「溝通團隊」而非單純 design owner 管理。
10. **Abstraction**：`Governance`、`Operations`。
11. **Visual Evidence**：本節主要是文字定義，視覺證據為概念圖。

### Vision and mission

1. **Chapter Purpose**：讓品牌規範直接承接組織使命與價值，而非任意選色選字。
2. **Definition**：定義 vision、mission 與 Diversity、Collaboration、Integrity、Ingenuity。
3. **Relationship**：後續 weave metaphor、inclusive tone、accessibility 和 partner visibility 都來自這些價值。
4. **Design Execution**：用價值敘述說明 organization 為何要促進 civic participation。
5. **Principles**：公共組織品牌必須同時表現 accountability、participation、diversity 與 innovation。
6. **Frameworks**：vision -> mission -> values -> identity metaphor。
7. **Patterns**：先定義制度角色，再定義表達規則。
8. **Rules**：AI 可檢查：品牌文案需尊重 diversity、collaboration、integrity、ingenuity；不得排除非專業受眾或弱化透明度。
9. **Reusable Knowledge**：mission/value 可被轉譯為 accessibility、tone 和 partnership governance 的檢查規則。
10. **Abstraction**：`Brand Strategy`、`Messaging`。
11. **Visual Evidence**：本節以文字為主，功能是策略證據。

## Our visual identity

1. **Chapter Purpose**：把 civic dialogue 與 collaboration 抽象成 weave/pattern 系統，建立可延展的視覺語言。
2. **Definition**：涵蓋 logo、pattern、typefaces、colours、special uses、sub-brand、examples/templates。
3. **Relationship**：承接 mission/value，並支撐後續 accessibility、images、visibility。
4. **Design Execution**：以三角形與編織圖案比喻政府與 civil society 的雙向對話。
5. **Principles**：視覺識別應從組織運作模型中生成，而非只做裝飾。
6. **Frameworks**：metaphor -> symbol -> pattern -> applications/templates。
7. **Patterns**：用 logo underlying structure 生成 secondary pattern 是可複用 identity pattern。
8. **Rules**：AI 可檢查：pattern 可 resize、flip、crop，但不可 rotate；logo safe area 與最小尺寸需遵守；專業與 in-house 用字需分流。
9. **Reusable Knowledge**：好的 visual identity 會把「意義來源」和「工具使用」一起記錄。
10. **Abstraction**：`Visual Identity`、`Pattern System`、`Templates`。
11. **Visual Evidence**：logo diagram、pattern samples、type specimens、colour palettes 與 templates 是完整視覺證據。

### Logo

1. **Chapter Purpose**：保護 symbol 與 logotype 的關係，避免組織識別被任意重排。
2. **Definition**：logo 由 illustrative C symbol 與 Commonwealth Foundation words 組成；有 horizontal/vertical versions。
3. **Relationship**：作為所有材料與 partner visibility 的主要識別。
4. **Design Execution**：用 minimum sizes、safe area 與 alternate versions 指導不同媒體使用。
5. **Principles**：logo 的空間關係與元件本身同等重要。
6. **Frameworks**：orientation choice + minimum size + safe area + alternate color。
7. **Patterns**：用字母高度 X 定義 safe area。
8. **Rules**：AI 可檢查：vertical logo 最小高度 25mm；horizontal logo 最小寬度 55mm；safe area 為 Commonwealth capital letters 高度的 2 倍；不得換字體、拆分重組、改變元件大小、旋轉或拉伸。
9. **Reusable Knowledge**：logo rule 應包含元件關係和使用情境，而不只是一張圖。
10. **Abstraction**：`Logo System`、`AI-Checkable Rules`。
11. **Visual Evidence**：safe area diagram、alternate versions 與 incorrect uses 是警示證據。

### Pattern

1. **Chapter Purpose**：讓 visual metaphor 在非 logo 場景中繼續運作。
2. **Definition**：五種 weave patterns，可用於 in-house 與外部製作材料。
3. **Relationship**：延伸 logo underlying structure，支援 examples/templates。
4. **Design Execution**：允許 resize、flip、crop，保留角度與三角結構。
5. **Principles**：secondary graphics 應保留核心識別的幾何 DNA。
6. **Frameworks**：fixed geometry + flexible composition。
7. **Patterns**：pattern library 作為品牌視覺動態來源。
8. **Rules**：AI 可檢查：pattern 不得旋轉；可裁切與翻轉；若需 bespoke version 應使用 raw files 並由 professional supplier 處理。
9. **Reusable Knowledge**：pattern 規範應說明哪些變形保留 identity、哪些破壞 identity。
10. **Abstraction**：`Pattern System`、`Visual Language`。
11. **Visual Evidence**：五個 pattern samples 是可複用素材證據。

### Typefaces

1. **Chapter Purpose**：讓專業出版和日常 in-house 文件都能維持相近的品牌聲音。
2. **Definition**：PMN Caecilia 為 professional use；Trebuchet MS 為 in-house 與 fallback。
3. **Relationship**：支援 templates、accessibility 與 house style。
4. **Design Execution**：提供字體樣張與選擇原因：charm、modern personality、small-size legibility。
5. **Principles**：品牌字體需要有可取得的替代方案，否則規範無法在組織內落地。
6. **Frameworks**：professional font + system font fallback。
7. **Patterns**：以字體性格相似性選擇 in-house substitute。
8. **Rules**：AI 可檢查：專業設計材料使用 PMN Caecilia；內部文件、簡報、信件、報告使用 Trebuchet MS；主文字不得小於 accessibility 規範。
9. **Reusable Knowledge**：typography guideline 需將 licensing/availability 納入設計決策。
10. **Abstraction**：`Typography System`、`Operations`。
11. **Visual Evidence**：PMN Caecilia 與 Trebuchet MS 字樣表承擔辨識證據。

### Colour

1. **Chapter Purpose**：把 logo 內色彩抽出成出版、圖表與插畫可用的色彩系統。
2. **Definition**：primary palette for text-based documents，enhanced palette for illustrations/charts，並提供 CMYK/RGB/HEX。
3. **Relationship**：支援 visual identity、templates、accessibility。
4. **Design Execution**：用色票與技術說明橋接印刷與數位。
5. **Principles**：限制色彩能確保所有出版品互相補足。
6. **Frameworks**：logo-derived palette -> use categories -> technical values。
7. **Patterns**：同一色彩提供 print/screen/web 三種 token。
8. **Rules**：AI 可檢查：gradients 必須至少由 primary 或 enhanced palette 中一色定義；色值需匹配核准 CMYK/RGB/HEX；text-based documents 優先 primary palette。
9. **Reusable Knowledge**：色彩章應同時教使用者理解 CMYK、RGB、HEX。
10. **Abstraction**：`Color System`、`Design Tokens`。
11. **Visual Evidence**：色票與 color matching systems 說明是 token 證據。

### Special uses and Sub-brand

1. **Chapter Purpose**：處理非典型 logo 用途與 Commonwealth Writers 關係。
2. **Definition**：logo elements 可分離使用、bespoke colour versions、Commonwealth Writers lock-up。
3. **Relationship**：為特殊媒體與子品牌 visibility 提供例外治理。
4. **Design Execution**：限定 Illustrator Recolor Artwork 作為改色方法，並定義 initiative lock-up。
5. **Principles**：例外需要被明確設計，不應成為任意修改。
6. **Frameworks**：special use -> approved method -> minimum size/lockup rules。
7. **Patterns**：對子品牌使用 adjusted logos，避免直接拼貼。
8. **Rules**：AI 可檢查：bespoke logo colors 僅可用 Adobe Illustrator Recolor Artwork；Commonwealth Writers lock-ups 不得改動；vertical lock-up 最小 40mm，horizontal 最小 24mm。
9. **Reusable Knowledge**：sub-brand governance 應提供 third-party 與 owned materials 的不同規則。
10. **Abstraction**：`Brand Architecture`、`Governance`。
11. **Visual Evidence**：logo lock-up examples 和 minimum-size diagrams 是治理證據。

### Examples and templates

1. **Chapter Purpose**：讓規範可直接轉為產出，尤其支援非設計者製作內部材料。
2. **Definition**：professional applications 與 in-house letterhead/report/PowerPoint templates。
3. **Relationship**：整合 logo、pattern、type、colour。
4. **Design Execution**：以 strategic plan、leaflet、folder、banner、poster、website、Twitter 與 office templates 展示。
5. **Principles**：模板是品牌民主化工具。
6. **Frameworks**：professional examples + in-house preformatted templates。
7. **Patterns**：把高品質外部應用與日常內部模板放在同一章。
8. **Rules**：AI 可檢查：in-house letter/report 使用 Trebuchet MS 11pt，paragraph line spacing multiple 1.15，header/footer 只在 first page。
9. **Reusable Knowledge**：template rules 應列出具體 Word/PowerPoint 可設定參數。
10. **Abstraction**：`Templates`、`Operations`。
11. **Visual Evidence**：多個應用 mockup 與模板截圖是成品參考證據。

## Accessibility

1. **Chapter Purpose**：把組織的 inclusive value 轉成實際可讀性規範。
2. **Definition**：涵蓋對比、字體清晰度、字級、大小寫、斜體、行距、對齊、圖片疊字、裝訂、紙張與非色彩標示。
3. **Relationship**：影響 visual identity、templates、images 與 digital communications。
4. **Design Execution**：用 poor/good/excellent contrast 對照與條列規則降低判斷成本。
5. **Principles**：accessibility 是品牌可信度的一部分，不是事後檢查。
6. **Frameworks**：legibility + contrast + material conditions + redundant cues。
7. **Patterns**：把 print-specific accessibility 與 screen-based accessibility 放在同一章。
8. **Rules**：AI 可檢查：main text 不得小於 9pt；不得使用大寫字作長文字；italics sparingly；large text flush left/rag right；不得右對齊正文；圖片疊字需最大對比；highlight 不得只靠顏色。
9. **Reusable Knowledge**：可及性規範應包含紙張 glare、binding crease 等實體材料條件。
10. **Abstraction**：`Accessibility`、`AI-Checkable Rules`。
11. **Visual Evidence**：contrast examples 是比較與警示證據。

## Using images

1. **Chapter Purpose**：確保照片合法、清楚、有敘事功能並反映 collaborative international remit。
2. **Definition**：包含 caption、credit/copyright、cropping/framing、photography subject、in-house imagery、camera resolution。
3. **Relationship**：支援 publications、web、visibility guidelines。
4. **Design Execution**：要求每張圖有 caption、credit，並選擇 narrative quality、strong focus 的影像。
5. **Principles**：影像必須服務故事與價值，而不是填補空間。
6. **Frameworks**：rights -> narrative relevance -> technical quality -> storage ownership。
7. **Patterns**：若沒有合適、合法、高品質影像，就不使用影像。
8. **Rules**：AI 可檢查：每張圖需有短 caption；需 credit/copyright/permission；避免無關建築或物件；裁切需保留原始來源完整性；in-house camera 應用最高解析度拍攝。
9. **Reusable Knowledge**：影像治理應結合法務、敘事與資產管理。
10. **Abstraction**：`Photography System`、`Content Safety`、`Operations`。
11. **Visual Evidence**：照片與 caption/credit example 是合法使用與敘事證據。

## House style

1. **Chapter Purpose**：把組織聲音變成可檢查的寫作規範，確保跨文件一致。
2. **Definition**：標準文字、術語、tone、代名詞、UK English、數字、標點、大小寫、縮寫、italics、references、contact details、web writing。
3. **Relationship**：承接 mission/value，影響所有 print、web、partner/grant materials。
4. **Design Execution**：用 preferred/not preferred examples 和具體格式規則建立內容 linting。
5. **Principles**：公共溝通需 approachable、inclusive、succinct、simple 且 authoritative without over-authoritative。
6. **Frameworks**：standard content -> tone -> grammar/style -> digital scanning。
7. **Patterns**：house style 同時管理內容、語法、格式與 web usability。
8. **Rules**：AI 可檢查：使用 UK spelling；避免 jargon、不解釋 acronyms、excessive adjectives；句子目標 15-20 words；web text 比 print 少 50%；不得使用 etc.；Foundation 作單數。
9. **Reusable Knowledge**：house style 是最容易轉為 AI linting 的 guideline 模組。
10. **Abstraction**：`Content Design`、`Voice and Tone`、`AI-Checkable Rules`。
11. **Visual Evidence**：本章以文字規則為主，視覺證據較少但規則密度高。

### Standard text and terms

1. **Chapter Purpose**：提供統一的組織描述與政策術語定義。
2. **Definition**：standard text、short description、strapline、civil society、governance、participatory governance、capacity development。
3. **Relationship**：支援 partner/grant visibility 與 public communications。
4. **Design Execution**：用可直接複製的正式文案降低重新撰寫造成的偏差。
5. **Principles**：政策型組織需要穩定術語，避免不同人用不同詞。
6. **Frameworks**：approved boilerplate + glossary。
7. **Patterns**：standard text 與 glossary 放在同一節。
8. **Rules**：AI 可檢查：短描述需寫「We are the Commonwealth’s agency for civil society」；strapline 為「More voices for a fairer world」；術語定義不得任意縮短到失去民主/participation 意義。
9. **Reusable Knowledge**：glossary 可作為 RAG/LLM 內容一致性來源。
10. **Abstraction**：`Messaging`、`Content Design`。
11. **Visual Evidence**：文字 boilerplate 是語意證據。

### Tone of voice and use of English

1. **Chapter Purpose**：讓語氣在國際發展與公共治理議題中保持清楚但不疏離。
2. **Definition**：approachable、inclusive、succinct、simple、authority/confidence、we/you/I、UK English。
3. **Relationship**：影響 house style 所有後續規則。
4. **Design Execution**：提供替代寫法，如 because 取代 due to the fact that。
5. **Principles**：權威不必等於複雜，清楚也是尊重受眾。
6. **Frameworks**：clarity + inclusivity + controlled authority。
7. **Patterns**：用 avoid/replace pair 直接轉成 rewrite rule。
8. **Rules**：AI 可檢查：第一次出現 acronym 需拼出；不用 they/them 指 Foundation；Foundation is 而非 are；使用 you 與受眾直接溝通；使用 colour、organisation、programme 等 UK spelling。
9. **Reusable Knowledge**：tone rules 可轉為 automated copy editor。
10. **Abstraction**：`Voice and Tone`、`Localization`。
11. **Visual Evidence**：文字示例是規則證據。

### Numbers and punctuation

1. **Chapter Purpose**：消除格式不一致造成的專業性損耗。
2. **Definition**：numbers、times、dates、money、apostrophes、commas、lists、quotation marks、capitals、symbols、abbreviations、italics、referencing。
3. **Relationship**：支援所有出版與 web 內容。
4. **Design Execution**：大量例句與例外條件使規則可執行。
5. **Principles**：細節一致性是機構可信度的一部分。
6. **Frameworks**：default rule + exception cases。
7. **Patterns**：每個語法規則都列出 when to override。
8. **Rules**：AI 可檢查：zero-nine 拼字、10+ 用數字；時間用 9am/9.30pm；日期用 21 December；百分比正文寫 25 per cent；bullet list 以 colon 引入且項目末不加標點；預設單引號；heading 只首字大寫；& 拼為 and，除非組織名固定用 &。
9. **Reusable Knowledge**：style guide 中的 punctuation/number rules 最適合轉為 deterministic checker。
10. **Abstraction**：`Content Design`、`AI-Checkable Rules`。
11. **Visual Evidence**：文字規則與例句是檢查證據。

### Contact details and Writing for the web

1. **Chapter Purpose**：統一聯絡資訊呈現，並讓 web content 更易掃描。
2. **Definition**：address/contact format 與 12 項 web writing tips。
3. **Relationship**：連接 print identity 與 digital content。
4. **Design Execution**：用完整地址格式與 web-specific checklist。
5. **Principles**：數位內容應先服務 scanning、accessibility 與任務完成。
6. **Frameworks**：contact hierarchy + inverted pyramid web writing。
7. **Patterns**：web guideline 從讀者行為出發，而非從出版習慣出發。
8. **Rules**：AI 可檢查：web content 開頭需先給 who/what/where/when/why/how；paragraph <60 words；避免 click here/more；圖片需 alt text；不要在 named URL 中使用 http://，自有 URL 寫 commonwealthfoundation.com。
9. **Reusable Knowledge**：web writing rules 可作為 CMS content validation。
10. **Abstraction**：`Content Design`、`Accessibility`。
11. **Visual Evidence**：本節以文字 checklist 為主。

## Visibility guidelines

1. **Chapter Purpose**：治理外部合作與補助關係中品牌如何被呈現。
2. **Definition**：包含 partners 與 grant recipients 的 logo/text、approval、web/social、press、print、photo、audio/video 規則。
3. **Relationship**：承接 logo、standard text、images、contact/enquiries。
4. **Design Execution**：用 bullet requirements 和 reciprocation rules 管理雙方可見性。
5. **Principles**：公共資金與合作關係需要透明、可追蹤、彼此承認。
6. **Frameworks**：relationship type -> required acknowledgement -> approval/reporting flow。
7. **Patterns**：partner visibility 與 grant visibility 分開，因權力關係和責任不同。
8. **Rules**：AI 可檢查：partner logos side by side，Foundation logo on left，logos equal size；text-only materials 使用指定 organized/supported by line；web 需 link to commonwealthfoundation.com；press releases 需提前給 Communications Manager proof。
9. **Reusable Knowledge**：visibility guideline 是品牌、規定、合約與資助治理的交叉模組。
10. **Abstraction**：`Governance`、`Compliance`、`Partnership Operations`。
11. **Visual Evidence**：partner lock-up examples 與 minimum size diagrams 是可見性證據。

### Partners

1. **Chapter Purpose**：確保合作方在共同材料中正確呈現 Foundation 並履行審核流程。
2. **Definition**：partner logo lock-up、standard text、press/print/web/social/photo/audio/video reciprocation。
3. **Relationship**：比 grant recipients 更強調 reciprocal visibility。
4. **Design Execution**：規定 proof sharing、schedule agreement 與 mutual standard text/link。
5. **Principles**：合作是雙向的，因此 visibility 也需互惠。
6. **Frameworks**：joint production + approval + reciprocal publishing。
7. **Patterns**：將品牌 visibility 納入 partnership agreements。
8. **Rules**：AI 可檢查：所有 partnership print/screen materials 需包含 logo lock-up；print proofs distribution/printing 前需 approval；social/web 需含連結和 standard text。
9. **Reusable Knowledge**：partner guideline 可轉為合作交付 checklist。
10. **Abstraction**：`Partnership Operations`、`Governance`。
11. **Visual Evidence**：lock-up examples 是配置證據。

### Grant recipients

1. **Chapter Purpose**：確保受補助專案公開承認資金來源並回傳素材。
2. **Definition**：Supported by logo lock-up、text-only acknowledgement、link、standard text、reporting copies。
3. **Relationship**：比 partner section 更偏向 accountability 與 reporting。
4. **Design Execution**：提供 logo download URL、colour/mono options 與 minimum width。
5. **Principles**：補助 visibility 是透明度與資金責任的一部分。
6. **Frameworks**：funding acknowledgement + web link + reporting evidence。
7. **Patterns**：受補助者可用 logo lock-up 或文字替代，但不可不承認。
8. **Rules**：AI 可檢查：print/screen materials related to funded project 必須 include Supported by logo lock-up；text-only 使用「This project is supported by the Commonwealth Foundation」；需送交 copies 作 reporting。
9. **Reusable Knowledge**：grant visibility 可抽象為 funder acknowledgement module。
10. **Abstraction**：`Compliance`、`Grant Operations`。
11. **Visual Evidence**：Supported by logo lock-up 與 minimum width 是治理證據。

## Digital files

1. **Chapter Purpose**：避免錯誤檔案格式造成品質或生產問題。
2. **Definition**：JPEG/PNG for in-house low-res RGB；TIFF for professional high-res CMYK；AI/EPS vector for designers/printers；logo index。
3. **Relationship**：支援所有 logo、template、print/digital 應用。
4. **Design Execution**：用用途、解析度、色彩模式與使用者類型分類。
5. **Principles**：資產治理不只管視覺，也管檔案格式與供應鏈。
6. **Frameworks**：file type -> use case -> resolution -> colour mode -> allowed recipient。
7. **Patterns**：把檔案索引放進 guideline 可降低錯拿素材。
8. **Rules**：AI 可檢查：JPEG/PNG 不得供 printers/design suppliers；TIFF 用於 leaflets/posters；AI/EPS 僅供 designers/printers 且需確認供應商接受。
9. **Reusable Knowledge**：asset management 應把 file formats 納入品牌規範。
10. **Abstraction**：`Operations`、`Asset Management`。
11. **Visual Evidence**：logo file index 是素材治理證據。

## Contact

1. **Chapter Purpose**：提供品牌與組織聯絡入口。
2. **Definition**：地址、電話、email、website。
3. **Relationship**：收束 governance 與 enquiries。
4. **Design Execution**：用固定格式列出聯絡資訊。
5. **Principles**：guideline 應提供可追問的責任端點。
6. **Frameworks**：document -> owner/contact。
7. **Patterns**：contact page 作為 escalation path。
8. **Rules**：AI 可檢查：外部材料的 contact details 應依文件格式與相對大小呈現。
9. **Reusable Knowledge**：每份 guideline 都應有 owner/contact metadata。
10. **Abstraction**：`Governance`、`Operations`。
11. **Visual Evidence**：無大量視覺，主要是治理資訊。
