# eftpos Brand Guidelines

來源：`sources/eftpos_Brand_Guidelines_2021.pdf`
版本：1.0
日期：Published March 2021
研究視角：Design Infrastructure Researcher

## Summary

eftpos 2021 品牌規範，從品牌策略、視覺識別、應用範例延伸到卡片、contactless、mobile 與 online 品牌規定要求。先把 eftpos 定位為 Australia’s original fintech，讓後續支付、卡片與商戶情境都有品牌人格支撐。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。金融科技品牌需同時讓人感到創新、可信、在地與實用。business / what we do / purpose → personality → values → manifesto。

執行上，這份 guideline 透過 Our Brand、Identity System、Logo、Typography and Colour 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。品牌價值可寫成內部工作方式，而不只是對外形容詞。

## Our Brand

1. **Chapter Purpose**：先把 eftpos 定位為 Australia’s original fintech，讓後續支付、卡片與商戶情境都有品牌人格支撐。
2. **Definition**：定義 business、what we do、purpose、personality、values 與 manifesto；不處理 logo 製作細節。
3. **Relationship**：此章為 identity system、photography、card branding 與 mobile requirements 提供策略與語氣依據。
4. **Design Execution**：以「Humble Australian Innovator」連結 down to earth、practical、bold、problem solver 與 family oriented 等特質。
5. **Principles**：金融科技品牌需同時讓人感到創新、可信、在地與實用。
6. **Frameworks**：business / what we do / purpose → personality → values → manifesto。
7. **Patterns**：可複用為 fintech brand foundation：把技術能力轉譯為社會目的與人性特質。
8. **Rules**：溝通應能支持 do good for Australia、customer success、test and learn、bold leadership、collaboration、family orientation；避免過度矽谷式或抽象創新語言。
9. **Reusable Knowledge**：品牌價值可寫成內部工作方式，而不只是對外形容詞。
10. **Abstraction**：`Brand Strategy`、`Voice and Tone`、`Product Language`。
11. **Visual Evidence**：此章以策略文字與 manifesto 為主，功能是價值與行為證據。

## Identity System

1. **Chapter Purpose**：建立足以支撐 corporate、merchant、card、digital、co-branding 的完整識別工具箱。
2. **Definition**：包含 core toolkit、logo、typography、colour、overlays、gradients、neumorphism、margins、grid、photography、iconography、co-branding。
3. **Relationship**：承接 Our Brand 的 personality，把「澳洲、動態、安全、照顧、數位」轉成可使用的視覺規則。
4. **Design Execution**：logo 的 ribbon-like e 表示 motion、forward momentum、safety、care、growth；色彩取自澳洲花卉與地景。
5. **Principles**：支付品牌的識別必須可在極小尺寸、卡片、POS、wallet 與大型傳播間維持辨識。
6. **Frameworks**：core toolkit → logo governance → type / color tokens → layout systems → imagery / co-branding。
7. **Patterns**：將 logo anatomy 連到 gradients、curves、neumorphism 與 grid，使整個視覺語言由核心符號派生。
8. **Rules**：不得使用非官方 logo artwork；不得 alter、re-draw、re-create；corporate communications 必須帶 registered trademark symbol；所有 brandmark artwork 需送 Brand and Marketing team approval。
9. **Reusable Knowledge**：高度受規範產業的 identity system 需要把品牌規則與 approval workflow 寫在同一層。
10. **Abstraction**：`Visual Identity`、`Design Tokens`、`Governance`。
11. **Visual Evidence**：logo variants、色票、字樣、grid、攝影與 co-branding diagrams 是此章的主要證據。

### Logo

1. **Chapter Purpose**：保護 eftpos 最珍貴資產在各種尺寸與卡片 / 數位環境的辨識。
2. **Definition**：定義 primary logo、horizontal logo、small use logo、clear space、minimum sizes、usage、placement、logo on imagery。
3. **Relationship**：是所有 corporate、card、mobile 與 co-branding 規則的共同基礎。
4. **Design Execution**：依 vertical、horizontal、small use 與 reversed mono 建立版本分工；small use 針對極小尺寸調整比例與 tint。
5. **Principles**：小尺寸不是縮小同一圖檔，而是需要專門優化版本。
6. **Frameworks**：application space → logo composition → colour mode → approval requirement。
7. **Patterns**：restricted-use variants 適合處理銀行卡、第三方背景與不可控環境。
8. **Rules**：清除空間等於 eftpos wordmark x-height；整體 logo 不得小於 8mm / 23px 或 17mm / 48px 版本門檻；primary with tagline 25mm / 70px，small use 10mm / 28px；不得 stretch、skew、rotate、flip、drop shadow、3D effects 或低對比背景。
9. **Reusable Knowledge**：logo 規範應提供不同尺寸的 composition switching rule。
10. **Abstraction**：`Visual Identity`、`AI-Checkable Rules`。
11. **Visual Evidence**：版本矩陣、clear space、minimum size、misuse examples 與 imagery examples 是規定證據。

### Typography and Colour

1. **Chapter Purpose**：建立 eftpos 的聲音與色彩辨識，兼顧品牌個性、Office 可用性與 accessibility。
2. **Definition**：Sharp Grotesk Semibold 23 為 headline，Manrope 為 longform，Arial 為 Office fallback；色彩包含 Waratah、Jacaranda、Eucalypt Flower、Midnight Purple、White、Black 與 functional colours。
3. **Relationship**：typography 支撐語句的 bold innovator 感；colour 連回澳洲自然與 logo。
4. **Design Execution**：以 vivid Australian palette 與 AA web accessibility for large text / graphic devices 作設計理由。
5. **Principles**：金融品牌的色彩要鮮明但不能犧牲對比與可用性。
6. **Frameworks**：brand palette for identity + functional palette for UX / reports / complex journeys。
7. **Patterns**：用 functional colours 作為 restricted secondary palette，可避免資料視覺化隨意擴色。
8. **Rules**：Waratah HEX `E50056`、Jacaranda `401463`、Eucalypt Flower `B0147D`、Midnight Purple `1F0038`；functional colours 僅在 primary palette 不足時少量使用，外部溝通需 approval。
9. **Reusable Knowledge**：色彩系統應分成 identity colour 與 functional colour，並對後者加治理門檻。
10. **Abstraction**：`Typography System`、`Color System`、`Accessibility`。
11. **Visual Evidence**：字體 specimen、色票、功能色圖表與 opacity overlay examples 是生產與對比證據。

### Layout and Imagery

1. **Chapter Purpose**：把視覺語言組裝成清楚、乾淨、數位感且澳洲在地的版面與影像系統。
2. **Definition**：涵蓋 neumorphism、margins、16x grid、photography areas、local heroes、Australia、iconography。
3. **Relationship**：將 logo 的曲線與 fintech 特質擴展到版面、按鈕感、照片與 icon。
4. **Design Execution**：margin 等於 logo 寬度一半；grid 可用 2x、4x、8x、16x；photography 聚焦 real Australians、local businesses、customer connection。
5. **Principles**：品牌圖像要呈現真實在地生活，而不是 cliché Australia symbols。
6. **Frameworks**：layout grid + authentic photography taxonomy + simple rounded icon rules。
7. **Patterns**：local heroes / customer experience / Australia / studio 是服務型品牌攝影分類模型。
8. **Rules**：避免澳洲地圖等 cliché symbols；攝影需真實、down to earth、不過度擺拍；icons 應簡單、粗線、圓角，只有圓形可實心。
9. **Reusable Knowledge**：在地品牌可用「真實人物與商戶」取代符號化地理 cliché。
10. **Abstraction**：`Layout System`、`Photography System`、`Iconography System`。
11. **Visual Evidence**：grid examples、photography draft pages、icon examples 與 neumorphism settings 是示範與流程證據。

### Co-branding

1. **Chapter Purpose**：確保 eftpos 在與其他支付、銀行或合作品牌並列時不被弱化。
2. **Definition**：定義 optical balance、clear space、divider keylines、horizontal / vertical co-branding。
3. **Relationship**：為 card、terminal、acceptance decals 等多品牌支付場景做前置規範。
4. **Design Execution**：以 wordmark 高度 X 與 2X spacing 管理 logo 與 divider。
5. **Principles**：支付網路品牌的權益很常在 co-branding 中流失，因此需要 parity 規則。
6. **Frameworks**：logo parity + clear space + divider keyline + approval。
7. **Patterns**：可複用為 payment network co-branding module。
8. **Rules**：eftpos logo 寬度至少等於其他 network logos；divider 與各 logo 間距為 2X；divider 延伸超過 logo edge 的高度為 X；所有 co-branding 需 approval。
9. **Reusable Knowledge**：co-branding 的 AI 檢查可量測 logo width、spacing 與 keyline extension。
10. **Abstraction**：`Brand Architecture`、`Legal and Trademark`、`AI-Checkable Rules`。
11. **Visual Evidence**：水平與垂直 lockup diagrams 是比例與 parity 證據。

## Application Examples

1. **Chapter Purpose**：示範識別系統如何在企業文件、簡報、商店、戶外、商品與標示中保持一致。
2. **Definition**：包含 look and feel、stationery、folder、document covers、flyers、corporate collateral、MS Word、PowerPoint、shopfront decal、merchandise、posters、OOH、signage。
3. **Relationship**：把 Identity System 的規則轉為可模仿的 production examples。
4. **Design Execution**：以 curves、solid shapes、gradients、cropping、holding shapes 與大型 Sharp Grotesk headline 建立 look。
5. **Principles**：手冊應展示「目標狀態」而不是只列素材規格。
6. **Frameworks**：corporate template system + campaign / retail application system。
7. **Patterns**：可複用為 guideline 的 application library：先純粹 look & feel，再列日常模板與外部媒體。
8. **Rules**：Word / PowerPoint 應使用提供模板與預設字級；poster 與 OOH 若為 working draft 或 reference images，應標示不可作最終授權素材。
9. **Reusable Knowledge**：品牌範例應標示概念性、內部 briefing 或未授權圖片狀態，避免被誤用。
10. **Abstraction**：`Examples`、`Templates`、`Operations`。
11. **Visual Evidence**：大量 mockups 與模板頁是設計方向、版面與生產流程證據。

## Card Branding and Contactless Requirements

1. **Chapter Purpose**：把一般品牌規範提升為支付卡與 contactless 場景的規定要求，管理會員、發卡方、收單方與第三方使用。
2. **Definition**：包含 compliance / approvals、licensing、contactless brandmark lockup、clear space、card appearance、only / prepaid / multi-network / co-brand cards、terminal devices、acceptance decals。
3. **Relationship**：這是 eftpos guideline 最具制度性的部分，直接連接品牌、法律、支付規則與技術規格。
4. **Design Execution**：以 approval process、Service Now Member Portal、eCAPS、EMVCo contactless marks sublicense 與卡片 mockups 建立治理鏈。
5. **Principles**：在支付系統中，品牌一致性同時是使用者辨識、規定與網路權益問題。
6. **Frameworks**：member responsibility → artwork approval → card type rules → contactless licensing → terminal / decal parity。
7. **Patterns**：可複用為 regulated brand requirements module。
8. **Rules**：所有 card designs 與 eftpos brandmark / corporate logo 使用必須先 approval；contactless lockup 不得小於 21mm / 60px；front of card 的 eftpos logo 與 contactless mark 必須分離；reverse mono logo restricted 且需 permission。
9. **Reusable Knowledge**：金融與支付品牌 guideline 應將視覺資產、技術規格與授權條款合併管理。
10. **Abstraction**：`Compliance`、`Legal and Trademark`、`Product Language`。
11. **Visual Evidence**：卡片正反面示意、contactless lockup、minimum size diagrams、terminal 與 decals 範例是規定證據。

### Compliance and Licensing

1. **Chapter Purpose**：確保所有會員與下游商戶對品牌使用負責，並處理 EMVCo contactless marks 的再授權。
2. **Definition**：定義 approval process、submission route、disclosure、Contactless Marks、Head Licensor、sublicence、termination。
3. **Relationship**：為後續所有卡片與 mobile 規格建立法律與程序前提。
4. **Design Execution**：以條款化語言描述會員責任、portal submission、商戶素材審批與 contactless marks 使用義務。
5. **Principles**：授權鏈越長，guideline 越需要明確寫出誰負責提交、誰能批准、誰能終止。
6. **Frameworks**：approval before use + licence conditions + infringement notification + termination windows。
7. **Patterns**：可複用為 B2B2C network brand governance。
8. **Rules**：會員必須經 Service Now Member Portal 提交 finished scale colour sample；未經書面同意不得公開文件；Contactless Mark 使用須包含 attribution 並遵守 EMVCo guidelines。
9. **Reusable Knowledge**：品牌文件本身也可能是受限文件，應在 guideline 中標示 disclosure rules。
10. **Abstraction**：`Governance`、`Compliance`、`Legal and Trademark`。
11. **Visual Evidence**：此小節以文字條款為主，功能是程序與權利證據。

### Card and Device Rules

1. **Chapter Purpose**：讓每種卡片與 POS / decals 場景都能被一致驗收。
2. **Definition**：涵蓋 eftpos only、contactless、prepaid、multi-network、co-brand、terminal devices、acceptance decals。
3. **Relationship**：把 licensing 轉成具體設計條件。
4. **Design Execution**：以卡片類型分頁說明 logo 位置、debit / prepaid wordmark、contactless lockup、電子使用提示與 reverse card rules。
5. **Principles**：支付介面的品牌規則需同時處理正面辨識、背面資訊與多網路 parity。
6. **Frameworks**：card type → required mark → position → size / spacing → approval。
7. **Patterns**：卡片規格可拆成正面規則、背面規則、wordmark 規則、contactless 規則。
8. **Rules**：eftpos only cards front 必須有 eftpos brandmark；debit / prepaid wordmark 不得與 logo 整合；multi-network cards 背面必須顯示 eftpos logo 或 contactless lockup；terminal logos 必須 parity；acceptance decals 背景通常 white。
9. **Reusable Knowledge**：多網路支付品牌需以 parity、position、separation、minimum size 作主要 AI-checkable criteria。
10. **Abstraction**：`Product Language`、`UI Components`、`AI-Checkable Rules`。
11. **Visual Evidence**：各卡種正反面 mockup、POS terminal 與 decals 是規定比對證據。

## Mobile and Online Brand Requirements

1. **Chapter Purpose**：將品牌要求延伸至 tokenized payment、mobile wallet、digitised card 與 third-party UI。
2. **Definition**：包含 minimum brand requirements、digitised card presentation、contactless brandmark device、Apple Pay、Google Pay、Samsung Pay、third party wallets。
3. **Relationship**：承接 card branding，將實體卡規範轉換到 mobile 與 online。
4. **Design Execution**：定義 payment token、BIN、digital card art、wallet-specific file formats、dimensions、privacy exclusions 與 logo composition rules。
5. **Principles**：數位卡面不能只是實體卡截圖，必須符合平台、隱私與品牌 parity。
6. **Frameworks**：digital card art requirements + wallet platform specs + brand approval。
7. **Patterns**：可複用為 mobile wallet card art module。
8. **Rules**：所有 UI 需 eftpos branding approval；digital card art 不得顯示 cardholder name、PAN、BIN、expiry、EMV chip 或實體卡照片；Apple / Google / Samsung card art 需 RGB、1536 x 969 PNG 輸出、無邊框留白、不可含 CVV。
9. **Reusable Knowledge**：數位支付品牌規範應把視覺品牌、隱私資訊與平台輸出規格寫在同一張檢查表。
10. **Abstraction**：`Product Language`、`Interaction Design`、`Compliance`。
11. **Visual Evidence**：wallet screenshots、digital card examples、spec tables 與 contactless brandmark diagrams 是平台驗收證據。
