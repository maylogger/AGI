## Summary

California DFPI 的品牌規範，涵蓋品牌介紹、logo、字體、色彩、攝影、圖像、格線、應用範例與檔案格式詞彙。先建立 DFPI 的公共任務，讓後續視覺規範服務於消費者保護、信任、創新與公平。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。政府金融品牌需要先建立可信任與公共責任，再談視覺個性。機構職能 → mission → vision 的公共品牌定位框架。

執行上，這份 guideline 透過 Brand Introduction、Brand Logo Design、Logo Clear Space、Logo Misuse 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。公共機構品牌可用任務合法性作為設計規範的前置條件。

## Brand Introduction

1. **Chapter Purpose**：先建立 DFPI 的公共任務，讓後續視覺規範服務於消費者保護、信任、創新與公平。
2. **Definition**：定義組織是金融監理、執法、教育與市場風險預防者；不負責細化單一政策或法律條文。
3. **Relationship**：使命與願景是 logo、影像與語氣一致性的合法性來源。
4. **Design Execution**：用 about us、mission、vision 三段把機構功能轉成品牌角色。
5. **Principles**：政府金融品牌需要先建立可信任與公共責任，再談視覺個性。
6. **Frameworks**：機構職能 → mission → vision 的公共品牌定位框架。
7. **Patterns**：適用於政府 guideline 的開場模式：先說明服務對象、權責與風險治理。
8. **Rules**：內容應能連回 protect consumers、foster trust、innovation、fairness；避免使用與監理公信力衝突的過度娛樂語氣。
9. **Reusable Knowledge**：公共機構品牌可用任務合法性作為設計規範的前置條件。
10. **Abstraction**：`Brand Strategy`、`Governance`、`Compliance`。
11. **Visual Evidence**：此章文字多於圖像，視覺功能是建立正式、可信與制度型語境。

## Brand Logo Design

1. **Chapter Purpose**：保護最核心識別，確保不同部門與供應商不因任意變形削弱品牌統一。
2. **Definition**：定義 full logo、logo symbol、monoline logo、clear space、misuse 與 color variations。
3. **Relationship**：承接公共信任定位，將「現代、乾淨、專業」落到標誌使用。
4. **Design Execution**：以 clear space、圖像禁用範例與色彩變體建立操作邊界。
5. **Principles**：政府品牌的 logo 需要穩定、可辨識、不可被裝飾化。
6. **Frameworks**：可用版本 → 空間保護 → 禁止變更 → 色彩變體的標誌治理流程。
7. **Patterns**：用「Do not」錯誤圖集是最直接的外包審稿模式。
8. **Rules**：不得套用效果、漸層、旋轉、扭曲、skew、置入形狀或放在同色背景；clear margin 應等於 logo icon 的 X。
9. **Reusable Knowledge**：logo 規範應同時包含允許版本與錯誤範例，AI 審查可先比對 forbidden operations。
10. **Abstraction**：`Visual Identity`、`Legal and Trademark`、`AI-Checkable Rules`。
11. **Visual Evidence**：使用 logo 尺寸示意、construction、錯誤示例與顏色變體，功能是比例與規定證據。

### Logo Clear Space / Misuse

1. **Chapter Purpose**：把主觀「看起來不擠」轉為可度量的版面規則。
2. **Definition**：clear space 是 logo 周圍不可被圖形或文字侵入的區域；misuse 是所有破壞形狀、顏色或可讀性的操作。
3. **Relationship**：這些規則是所有應用模板與輸出物的基礎驗收條件。
4. **Design Execution**：以 X 作為可縮放單位，讓不同尺寸都能維持同樣保護比例。
5. **Principles**：用 logo 自身元素作測量單位，比固定毫米數更容易跨媒介。
6. **Frameworks**：X-based spacing + forbidden transformations。
7. **Patterns**：可複用為任何品牌的 logo QA 模組。
8. **Rules**：檢查 logo 外框四側是否至少有 X；檢查 CSS / design layer 是否有 rotate、skew、drop-shadow、gradient、mask shape 或非核准背景。
9. **Reusable Knowledge**：AI 可用圖層屬性與色彩對比檢查 logo misuse。
10. **Abstraction**：`Layout System`、`AI-Checkable Rules`。
11. **Visual Evidence**：clear space diagram 與 misuse thumbnails 是規範本身的主要證據。

## Brand Typography

1. **Chapter Purpose**：在 print、digital 與 office 環境中建立可控的字體階層與可讀性。
2. **Definition**：指定 Arial 作為 print primary，Gotham 作為 digital primary，Segoe UI 作為 secondary，並定義 headline、heading、subheadline、body copy。
3. **Relationship**：字體讓 DFPI 的專業語氣在政策、教育與行銷內容中一致。
4. **Design Execution**：提供字樣 specimen、字重、字級與行距。
5. **Principles**：公共資訊設計需優先處理可讀性與文件生產可行性。
6. **Frameworks**：媒介分流字體系統：print font、digital font、secondary / body font。
7. **Patterns**：字體規範搭配用途說明，而非只列字體名稱。
8. **Rules**：headline 應 sparingly 使用；body copy 使用 Segoe UI Regular 10pt / 14pt leading；heading 用於多頁文件或網站新段落。
9. **Reusable Knowledge**：字體系統應包含 fallback 與使用場景，降低跨部門產製錯誤。
10. **Abstraction**：`Typography System`、`Content Design`。
11. **Visual Evidence**：字母與數字 specimen、leading 範例與階層示例提供可讀性證據。

## Brand Color System

1. **Chapter Purpose**：把品牌辨識固定在可實作的數值，減少不同材料中的色差與隨意配色。
2. **Definition**：定義 primary colors 與 secondary colors，包含 Hex 與 RGB。
3. **Relationship**：色彩支撐 logo、圖像、資料視覺化與模板應用。
4. **Design Execution**：用色票清單呈現主色與多組次色，滿足行銷與數位需求。
5. **Principles**：色彩規範要把品牌核心色與功能擴充色分開。
6. **Frameworks**：Primary for identity，secondary for flexibility。
7. **Patterns**：色彩 token 清單可直接轉成設計系統或 AI 檢查表。
8. **Rules**：僅使用列出的 Hex / RGB；主色包含 `#AA8A00` 與 `#002D72`；新增色彩需被視為不符合規定。
9. **Reusable Knowledge**：公共品牌可用有限主色維持權威感，再以次色支援溝通彈性。
10. **Abstraction**：`Color System`、`Design Tokens`。
11. **Visual Evidence**：色票與數值是此章的核心視覺證據。

## Brand Visuals

1. **Chapter Purpose**：控制照片與圖像主題，確保政府金融品牌呈現多元、公平、包容與服務場景。
2. **Definition**：定義 approved photos / images、photography categories、data visualization、social icons、web icons。
3. **Relationship**：將 brand introduction 中的 consumer protection 與 public awareness 轉成可見人物與服務情境。
4. **Design Execution**：以 executive、industry、consumer services / financial protection、legal 等範圍區分素材用途。
5. **Principles**：影像不只是裝飾，而是組織服務承諾的證據。
6. **Frameworks**：Audience / function-based image taxonomy。
7. **Patterns**：用分類素材庫降低各部門自行找圖造成的不一致。
8. **Rules**：行銷材料使用的照片應屬於核准範圍；影像應能說明 diversity、equity、inclusion；不得使用與金融保護任務無關的通用 stock image。
9. **Reusable Knowledge**：攝影規範應同時說明「可用主題」與「品牌價值證據」。
10. **Abstraction**：`Photography System`、`Information Design`。
11. **Visual Evidence**：多頁照片範例與 icon / data visualization 頁面提供可選素材與分類證據。

## Brand Grid System

1. **Chapter Purpose**：將品牌元素放入文件、網站與印刷物的版面結構，避免只規範資產而沒有排版方法。
2. **Definition**：定義 article grid system、business card、letterhead 等官方應用。
3. **Relationship**：承接字體、色彩與 logo，將它們組裝成可交付物。
4. **Design Execution**：以 lorem ipsum 範例展示欄寬、字級、標題、聯絡資訊與頁面層級。
5. **Principles**：格線是把品牌一致性落到日常文件的操作層。
6. **Frameworks**：Article layout → stationery / card / letterhead applications。
7. **Patterns**：以模板示例降低非設計人員的版面決策負擔。
8. **Rules**：文章版面應維持指定 typography grid；business card 與 letterhead 應用官方聯絡資訊排列與 logo 格式。
9. **Reusable Knowledge**：品牌手冊應提供最常見辦公輸出模板，而非只提供行銷素材。
10. **Abstraction**：`Layout System`、`Operations`、`Templates`。
11. **Visual Evidence**：網格、名片與信紙 mockup 是版面規範的主要證據。

## Brand Summary and Glossary

1. **Chapter Purpose**：重申品牌資產的價值，並補足執行者需要的檔案格式知識。
2. **Definition**：summary 定義品牌一致性的原因；glossary 定義 JPEG、AI、PNG、TIFF、PDF、EPS 等格式。
3. **Relationship**：收束前述規範，讓使用者理解不遵守會削弱外部一致性。
4. **Design Execution**：以強調句說明 identity 是 logo、產品、名稱、商標、廣告、簡報的總體效果。
5. **Principles**：品牌治理包含教育使用者理解檔案類型，否則規範難以正確生產。
6. **Frameworks**：Asset value reminder + production glossary。
7. **Patterns**：在手冊結尾加入檔案詞彙，適合給跨部門與外包協作者使用。
8. **Rules**：輸出前應確認使用正確檔案格式；不得以壓縮失真或錯誤格式交付需印刷或向量的品牌資產。
9. **Reusable Knowledge**：格式教育是品牌系統運營的一部分，不只是附錄。
10. **Abstraction**：`Operations`、`Design Process`。
11. **Visual Evidence**：以文字詞彙為主，視覺功能是作為生產知識清單。
