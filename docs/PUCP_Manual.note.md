# PUCP Manual de marca medios impresos

來源：`sources/PUCP_Manual.pdf`
研究視角：Design Infrastructure Researcher

## Summary

這份印刷品牌手冊以問答式與範例式結構，治理 PUCP 的 imagotipo、視覺領域、版面系統、品牌架構、文具與術語。此單元的目的，是把「Introducción al manual」轉成可執行的品牌治理接口；它避免使用者只憑審美直覺操作，並讓 手冊使用方式與知識入口 在不同媒介中維持一致。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。隱含原則是清晰優先、一致優先、可辨識優先；創意被允許，但必須在不破壞 手冊使用方式與知識入口 的前提下發生。可抽象為「核心資產 -> 使用條件 -> 例外處理 -> 錯誤排除 -> 應用驗證」框架，讓設計者能先判斷情境，再選擇正確規則。

執行上，這份 guideline 透過 Introducción al manual、Logotipo、Territorio visual、Lineamientos del sistema 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。可重用知識是將品牌規範拆成「資產、條件、度量、禁用、範例、審核」六類資料，讓不同品牌都能用同一種知識結構管理一致性。

## Introducción al manual

1. **Chapter Purpose**：此單元的目的，是把「Introducción al manual」轉成可執行的品牌治理接口；它避免使用者只憑審美直覺操作，並讓 手冊使用方式與知識入口 在不同媒介中維持一致。
2. **Definition**：它定義 把常見問題依主題整理，讓 PUCP 社群能正確製作 institucional print materials 的邊界與使用條件；不負責重述整個品牌故事，而是把相關資產、限制、例外與交付情境說清楚。
3. **Relationship**：它承接前面的品牌定位或基礎識別，並影響後續應用、範本、聯名或內容產製；在整體架構中扮演從抽象意圖到具體檢查點的轉譯層。
4. **Design Execution**：作者透過比例、色彩、版面、文字或範例對照，把 把常見問題依主題整理，讓 PUCP 社群能正確製作 institucional print materials 具體化；執行上通常以主版本、允許版本、例外版本與禁用案例建立操作秩序。
5. **Principles**：隱含原則是清晰優先、一致優先、可辨識優先；創意被允許，但必須在不破壞 手冊使用方式與知識入口 的前提下發生。
6. **Frameworks**：可抽象為「核心資產 -> 使用條件 -> 例外處理 -> 錯誤排除 -> 應用驗證」框架，讓設計者能先判斷情境，再選擇正確規則。
7. **Patterns**：可重複使用的模式包括最小尺寸、留白單位、色彩角色、字體層級、背景相容性、範本化應用與 do/don’t 對照。
8. **Rules**：AI-checkable criteria：確認使用的資產版本屬於核准清單；確認尺寸不低於最小值；確認留白未被文字或圖像侵入；確認色彩、字體、對齊與背景條件符合章節限制；確認沒有旋轉、變形、改色、重繪或自製替代物。
9. **Reusable Knowledge**：可重用知識是將品牌規範拆成「資產、條件、度量、禁用、範例、審核」六類資料，讓不同品牌都能用同一種知識結構管理一致性。
10. **Abstraction**：Knowledge Architecture / Operations；此單元可作為 Agentic Guideline Intelligence 中的治理模組，供 AI 或設計系統檢查器引用。
11. **Visual Evidence**：此單元通常依賴示意圖、比例圖、色票、版面範例或錯誤示範；這些圖像提供比例、比較、警示與應用情境證據，而不只是裝飾。

1. **Chapter Purpose**：此單元的目的，是把「Glosario」轉成可執行的品牌治理接口；它避免使用者只憑審美直覺操作，並讓 設計治理的語義基礎 在不同媒介中維持一致。
