# Bambu Brand Style Guide

來源：`sources/bambu_Brand_Style_Guide_2020.pdf`  
版本：Vol 1  
日期：2020  
研究視角：Design Infrastructure Researcher

## Summary

Bambu 品牌風格指南以 21 頁短版工具包定義標誌、圖示、字體、色彩、應用與 Bambu/BambuGO/BambuBUILD 的溝通識別。此章把 Bambu 的初創金融科技品牌拆成可立即生產的視覺工具包，解決 logo、產品標記、字體與色彩在不同素材中快速失控的問題。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。原則是簡潔、技術感、可讀性、產品從屬於主品牌、跨線上線下媒介一致。框架是 Brand Element Kit：logo/icon -> lockup -> typography -> pictogram -> color -> applications。

執行上，這份 guideline 透過 Visuals Section、Our Logo and Icon System、Lockups and Logo Usage、Typography, Pictograms and Color 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。通用知識是短版品牌指南也應建立『核心品牌 -> 產品子品牌 -> 應用素材』的層級，避免產品 logo 搶走主品牌。

## Visuals Section

1. **Chapter Purpose**：此章把 Bambu 的初創金融科技品牌拆成可立即生產的視覺工具包，解決 logo、產品標記、字體與色彩在不同素材中快速失控的問題。
2. **Definition**：它定義主 logo、icon 版本、lockup、產品標記、字體階層、pictogram、色票與基本應用；邊界是不提供完整網頁 UI 或互動規範。
3. **Relationship**：Visuals Section 是整份短版指南的核心，後續 Communications Section 會重用相同 logo grid、text hierarchy 與產品識別。
4. **Design Execution**：作者用圖形建構、清楚留白、產品 logo 批准、字體級別與色票數值，將抽象的 fintech confidence 轉成可複製資產。
5. **Principles**：原則是簡潔、技術感、可讀性、產品從屬於主品牌、跨線上線下媒介一致。
6. **Frameworks**：框架是 Brand Element Kit：logo/icon -> lockup -> typography -> pictogram -> color -> applications。
7. **Patterns**：可複用模式包含 icon construction、watermark/vector/decorative 分流、clear space、product mark hierarchy、do-not examples。
8. **Rules**：可檢查規則：產品 mark 必須次於主 logo；clear space/exclusion zone 不得被侵入；2-tone vector 通常用於黑白背景，其他背景使用 monochrome；不得加陰影、旋轉、重製或以圖形包圍 logo；字體需使用 Gilroy/Lato 指定階層。
9. **Reusable Knowledge**：通用知識是短版品牌指南也應建立『核心品牌 -> 產品子品牌 -> 應用素材』的層級，避免產品 logo 搶走主品牌。
10. **Abstraction**：Guideline 模組類型：Visual Identity、Typography System、Color System、Iconography System、Operations。
11. **Visual Evidence**：視覺證據高度密集，包含 icon grid、golden ratio、clear space、logo misuse、字體樣張、pictogram、色票與名片/行銷素材 mockup。圖片承擔比例、禁例與可生產樣板功能。

## Communications Section

1. **Chapter Purpose**：此章把視覺系統連回品牌與產品敘事，避免 Bambu、BambuGO、BambuBUILD 只靠 logo 區分而缺少人格與溝通邏輯。
2. **Definition**：它定義品牌資訊、產品資訊、personality、archetype、values、icon concept、logo grid 與 text hierarchy；邊界是不提供完整文案模板。
3. **Relationship**：Communications Section 依賴 Visuals Section 的資產，並把 Bambu official 與產品品牌放入同一個識別家族。
4. **Design Execution**：作者用 archetype 與價值詞組把品牌/產品定位轉成可被文案與設計引用的資料卡。
5. **Principles**：原則是主品牌應承擔 inspirational/heroic 使命，產品品牌則以 caregiver、fast、reliable、easy 等屬性降低採用阻力。
6. **Frameworks**：框架是 Brand/Product Identity Card：info -> personality -> archetype -> values -> visual elements -> text hierarchy。
7. **Patterns**：可複用模式包含品牌資料卡、產品人格分流、同一字體階層跨品牌使用、icon concept 語意化。
8. **Rules**：可檢查規則：Bambu official 不應被描述成單純投資產品，而是 robo-advisory technology provider；BambuGO/BambuBUILD 應保留主品牌階層；產品價值需落在 fast/reliable/easy/driven/compassionate 等語意域。
9. **Reusable Knowledge**：通用知識是產品線指南可以用固定資料卡格式，讓每個子品牌在同一欄位下被比較與治理。
10. **Abstraction**：Guideline 模組類型：Brand Strategy、Messaging、Product Language、Knowledge Architecture。
11. **Visual Evidence**：視覺證據包含品牌/產品資料卡、產品 logo、icon concept、logo grid 與 text hierarchy。圖片功能是將人格與視覺元素對齊。

## Applications and Governance

1. **Chapter Purpose**：此章把前面的規則落到高頻素材，讓使用者看到品牌系統如何從資產變成可交付物。
2. **Definition**：它定義的是示例應用與聯絡出口；邊界是不列出完整生產規格或所有模板尺寸。
3. **Relationship**：Applications 承接 logo、type、color 與 communication cards，讓抽象品牌在 stationery 與 marketing 中被驗證。
4. **Design Execution**：作者使用 mockup 而非長篇文字，展示層級、訊息與資產如何被組合。
5. **Principles**：原則是品牌規則要能在真實接觸點中成立，且使用者遇到問題時要有聯絡入口。
6. **Frameworks**：框架是 Rule -> Example -> Contact Loop，用案例降低理解成本，用聯絡方式處理例外。
7. **Patterns**：可複用模式包含名片/信紙/行銷海報示例、社會證明訊息、聯絡頁收尾。
8. **Rules**：可檢查規則：應用素材需使用核准 logo、色彩與字體；聯絡資訊格式需完整；行銷素材不可讓產品 logo 高於主品牌；需要提供品牌查詢信箱或網站。
9. **Reusable Knowledge**：通用知識是 guideline 不應只列 token，也要提供少量代表性 touchpoint 作為使用者校準。
10. **Abstraction**：Guideline 模組類型：Operations、Examples、Governance。
11. **Visual Evidence**：視覺證據是名片、行銷視覺與 newsletter mockup。圖片功能是示範，不是完整模板規格。
