# Barbican Identity Guidelines

來源：`sources/Barbican_Centre_brand_guidelines_2017.pdf`  
日期：2017  
研究視角：Design Infrastructure Researcher

## Summary

Barbican identity guidelines 以 logo、lock-up、grid、Futura 字體、typography、art forms 與 design examples 建立高度系統化的藝文機構視覺識別。此章建立『identity is not just a logo』的前提，讓讀者理解 Barbican 的品牌一致性來自多個核心元素的共同作用。

最上層的原則是先建立判斷標準，再進入視覺、語氣或應用規格。原則是高度創意彈性仍需要清晰共同基礎，品牌可辨識度來自元素之間的持續關係。框架是 Core Elements -> Art Forms -> Design Examples，先建構規則，再展示分支與應用。

執行上，這份 guideline 透過 Core Elements Introduction、Contents、Introduction、Logo 等章節，把抽象原則落到可操作的資產、版面、語氣、流程或治理規則。通用知識是文化機構指南需要在創意彈性與共同結構間建立清楚契約。

## Core Elements Introduction

1. **Chapter Purpose**：此章建立『identity is not just a logo』的前提，讓讀者理解 Barbican 的品牌一致性來自多個核心元素的共同作用。
2. **Definition**：它定義文件架構與核心元素的角色；邊界是不在此提供每個元素的細節規格。
3. **Relationship**：Introduction 是 Core Elements、Art Forms 與 Design Examples 的導航層，讓後續規則被理解為一套設計方案而非孤立資產。
4. **Design Execution**：作者用內容架構與簡短宣告，把高度設計化的藝文品牌整理成可學習系統。
5. **Principles**：原則是高度創意彈性仍需要清晰共同基礎，品牌可辨識度來自元素之間的持續關係。
6. **Frameworks**：框架是 Core Elements -> Art Forms -> Design Examples，先建構規則，再展示分支與應用。
7. **Patterns**：可複用模式包含『不是 logo』開場、核心元素索引、分層目錄、以範例證明彈性。
8. **Rules**：可檢查規則：文件需能將每個應用追溯到 logo、lock-up、grid、typeface 或 typography；不得把單一 logo 使用視為完整 identity。
9. **Reusable Knowledge**：通用知識是文化機構指南需要在創意彈性與共同結構間建立清楚契約。
10. **Abstraction**：Guideline 模組類型：Knowledge Architecture、Brand Strategy、Visual Identity。
11. **Visual Evidence**：視覺證據包含目錄、核心元素介紹與版面範例，功能是建立系統地圖。

## Logo

1. **Chapter Purpose**：此章保護 Barbican 最可見的識別信號，並使圓形 logo 成為跨藝術節目的一致保證。
2. **Definition**：它定義 logo 出血定位、exclusion zone、背景清晰度與顏色對比；邊界是不定義所有版型內容。
3. **Relationship**：Logo 與 lock-up、grid 有強依賴，因為其位置與出血形式會影響整個版面系統。
4. **Design Execution**：作者將 logo 設計成需貼齊頁緣並出血的結構，讓它不是單純角落簽名，而是版面建築的一部分。
5. **Principles**：原則是清晰、對比、固定定位、品牌資產必須完整且可讀。
6. **Frameworks**：框架是 Logo Integrity：positioning、exclusion、contrast、special-case approval。
7. **Patterns**：可複用模式包含出血 logo、crop marks、exclusion zone、對比色規則與例外聯絡窗口。
8. **Rules**：可檢查規則：logo 應使用完整圓形 artwork；最終稿不得保留 crop marks；logo full-circle 必須與背景形成對比；Barbican word 必須與半圓形成黑/白或設計色對比；特殊例外需聯絡 printdesign。
9. **Reusable Knowledge**：通用知識是 logo 可以成為版面結構元素，但必須補上可見性與生產流程規則。
10. **Abstraction**：Guideline 模組類型：Visual Identity、Layout System、Governance、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據包含 logo 出血示意、crop marks、exclusion zone 與色彩對比範例，功能是比例與可讀性治理。

## Lock-up

1. **Chapter Purpose**：此章把品牌署名與實用資訊整合成固定結構，解決藝文機構需要同時露出節目、票務與聯絡資訊的問題。
2. **Definition**：它定義 lock-up 關係、字體、tracking/leading 不可改、不同格式定位與合作例外；邊界是不重做一般排版系統。
3. **Relationship**：Lock-up 是 Logo 與 Grid 之間的橋，讓品牌符號與必要資訊可以在多格式中穩定出現。
4. **Design Execution**：作者提供模板與固定 typography values，降低每次排版重新判斷的成本。
5. **Principles**：原則是資訊一致性、可讀性、固定關係不可任意變更，但特殊合作可受控調整。
6. **Frameworks**：框架是 Fixed Relationship + Format Placement + Special Case Exception。
7. **Patterns**：可複用模式包含品牌資訊 lock-up、模板索取、合作方共存、固定 typographic values。
8. **Rules**：可檢查規則：lock-up relationship 不得改變；Futura Book 可在 legibility issue 時改 Futura Bold；tracking/leading 不得調整；合作 lock-up 優先黑或反白並固定位置。
9. **Reusable Knowledge**：通用知識是品牌 lock-up 不只是 logo 排列，也可以承載常用營運資訊，且需治理例外。
10. **Abstraction**：Guideline 模組類型：Visual Identity、Information Design、Operations、Governance。
11. **Visual Evidence**：視覺證據包含 lock-up 模板、資訊排列與特殊合作示例，功能是示範固定關係與例外情境。

## Grid

1. **Chapter Purpose**：此章建立 Barbican 視覺系統的結構骨架，讓不同藝術節目與媒體材料仍有共同秩序。
2. **Definition**：它定義 format sets、vertical units、six-column grid 與圖文對齊方式；邊界是不規定每張海報的創意內容。
3. **Relationship**：Grid 支撐 logo、lock-up、typography 與 art form 應用，是創意彈性的共同底盤。
4. **Design Execution**：作者用共用 measurement units 與六欄結構，把複雜的行銷材料轉化成可重複布局。
5. **Principles**：原則是秩序、比例、平衡、跨格式一致，以及讓內容在框架內保有彈性。
6. **Frameworks**：框架是 Format Set -> Vertical Units -> Column Grid -> Element Alignment。
7. **Patterns**：可複用模式包含 format families、shared units、six-column literature grid、aligned balanced proportional rule。
8. **Rules**：可檢查規則：所有元素應對齊 grid；格式應使用核准 grid set；文字、圖片與 logo/lock-up 需保持平衡與比例；不得任意偏離欄位造成不專業版面。
9. **Reusable Knowledge**：通用知識是文化品牌的彈性不是無規則，而是建立可跨內容承載的 grid infrastructure。
10. **Abstraction**：Guideline 模組類型：Layout System、Design Tokens、Operations、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據包含 grid set、欄位圖、A5/A4/A3 等格式示意，功能是結構與比例證明。

## Typeface

1. **Chapter Purpose**：此章把 Barbican 的聲音轉成字體資產，讓品牌在多藝術形式中仍有一致文字表情。
2. **Definition**：它定義 Scangraphic Futura、可用字重、購買來源與 specimen；邊界是不等同完整 typography layout 規範。
3. **Relationship**：Typeface 是 Typography 章的材料層，先定義可用字體，再由排版規則決定如何使用。
4. **Design Execution**：作者用字重展示與 specimen 讓設計者清楚知道可用字體範圍與表現能力。
5. **Principles**：原則是乾淨、獨特、可讀，且同一字族應能表達 contemporary/classical qualities。
6. **Frameworks**：框架是 Font Source -> Weight Range -> Specimen Verification。
7. **Patterns**：可複用模式包含官方字庫來源、主要字重、替代粗體選項、完整 glyph specimen。
8. **Rules**：可檢查規則：Barbican identity 使用 Scangraphic Futura；Futura Bold 是主要行銷字重；Futura SB Bold 或 SH Bold 依既有模板使用；不得混用未核准 Futura 版本。
9. **Reusable Knowledge**：通用知識是字體指南要說明 font foundry/source，否則不同版本同名字體會造成視覺漂移。
10. **Abstraction**：Guideline 模組類型：Typography System、Operations、Design Tokens。
11. **Visual Evidence**：視覺證據是大量字體樣張與字重展示，功能是確認字形、字重與符號覆蓋。

## Typography

1. **Chapter Purpose**：此章把 Futura 從字體資產轉成可讀且具表現力的排版系統。
2. **Definition**：它定義文字排列、行距、字距、強調與階層；邊界是不限定所有內容模板。
3. **Relationship**：Typography 依賴 Typeface 與 Grid，並影響 Art Forms 與 Design Examples 的最終感受。
4. **Design Execution**：作者從 left range、rag control、sentence case 到 emphasis/hierarchy，逐步把可讀性轉成操作規則。
5. **Principles**：原則是功能、格式與目的決定排版；可讀性與整潔度是創意表達的前提。
6. **Frameworks**：框架是 Text Arrangement -> Spacing -> Emphasis -> Hierarchy。
7. **Patterns**：可複用模式包含左對齊規則、ragged edge 修整、禁止全大寫句子、字距與行距控制。
8. **Rules**：可檢查規則：Barbican typography always ranged left；句子不得 solely in capitals；ragged edge 應平衡；typographic elements 需共同支援 hierarchy。
9. **Reusable Knowledge**：通用知識是排版指南應處理 micro-typography，因為品牌質感常由字距、行距與 rag 決定。
10. **Abstraction**：Guideline 模組類型：Typography System、Accessibility、AI-Checkable Rules。
11. **Visual Evidence**：視覺證據包含文字塊、行距、字距與階層範例，功能是展示可讀性與品牌表情如何共存。

## Art Forms

1. **Chapter Purpose**：此章證明核心元素能支援多元藝術節目，而不需要每個 art form 另起品牌。
2. **Definition**：它定義各藝術形式如何在共同 identity 中表現差異；邊界是不替每個活動製定 campaign strategy。
3. **Relationship**：Art Forms 承接 Core Elements，並為 Design Examples 提供具體場景。
4. **Design Execution**：作者用分支頁面展示同一 grid、type、logo/lock-up 如何承載不同內容氣質。
5. **Principles**：原則是 diversity within system：藝術形式可不同，但底層識別不可斷裂。
6. **Frameworks**：框架是 Shared Core + Art-Form Expression。
7. **Patterns**：可複用模式包含依內容垂直分支示例、同系統多語氣展示、節目類型與品牌架構共存。
8. **Rules**：可檢查規則：每個 art form 應仍使用 Barbican core elements；不得建立與核心字體/grid/logo 無關的獨立視覺；分支差異應主要來自內容、圖像與版面節奏。
9. **Reusable Knowledge**：通用知識是多品類文化品牌可用同一設計基礎支援多節目，而非為每節目建立子品牌。
10. **Abstraction**：Guideline 模組類型：Brand Architecture、Examples、Visual Language。
11. **Visual Evidence**：視覺證據是各藝術形式的版面與宣傳素材，功能是展示系統彈性。

## Design Examples

1. **Chapter Purpose**：此章用案例補足規則無法完全描述的設計判斷，讓使用者校準『什麼看起來像 Barbican』。
2. **Definition**：它定義的是示範範圍與設計語感；邊界是不把所有案例變成硬性模板。
3. **Relationship**：Design Examples 是文件的驗證層，回頭證明 Logo、Lock-up、Grid、Typeface、Typography 與 Art Forms 可以組成高彈性輸出。
4. **Design Execution**：作者透過大量完成品示例，讓規則從抽象規範變成可被模仿的設計行為。
5. **Principles**：原則是範例應展示可能性，同時維持核心元素的可追溯性。
6. **Frameworks**：框架是 Rule Demonstration Gallery：每個案例都是多個核心元素的組合證據。
7. **Patterns**：可複用模式包含完成品 gallery、特殊 campaign 範例、核心元素追蹤、對設計自由度的視覺校準。
8. **Rules**：可檢查規則：案例應可辨識 logo/lock-up/grid/typeface 的使用；範例不得被當成任意偏離核心規則的授權；特殊案例需能說明例外理由。
9. **Reusable Knowledge**：通用知識是 guideline 最後需要 examples 作為 tacit knowledge 的載體，因為設計判斷不可能完全規則化。
10. **Abstraction**：Guideline 模組類型：Examples、Knowledge Architecture、Governance。
11. **Visual Evidence**：視覺證據是整章主體，功能包括示範、比較、風格校準與例外治理。
